## 一、消息可靠性保证

### 1. 生产者可靠性

```java
// 确认机制
@Configuration
public class KafkaConfig {
    
    @Bean
    public ProducerFactory<String, String> producerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ProducerConfig.ACKS_CONFIG, "all");  // 所有ISR确认
        props.put(ProducerConfig.RETRIES_CONFIG, 3);
        props.put(ProducerConfig.ENABLE_IDEMPOTENCE_CONFIG, true);  // 幂等
        return new DefaultKafkaProducerFactory<>(props);
    }
}

// 消息确认回调
@Service
public class ReliableProducer {
    
    public void sendWithConfirm(String topic, String key, String value) {
        ListenableFuture<SendResult<String, String>> future = 
            kafkaTemplate.send(topic, key, value);
        
        future.addCallback(
            result -> {
                RecordMetadata metadata = result.getRecordMetadata();
                log.info("发送成功: partition={}, offset={}", 
                    metadata.partition(), metadata.offset());
            },
            ex -> {
                log.error("发送失败，进入重试或死信队列", ex);
                // 保存到数据库或死信队列
                saveToDeadLetterQueue(topic, key, value);
            }
        );
    }
}
```

### 3. 消费者可靠性

```java
@Service
public class ReliableConsumer {
    
    @KafkaListener(topics = "order-topic")
    public void consume(ConsumerRecord<String, String> record, Acknowledgment ack) {
        String messageId = record.key();
        
        try {
            // 1. 幂等检查
            if (isProcessed(messageId)) {
                ack.acknowledge();
                return;
            }
            
            // 2. 业务处理
            processMessage(record.value());
            
            // 3. 标记已处理
            markAsProcessed(messageId);
            
            // 4. 提交offset
            ack.acknowledge();
            
        } catch (RetryableException e) {
            // 可重试异常，不提交offset
            log.warn("可重试异常，消息将重新消费", e);
            
        } catch (NonRetryableException e) {
            // 不可重试异常，进入死信队列
            log.error("不可重试异常，进入死信队列", e);
            sendToDeadLetterQueue(record);
            ack.acknowledge();  // 提交，避免无限重试
        }
    }
}
```

## 二、消息顺序性

### 1. 分区内有序

```
Kafka保证：
  - 同一分区内的消息有序
  - 不同分区的消息无序

实现顺序消费：
  1. 相关消息发送到同一分区
  2. 单消费者消费该分区
  3. 顺序处理消息
```

### 2. 分区策略

```java
// 自定义分区器
public class OrderPartitioner implements Partitioner {
    
    @Override
    public int partition(String topic, Object key, byte[] keyBytes, 
                        Object value, byte[] valueBytes, Cluster cluster) {
        // 按订单ID分区，保证同一订单的消息顺序
        if (key instanceof String) {
            String orderId = (String) key;
            return Math.abs(orderId.hashCode()) % cluster.partitionCountForTopic(topic);
        }
        return 0;
    }
}

// 配置
props.put(ProducerConfig.PARTITIONER_CLASS_CONFIG, OrderPartitioner.class);

// 发送时指定key
kafkaTemplate.send("order-topic", order.getId(), orderJson);
```

### 3. 顺序消费实现

```java
@Service
public class OrderedConsumer {
    
    // 单线程消费保证顺序
    @KafkaListener(
        topics = "order-topic",
        containerFactory = "singleThreadFactory"
    )
    public void consumeOrdered(ConsumerRecord<String, String> record) {
        // 单线程顺序处理
        processInOrder(record);
    }
    
    // 配置单线程消费
    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, String> singleThreadFactory() {
        ConcurrentKafkaListenerContainerFactory<String, String> factory = 
            new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConcurrency(1);  // 单线程
        factory.getContainerProperties().setAckMode(ContainerProperties.AckMode.MANUAL);
        return factory;
    }
}
```

## 三、死信队列

### 1. 配置

```java
@Configuration
public class DeadLetterConfig {
    
    @Bean
    public KafkaTemplate<String, String> kafkaTemplate() {
        return new KafkaTemplate<>(producerFactory());
    }
    
    // 死信队列Topic
    public static final String DLT_TOPIC = "order-topic.DLT";
}

// 消费失败转发到死信队列
@Service
public class DeadLetterHandler {
    
    @Autowired
    private KafkaTemplate<String, String> kafkaTemplate;
    
    public void sendToDLT(ConsumerRecord<String, String> record, Exception e) {
        // 原始消息 + 错误信息
        Map<String, Object> dltMessage = new HashMap<>();
        dltMessage.put("originalTopic", record.topic());
        dltMessage.put("originalPartition", record.partition());
        dltMessage.put("originalOffset", record.offset());
        dltMessage.put("key", record.key());
        dltMessage.put("value", record.value());
        dltMessage.put("error", e.getMessage());
        dltMessage.put("timestamp", System.currentTimeMillis());
        
        kafkaTemplate.send(DeadLetterConfig.DLT_TOPIC, record.key(), 
            JSON.toJSONString(dltMessage));
    }
    
    // 死信队列消费者
    @KafkaListener(topics = "order-topic.DLT")
    public void handleDLT(String message) {
        // 人工处理或告警
        log.error("死信队列消息: {}", message);
        // 发送告警
        alertService.sendAlert(message);
    }
}
```
