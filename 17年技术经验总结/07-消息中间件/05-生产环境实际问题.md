## 一、RabbitMQ问题

### 案例1：消息积压

**问题现象：**
队列中消息堆积，消费速度跟不上生产速度。

**分析过程：**
```bash
# 查看队列状态
rabbitmqctl list_queues name messages messages_ready messages_unacknowledged

# 查看消费者
rabbitmqctl list_consumers
```

**解决方案：**
1. 增加消费者数量
2. 提高消费速度（批量处理）
3. 优化消费者代码
4. 暂停生产端

---

### 案例2：队列满导致生产者阻塞

**问题现象：**
生产者发送消息失败，队列已满。

**分析过程：**
```bash
# 查看队列内存使用
rabbitmqctl list_queues name memory

# 查看告警
rabbitmqctl status
```

**解决方案：**
1. 增加内存限制
2. 设置队列长度限制
3. 死信队列处理
4. 清理过期消息

```bash
# 配置队列长度限制
rabbitmqctl set_policy max-length "^order\." \
  '{"max-length":100000}' --apply-to queues
```

## 二、Kafka问题

### 案例3：消费者Rebalance频繁

**问题现象：**
消费者频繁发生Rebalance，导致消费暂停。

**分析过程：**
```bash
# 查看消费者组
kafka-consumer-groups.sh --describe --group order-group

# 查看日志
# 发现：消费者心跳超时
```

**解决方案：**
```properties
# 调整配置
session.timeout.ms=30000        # 会话超时
heartbeat.interval.ms=10000     # 心跳间隔
max.poll.interval.ms=300000     # 两次poll最大间隔
max.poll.records=100            # 减少单次拉取数量
```

### 案例4：分区Leader切换频繁

**问题现象：**
分区Leader频繁切换，影响性能。

**分析过程：**
```bash
# 查看分区状态
kafka-topics.sh --describe --topic order-topic

# 查看Controller
kafka-broker-api-versions.sh --bootstrap-server localhost:9092
```

**解决方案：**
1. 检查Broker健康状况
2. 调整unclean.leader.election.enable=false
3. 监控Broker资源使用

### 案例5：消息丢失排查

**问题现象：**
部分消息未消费。

**排查步骤：**
```
1. 生产者确认：检查acks配置
2. Broker持久化：检查副本同步
3. 消费者提交：检查offset提交日志
4. 死信队列：检查DLT
```
