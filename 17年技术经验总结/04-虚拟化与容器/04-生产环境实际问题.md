## 一、Docker问题

### 案例1：容器内无法访问外网

**问题现象：**
容器内无法ping通外网，但宿主机正常。

**分析过程：**
```bash
# 1. 检查宿主机网络
ping 8.8.8.8

# 2. 检查容器网络
docker exec <container> ping 8.8.8.8
# 失败

# 3. 检查iptables
iptables -L -n -t nat
# 查看FORWARD链

# 4. 检查IP转发
cat /proc/sys/net/ipv4/ip_forward
# 如果为0，IP转发未开启
```

**解决方案：**
```bash
# 开启IP转发
sysctl -w net.ipv4.ip_forward=1

# 永久保存
echo "net.ipv4.ip_forward=1" >> /etc/sysctl.conf

# 允许Docker容器流量
iptables -A FORWARD -i docker0 -o eth0 -j ACCEPT
iptables -A FORWARD -i eth0 -o docker0 -j ACCEPT

# 重启Docker
systemctl restart docker
```

### 案例2：OOM被Kill

**问题现象：**
Docker容器被OOM Killer杀死。

**分析过程：**
```bash
# 1. 查看容器状态
docker ps -a | grep Exited

# 2. 查看日志
docker logs <container>

# 3. 查看系统日志
dmesg | grep -i "killed process"

# 4. 检查内存限制
docker inspect <container> | grep -i memory
```

**解决方案：**
1. 增加内存限制
```bash
docker run -m 2g nginx
```

2. 优化应用内存使用
3. 添加Swap
```bash
docker run --memory-swap=4g --memory=2g nginx
# memory-swap必须大于memory
```

## 二、K8s问题

### 案例3：Pod一直处于Pending

**问题现象：**
Pod创建后一直处于Pending状态。

**分析过程：**
```bash
# 1. 查看详情
kubectl describe pod <name>
# 查看Events

# 2. 常见原因：
# - 资源不足（CPU/内存）
# - 没有匹配的Node
# - PVC未绑定
# - 调度失败

# 3. 检查Node资源
kubectl describe nodes | grep -A 5 "Allocated resources"
```

**解决方案：**
```bash
# 1. 扩容Node
# 2. 减少Pod资源请求
resources:
  requests:
    memory: "64Mi"
    cpu: "50m"
  limits:
    memory: "128Mi"
    cpu: "100m"

# 3. 清理Evicted Pod
kubectl delete pods --field-selector=status.phase=Evicted
```

### 案例4：Ingress访问异常

**问题现象：**
通过Ingress访问服务返回404或502。

**分析过程：**
```bash
# 1. 检查Ingress配置
kubectl describe ingress <name>

# 2. 检查Backend Service
kubectl get ingress -o wide

# 3. 检查Service Endpoints
kubectl get endpoints <service-name>

# 4. 测试直接访问Service
kubectl run test --rm -it --image=busybox -- wget -qO- http://<service>:80
```

**解决方案：**
```yaml
# 检查Ingress配置
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: app-ingress
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
spec:
  rules:
  - host: example.com
    http:
      paths:
      - path: /api
        pathType: Prefix
        backend:
          service:
            name: api-service
            port:
              number: 80
```

## 三、监控与日志

### 日志收集架构

```yaml
# Fluent Bit收集日志
apiVersion: v1
kind: ConfigMap
metadata:
  name: fluent-bit-config
data:
  fluent-bit.conf: |
    [SERVICE]
        Flush         5
        Log_Level     info

    [INPUT]
        Name              tail
        Path              /var/log/containers/*.log
        Parser            docker
        Tag               containers.*
        Refresh_Interval  5

    [OUTPUT]
        Name              es
        Match             containers.*
        Host              elasticsearch
        Port              9200
        Logstash_Format   On
```
