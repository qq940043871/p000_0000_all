## 一、理论基础

### 1. 虚拟化概述

```
虚拟化分类：

硬件虚拟化（全虚拟化/半虚拟化）
  - QEMU/KVM、Xen、VMware、Hyper-V
  - 模拟完整硬件环境
  - Guest OS无需修改

操作系统虚拟化（容器）
  - Docker、LXC、Podman
  - 共享宿主机内核
  - 轻量级隔离

语言级虚拟化
  - JVM、.NET CLR
  - 虚拟机字节码
  - 平台无关

模拟器
  - Bochs、Valgrind
  - 软件模拟CPU指令
  - 速度慢，用于调试
```

### 2. KVM/QEMU架构

```
┌─────────────────────────────────────────────────┐
│                    QEMU                          │  用户态进程
│  ┌──────────┐  ┌──────────┐  ┌──────────────┐  │
│  │ Device   │  │  Memory  │  │ CPU Emulator │  │
│  │ Emulator │  │  Manager │  │              │  │
│  └────┬─────┘  └────┬─────┘  └──────┬───────┘  │
│       │             │               │           │
│       └─────────────┴───────────────┘           │
│                    │                              │
│            ┌───────▼────────┐                    │
│            │  KVM内核模块    │                    │
│            │  (/dev/kvm)     │                    │
│            └───────┬────────┘                    │
└────────────────────┼────────────────────────────┘
                     │ ioctl系统调用
┌────────────────────▼────────────────────────────┐
│              Linux Kernel                         │
│  ┌──────────────────────────────────────────┐  │
│  │              Virtualization Extensions     │  │
│  │           (Intel VT-x / AMD-V)            │  │
│  └──────────────────────────────────────────┘  │
│  ┌────────┐ ┌────────┐ ┌────────┐             │
│  │ VM #1  │ │ VM #2  │ │ VM #3  │  ← 虚拟机    │
│  └────────┘ └────────┘ └────────┘             │
└─────────────────────────────────────────────────┘

工作原理：
  QEMU：模拟硬件设备，处理I/O
  KVM：提供CPU/内存虚拟化支持
  组合：QEMU调用KVM实现高性能虚拟化
```

### 3. CPU虚拟化

```
两种CPU虚拟化方式：

1. 陷入-模拟模式（全虚拟化）
   - VMM捕获特权指令
   - 模拟执行
   - 速度较慢

2. 硬件辅助虚拟化（半虚拟化）
   - Intel VT-x / AMD-V
   - VMX root/non-root模式
   - CPU指令直接执行
   - 性能接近物理机

VMCS（Virtual Machine Control Structure）：
  - 保存虚拟机CPU状态
  - VM Entry / VM Exit
  - I/O bitmap、MSRbitmap

CPU虚拟化流程：
  Guest执行指令
    → VMX non-root模式
      → 如果是敏感指令 → VM Exit到VMM
        → VMM模拟执行
      → 否则直接执行（VMX non-root）
    → VM Entry返回Guest
```

### 4. 内存虚拟化

```
内存虚拟化原理：

Guest虚拟地址（GVA）→ Guest物理地址（GPA）→ Host虚拟地址（HVA）→ Host物理地址（HPA）

EPT（Extended Page Table，Intel）：
  - 两级页表自动转换
  - 减少内存开销
  - 内存访问额外开销约5-10%

影子页表（Shadow Page Table）：
  - 早期方案（无EPT时）
  - VMM维护实际页表
  - 开销大，已被EPT替代
```

### 5. 网络虚拟化

```
虚拟网络类型：

1. 桥接模式（Bridged）
   VM ──┐
        ├── br0 ─── eth0 ─── 物理网络
   VM ──┘

2. NAT模式
   VM ── veth ── docker0 ── eth0 ─── 物理网络
   VM共享宿主机IP

3. Host模式
   VM共享宿主机网络命名空间

4. Overlay网络
   VM ── veth ── VXLAN/GRE ── 物理网络
   跨主机容器网络
```

## 二、实践应用

### 1. KVM管理

```bash
# 安装
apt install qemu-kvm libvirt-daemon-system libvirt-clients virt-manager

# virsh命令
virsh list --all              # 列出所有VM
virsh start <vm-name>         # 启动
virsh shutdown <vm-name>     # 关机
virsh destroy <vm-name>      # 强制停止
virsh reboot <vm-name>       # 重启

# 创建VM
virt-install \
  --name=centos7 \
  --vcpus=2 \
  --memory=4096 \
  --disk path=/var/lib/libvirt/images/centos7.qcow2,size=40 \
  --os-variant=rhel7.6 \
  --network bridge=br0 \
  --graphics vnc \
  --location=/path/to/centos7.iso

# 导出/导入
virsh dumpxml <vm-name> > vm.xml
virsh define vm.xml

# 快照
virsh snapshot-create <vm-name>
virsh snapshot-list <vm-name>
virsh snapshot-revert <vm-name> <snapshot-name>
```

### 2. 存储管理

```bash
# 创建qcow2镜像
qemu-img create -f qcow2 /path/to/disk.qcow2 100G

# 格式转换
qemu-img convert -f raw -O qcow2 input.raw output.qcow2

# 扩容
qemu-img resize /path/to/disk.qcow2 +50G

# 查看信息
qemu-img info /path/to/disk.qcow2

# 快照
qemu-img snapshot -c snap1 /path/to/disk.qcow2
qemu-img snapshot -a snap1 /path/to/disk.qcow2
qemu-img snapshot -d snap1 /path/to/disk.qcow2
```

### 3. 性能优化

```bash
# CPU绑定（提升性能）
virsh emulatorpin <vm-name> --cpulist 0-3

# 内存大页
# /etc/default/grub
GRUB_CMDLINE_LINUX="default_hugepagesz=2MB hugepagesz=2MB hugepages=1024"

# IO调度器（SSD用noop）
virsh setemb丝ksched <vm-name> --scheduler noop

# virtio驱动（半虚拟化，高性能）
# 创建VM时指定
--disk path=/path/to/disk.qcow2,driver=virtio
--network model=virtio

# NUMA亲和性
virsh numatune <vm-name> --nodeset 0 --mode strict
```

## 三、生产环境问题案例

### 案例1：虚拟机性能差

**问题现象：**
KVM虚拟机CPU和IO性能明显低于物理机。

**分析过程：**
```bash
# 1. 检查虚拟化支持
grep -E '(vmx|svm)' /proc/cpuinfo
# 有输出说明CPU支持

# 2. 检查virtio驱动
virsh dumpxml <vm-name> | grep -E 'disk|interface'
# 如果是ide而不是virtio，说明未优化

# 3. 检查CPU模式
virsh dumpxml <vm-name> | grep cpu
# <cpu mode='host-passthrough'/>
# mode='host-model' 性能较差

# 4. 检查IO队列
# 默认队列深度可能不够
```

**解决方案：**
1. 使用virtio驱动
```xml
<disk type='file' device='disk'>
  <driver name='qemu' type='qcow2'/>
  <source file='/path/to/disk.qcow2'/>
  <target dev='vda' bus='virtio'/>
</disk>
```

2. CPU模式改为host-passthrough
```xml
<cpu mode='host-passthrough'/>
```

3. 启用CPU大页
```xml
<memoryBacking>
  <hugepages/>
</memoryBacking>
```

**经验教训：**
- virtio驱动是KVM性能关键
- CPU模式影响很大
- 存储和网络都要用virtio

### 案例2：虚拟机无法启动

**问题现象：**
虚拟机启动报错，无法进入系统。

**分析过程：**
```bash
# 1. 查看错误日志
virsh console <vm-name>
journalctl -u libvirtd

# 2. 检查磁盘文件
qemu-img check /path/to/disk.qcow2

# 3. 检查网络
virsh dumpxml <vm-name> | grep bridge

# 4. 检查资源
virsh nodecpustats
virsh nodememstats
```

**常见原因：**
1. 磁盘文件损坏
2. 网络桥接配置错误
3. 内存不足
4. 存储空间不足

**解决方案：**
```bash
# 修复磁盘
qemu-img check -r all /path/to/disk.qcow2

# 从快照恢复
qemu-img snapshot -a <snapshot-name> /path/to/disk.qcow2

# 扩容磁盘
qemu-img resize /path/to/disk.qcow2 +20G
# 然后在VM内扩展分区和文件系统
```
