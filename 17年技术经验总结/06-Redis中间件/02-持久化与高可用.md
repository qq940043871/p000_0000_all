# Redis持久化与高可用

> 模块：Redis中间件
> 更新时间：2026-03-28

## 一、持久化

### 1. RDB

```
RDB（Redis Database）：
  - 全量快照
  - fork子进程生成dump.rdb
  - 自动触发：save 900 1（900秒内1次写就保存）
  - 适合备份

原理：
  BGSAVE → fork子进程 → 子进程遍历内存写入RDB文件
```

### 2. AOF

```
AOF（Append Only File）：
  - 增量日志
  - 每次写操作记录
  - 配置：appendfsync always/everysec/no
  - 支持重写压缩

appendfsync选项：
  always：每次写都刷盘，最安全但最慢
  everysec：每秒刷盘，默认，足够安全
  no：依赖OS，性能最好但可能丢数据

AOF重写：
  BGREWRITEAOF → 压缩日志文件
  自动触发：上次重写后文件增长100%
```

### 3. 混合持久化（5.0+）

```
混合模式：
  - RDB格式开头（基准数据）
  - AOF格式后续（增量操作）
  - 恢复时先加载RDB，再加载AOF

开启：
  aof-use-rdb-preamble yes
```

## 二、主从复制

### 1. 复制配置

```bash
# 主库配置
bind 0.0.0.0
port 6379

# 从库配置
replicaof master_ip 6379
replica-serve-stale-data yes  # 主库故障时是否响应
replica-read-only yes

# 增量复制
repl-diskless-sync yes   # 无盘复制
repl-backlog-size 10mb   # 增量复制缓冲区
```

### 2. 哨兵（Sentinel）

```bash
# 哨兵配置
cat > sentinel.conf << 'EOF'
port 26379
sentinel monitor mymaster master_ip 6379 2
sentinel down-after-milliseconds mymaster 30000
sentinel parallel-syncs mymaster 1
sentinel failover-timeout mymaster 180000
EOF

# 启动哨兵
redis-sentinel sentinel.conf
# 或
redis-server sentinel.conf --sentinel
```

### 3. 集群（Cluster）

```bash
# 创建集群
redis-cli --cluster create node1:7000 node2:7000 node3:7000 \
  --cluster-replicas 1

# 查看集群状态
redis-cli -c -p 7000 cluster nodes
redis-cli -c -p 7000 cluster info
redis-cli -c -p 7000 cluster slots

# 故障转移
redis-cli -c -p 7000 cluster failover
```

## 三、生产问题

### 案例：RDB写入阻塞

**问题现象：**
Redis在BGSAVE时短暂阻塞。

**分析过程：**
```bash
redis-cli INFO stats | grep rdb
# 查看BGSAVE状态
```

**解决方案：**
```bash
# 1. 使用AOF替代RDB
appendonly yes
appendfsync everysec

# 2. 配置自动AOF重写
auto-aof-rewrite-percentage 100
auto-aof-rewrite-min-size 64mb

# 3. 低峰期手动BGSAVE
```
