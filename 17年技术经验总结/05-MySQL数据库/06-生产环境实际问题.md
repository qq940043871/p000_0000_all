## 一、崩溃恢复

### 案例1：MySQL无法启动

**问题现象：**
MySQL服务启动失败。

**分析过程：**
```bash
# 1. 查看错误日志
tail -100 /var/log/mysql/error.log

# 2. 常见错误：
# - ibdata1损坏
# - ib_logfile损坏
# - 无法找到表空间文件
```

**解决方案：**
```bash
# 1. 恢复ib_logfile
mv /var/lib/mysql/ib_logfile0 /var/lib/mysql/ib_logfile0.bak
mv /var/lib/mysql/ib_logfile1 /var/lib/mysql/ib_logfile1.bak
# MySQL会自动创建新的日志文件

# 2. 恢复表空间
# 如果有备份
innodb_force_recovery = 1  # 临时启动
# 导出数据
mysqldump --all-databases > backup.sql
# 重新初始化MySQL
# 导入数据
```

## 二、数据恢复

### 案例2：误删除数据恢复

**问题现象：**
执行了DELETE/UPDATE忘记WHERE条件。

**分析过程：**
```sql
-- 1. 检查binlog
SHOW MASTER LOGS;
SHOW BINLOG EVENTS IN 'mysql-bin.000001' LIMIT 10;

-- 2. 找到误操作的position
```

**解决方案：**
1. 利用binlog恢复
```bash
mysqlbinlog mysql-bin.000001 --start-position=100 --stop-position=200 > recovery.sql
mysqlbinlog mysql-bin.000001 | grep -A5 'DELETE FROM users' > recovery.sql
```

2. 使用延迟从库
3. 定期备份

## 三、性能问题

### 案例3：连接复用导致性能差

**问题现象：**
短连接频繁建立断开，响应时间长。

**分析过程：**
```sql
-- 查看连接建立次数
SHOW STATUS LIKE 'Connections';
SHOW STATUS LIKE 'Uptime';
-- 计算平均每秒连接数

-- 查看最大连接
SHOW VARIABLES LIKE 'max_connections';
```

**解决方案：**
1. 使用连接池（推荐）
2. 配置持久连接
```php
# PHP
mysqli_connect('p:' . $host, $user, $pass);
```
3. 使用ProxySQL中间件
4. 设置合理的连接超时
