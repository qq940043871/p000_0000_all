## 一、CAP与BASE

### 1. CAP理论

```
C - Consistency（一致性）
A - Availability（可用性）
P - Partition Tolerance（分区容错性）

在分布式系统中，P是必须的：
  CP：保证一致性，牺牲可用性（如ZooKeeper）
  AP：保证可用性，牺牲强一致性（如Cassandra）
  CA：单机系统（无分区）
```

### 2. BASE理论

```
Basically Available（基本可用）
  - 允许部分降级
  - 响应时间损失
  - 功能降级

Soft State（软状态）
  - 允许中间状态
  - 数据不一致是常态

Eventually Consistent（最终一致性）
  - 最终达到一致
  - 时间窗口内不一致
```

## 二、分布式事务

### 1. 两阶段提交（2PC）

```
阶段1：Prepare
  协调者 → 参与者：准备提交
  参与者 → 协调者：同意/拒绝

阶段2：Commit/Abort
  全部同意 → 提交
  有拒绝 → 回滚

缺点：
  - 同步阻塞
  - 单点故障
  - 数据不一致
```

### 2. 三阶段提交（3PC）

```
阶段1：CanCommit
阶段2：PreCommit
阶段3：DoCommit

改进：
  - 超时机制
  - PreCommit减少阻塞
```

### 3. TCC模式

```java
public interface OrderService {
    // Try：预留资源
    boolean tryCreateOrder(OrderDTO orderDTO);
    
    // Confirm：确认提交
    boolean confirmCreateOrder(OrderDTO orderDTO);
    
    // Cancel：取消预留
    boolean cancelCreateOrder(OrderDTO orderDTO);
}
```

### 4. 本地消息表

```sql
-- 本地事务 + 消息表
CREATE TABLE local_message (
    id BIGINT PRIMARY KEY,
    biz_id VARCHAR(64),
    message TEXT,
    status TINYINT,  -- 0:待发送 1:已发送
    created_at TIMESTAMP
);

-- 事务中同时操作业务表和消息表
BEGIN;
INSERT INTO orders ...;
INSERT INTO local_message (id, biz_id, message, status) VALUES (...);
COMMIT;

-- 定时任务扫描发送
SELECT * FROM local_message WHERE status = 0 LIMIT 100;
```

## 三、分布式ID

### 1. UUID

```
优点：
  - 本地生成
  - 无需协调

缺点：
  - 无序
  - 过长（36字符）
  - 索引性能差
```

### 2. 数据库自增

```
优点：
  - 简单
  - 有序

缺点：
  - 单点故障
  - 性能瓶颈
  - 分库分表不适用
```

### 3. 号段模式

```sql
CREATE TABLE id_segment (
    biz_type VARCHAR(64) PRIMARY KEY,
    max_id BIGINT,
    step INT,
    version INT
);

-- 获取号段
UPDATE id_segment 
SET max_id = max_id + step, version = version + 1
WHERE biz_type = 'order' AND version = ?;

-- 批量分配ID
```

### 4. Snowflake

```
64位ID结构：
| 1位符号 | 41位时间戳 | 10位机器ID | 12位序列号 |

优点：
  - 有序
  - 高性能
  - 分布式

缺点：
  - 时钟回拨问题
```

## 四、分布式锁

### 1. Redis实现

```java
public class RedisLock {
    
    public boolean tryLock(String key, String value, long expireSeconds) {
        return redis.set(key, value, "NX", "EX", expireSeconds);
    }
    
    public void unlock(String key, String value) {
        String script = 
            "if redis.call('get', KEYS[1]) == ARGV[1] then " +
            "  return redis.call('del', KEYS[1]) " +
            "else return 0 end";
        redis.eval(script, 1, key, value);
    }
}
```

### 2. ZooKeeper实现

```java
public class ZkLock {
    
    public void lock(String path) {
        // 创建临时顺序节点
        String node = zk.create(path + "/lock-", 
            EPHEMERAL | SEQUENTIAL);
        
        // 获取所有节点
        List<String> children = zk.getChildren(path);
        
        // 判断是否最小节点
        if (isMinNode(node, children)) {
            return;  // 获得锁
        }
        
        // 监听前一个节点
        String prevNode = getPrevNode(node, children);
        zk.exists(prevNode, watcher);
        
        // 等待
        wait();
    }
}
```
