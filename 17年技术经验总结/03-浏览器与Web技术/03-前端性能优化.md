## 一、理论基础

### 1. 关键渲染路径

```
关键渲染路径（Critical Rendering Path）：

HTML ──► DOM ──► CSSOM ──► Render Tree ──► Layout ──► Paint ──► Composite
         ↓              ↓
      阻塞渲染      JS执行

阻塞因素：
  HTML解析：遇到<script>暂停（无defer/async）
  CSS加载：阻止渲染（放入<head>）
  JS执行：阻止DOM构建

优化目标：
  - 最小化关键资源
  - 最小化关键字节数
  - 最小化关键路径长度
```

### 2. 资源加载优先级

```
浏览器自动优先级：
  高优先级：CSS、字体、preload资源
  中优先级：JS脚本（同步）
  低优先级：图片、异步脚本

手动控制优先级：
  - <link rel="preload">：提前加载
  - <link rel="prefetch">：空闲时下载
  - <link rel="preconnect">：预建立连接
  - <link rel="dns-prefetch">：预DNS解析

示例：
  <link rel="preload" href="font.woff2" as="font" crossorigin>
  <link rel="preconnect" href="https://api.example.com">
  <link rel="prefetch" href="next-page.html">
```

### 3. 性能指标

```
Core Web Vitals（核心Web指标）：

LCP（Largest Contentful Paint）：
  - 最大内容绘制
  - 衡量加载性能
  - 目标：< 2.5秒

FID/INP（First Input Delay / Interaction to Next Paint）：
  - 首次输入延迟
  - 衡量交互性
  - 目标：< 100毫秒

CLS（Cumulative Layout Shift）：
  - 累计布局偏移
  - 衡量稳定性
  - 目标：< 0.1

其他指标：
  TTFB（Time to First Byte）：首字节时间
  FCP（First Contentful Paint）：首次内容绘制
  TTDI（Time to DOM Interactive）：DOM可交互时间
```

### 4. 缓存策略

```
缓存层次：
  浏览器缓存 → CDN缓存 → 反向代理缓存 → 应用缓存

HTTP缓存头：
  强缓存（不发请求）：
    Cache-Control: max-age=31536000, immutable
    Expires: Wed, 21 Oct 2026 07:28:00 GMT
  
  协商缓存（发请求验证）：
    Last-Modified: Wed, 21 Oct 2020 07:28:00 GMT
    If-Modified-Since: Wed, 21 Oct 2020 07:28:00 GMT
    
    ETag: "33a64df551425fcc55e4d42a148795d9"
    If-None-Match: "33a64df551425fcc55e4d42a148795d9"

缓存策略选择：
  - 静态资源（JS/CSS/图片）：长期缓存 + hash命名
  - HTML：协商缓存或短期强缓存
  - API响应：按业务需求
```

### 3. CSS优化

```css
/* 1. 关键CSS内联 */
<style>
  /* 首屏必需样式 */
  .header { background: #333; }
  .hero { min-height: 100vh; }
</style>

/* 2. 避免阻塞渲染的CSS */
<!-- 非关键CSS异步加载 -->
<link rel="preload" href="styles.css" as="style" onload="this.rel='stylesheet'">
<noscript><link rel="stylesheet" href="styles.css"></noscript>

/* 3. 使用CSS变量 */
:root {
  --primary-color: #333;
}
.button { color: var(--primary-color); }

/* 4. 减少复杂选择器 */
.button { color: #333; }           /* 快 */
div.container .button { color: #333; }  /* 慢 */

/* 5. will-change优化动画 */
.animated {
  will-change: transform, opacity;
  transition: transform 0.3s;
}
```

### 4. JavaScript优化

```javascript
// 1. 防抖/节流
function debounce(func, wait) {
  let timeout;
  return function(...args) {
    clearTimeout(timeout);
    timeout = setTimeout(() => func.apply(this, args), wait);
  };
}

function throttle(func, limit) {
  let inThrottle;
  return function(...args) {
    if (!inThrottle) {
      func.apply(this, args);
      inThrottle = true;
      setTimeout(() => inThrottle = false, limit);
    }
  };
}

// 2. 虚拟滚动
// react-window / vue-virtual-scroller
import { FixedSizeList } from 'react-window';
<FixedSizeList height={400} itemCount={10000} itemSize={50}>
  {({ index, style }) => (
    <div style={style}>Row {index}</div>
  )}
</FixedSizeList>

// 3. Web Worker
const worker = new Worker('heavy-task.js');
worker.postMessage({ data: largeArray });
worker.onmessage = (e) => {
  console.log('Result:', e.data);
};

// 4. 懒加载组件
const HeavyComponent = React.lazy(() => import('./HeavyComponent'));
<Suspense fallback={<Loading />}>
  <HeavyComponent />
</Suspense>
```

## 三、生产环境问题案例

### 案例1：首屏加载慢

**问题现象：**
用户进入页面要等待3-5秒才看到内容。

**分析过程：**
```javascript
// Lighthouse分析
// Performance: 30分
// 主要问题：
//   - JSBundle过大（2MB）
//   - 关键CSS未内联
//   - 图片未优化

// Network面板分析
// main.js: 800KB（未压缩）
// vendor.js: 1200KB
// 未使用gzip/brotli压缩
```

**解决方案：**
1. 代码分割，按需加载
2. 关键CSS内联
3. 启用压缩（gzip/brotli）
4. 图片优化（WebP + 懒加载）
5. CDN加速

```nginx
# Nginx配置
gzip on;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
gzip_min_length 1000;
gzip_comp_level 6;
gzip_vary on;
```

**效果：** Lighthouse性能分从30提升到85。

### 案例2：滚动时图片加载闪烁

**问题现象：**
页面滚动时，图片加载导致页面跳动。

**分析过程：**
```css
/* 分析原因 */
img {
  /* 没有设置宽高 */
}

/* 或者 */
.container img {
  /* 实际尺寸和显示尺寸不符 */
}
```

**解决方案：**
```css
/* 1. 设置宽高比 */
img {
  width: 100%;
  height: auto;
  aspect-ratio: 16 / 9;  /* 预留空间 */
  background-color: #f0f0f0;  /* 占位背景 */
}

/* 2. object-fit */
img {
  width: 100%;
  height: 100%;
  object-fit: cover;
}

/* 3. 使用aspect-ratio容器 */
.aspect-ratio-16-9 {
  aspect-ratio: 16 / 9;
  background: #f0f0f0;
}
```

**经验教训：**
- 始终为图片设置尺寸
- 使用aspect-ratio预留空间
- 骨架屏提升体验

### 案例3：大量数据渲染卡顿

**问题现象：**
表格渲染10000行数据，页面卡死。

**分析过程：**
```javascript
// 问题代码
const rows = data.map(item => (
  <tr key={item.id}>
    <td>{item.name}</td>
    <td>{item.value}</td>
  </tr>
));
return <tbody>{rows}</tbody>;
// 一次性创建10000个DOM节点
```

**解决方案：**
1. 虚拟滚动（只渲染可视区域）
```javascript
import { FixedSizeList } as List from 'react-window';
<List height={600} width="100%" itemCount={10000} itemSize={50}>
  {({ index, style }) => (
    <div style={style}>
      Row {index}
    </div>
  )}
</List>
```

2. 分页加载
3. 无限滚动 + 加载更多
4. Web Worker处理数据

**效果：** 从卡死3秒变为流畅60fps。
