# 1.3 术语表

> 本术语表收录消息中间件相关的所有核心概念，统一中英文对照，便于团队沟通和文档编写。

---

## 基础概念

| 术语（EN）| 术语（CN）| 别名 | 定义 |
|-----------|-----------|------|------|
| Message Broker | 消息代理 | Broker / 代理节点 | 消息中间件的服务进程，负责接收、存储和转发消息 |
| Message Queue | 消息队列 | Queue | 消息的临时存储容器，采用 FIFO 顺序（可配置）|
| Message Topic | 消息主题 | Topic | 发布订阅模式下的消息分类标识 |
| Message | 消息 | 消息体 | 从生产者传递到消费者的业务数据单元 |
| Payload | 消息体 | Body | 消息的实际业务数据内容 |
| Header | 消息头 | 消息元数据 | 消息的系统属性（ID、时间戳、类型等）|
| Property | 消息属性 | 扩展属性 | 消息的用户自定义键值对 |

---

## 生产者与消费者

| 术语（EN）| 术语（CN）| 定义 |
|-----------|-----------|------|
| Producer | 生产者 | 向消息中间件发送消息的客户端应用 |
| Consumer | 消费者 | 从消息中间件接收并处理消息的客户端应用 |
| Consumer Group | 消费者分组 | 多个消费者组成的小组，协同消费同一队列，实现负载均衡 |
| Subscription | 订阅关系 | 消费者与主题之间的关联，决定消费者接收哪些消息 |
| Client ID | 客户端标识 | 生产者或消费者的唯一标识，用于追踪和管理 |

---

## 路由与交换机

| 术语（EN）| 术语（CN）| 定义 |
|-----------|-----------|------|
| Exchange | 交换机 | 消息进入 Broker 后的第一站，负责根据规则路由到队列 |
| Binding | 绑定关系 | 交换机与队列之间的关联规则 |
| Routing Key | 路由键 | 生产者发送消息时指定的键，交换机依据它决定路由方向 |
| Binding Key | 绑定键 | 绑定规则中的模式匹配键，与 Routing Key 配合实现灵活路由 |
| Exchange Type | 交换机类型 | 交换机的路由策略：Direct / Fanout / Topic / Headers |

---

## 交换机类型详解

| 类型 | 英文 | 路由规则 |
|------|------|----------|
| 直连交换机 | Direct | 精确匹配 Routing Key = Binding Key |
| 广播交换机 | Fanout | 无条件投递到所有绑定的队列 |
| 主题交换机 | Topic | 按通配符规则匹配（`*` 匹配一个词，`#` 匹配零个或多个词）|
| 首部交换机 | Headers | 根据消息头的键值对而非路由键进行匹配 |

---

## 消息生命周期

| 术语（EN）| 术语（CN）| 定义 |
|-----------|-----------|------|
| Pending | 待消费 | 消息已到达队列，等待消费者拉取 |
| In-Flight | 飞行中 | 消息已被消费者拉取，但尚未确认（ACK）|
| Acknowledged | 已确认 | 消费者成功处理消息，Broker 删除该消息 |
| Unacknowledged | 未确认 | 消费者处理失败或超时，消息待重投 |
| Dead Letter | 死信 | 经最大重投次数仍失败的消息，进入死信队列 |
| Expired | 已过期 | 超过 TTL 的消息被自动清除 |

---

## 消息特性

| 术语（EN）| 术语（CN）| 定义 |
|-----------|-----------|------|
| Persistent | 持久消息 | 消息会写入磁盘，Broker 重启后不丢失 |
| Transient | 非持久消息 | 消息仅存在内存，Broker 重启后丢失 |
| Priority | 消息优先级 | 高优先级消息可优先被消费（0-9，9最高）|
| TTL | 生存时间 | 消息的最大存活时长，超过后自动删除 |
| Delay | 延迟消息 | 消息延迟指定时间后才可被消费 |
| Scheduled | 定时消息 | 消息在指定时间点才可被消费 |
| Redelivered | 重投消息 | 因消费者未确认而重新投递的消息 |

---

## 消费模式

| 术语（EN）| 术语（CN）| 定义 |
|-----------|-----------|------|
| Pull Mode | 拉取模式 | 消费者主动从 Broker 拉取消息 |
| Push Mode | 推送模式 | Broker 主动将消息推送给消费者 |
| At Least Once | 至少一次 | 消息一定会被消费，但可能出现重复 |
| At Most Once | 至多一次 | 消息最多被消费一次，可能出现丢失 |
| Exactly Once | 恰好一次 | 消息恰好被消费一次（代价高昂）|

---

## 存储与索引

| 术语（EN）| 术语（CN）| 定义 |
|-----------|-----------|------|
| Segment | 消息段 | 存储文件中连续的消息集合，按偏移量有序排列 |
| Index | 索引 | 消息偏移量到物理位置的映射，加速消息查询 |
| WAL | 预写日志 | Write-Ahead Logging，保证数据持久性的日志机制 |
| Commit Log | 提交日志 | 消息写入的主日志，所有消息追加写入 |
| Compaction | 日志压缩 | 删除同一 Key 的旧数据，保留最新值（Kafka 类系统）|

---

## 集群与高可用

| 术语（EN）| 术语（CN）| 定义 |
|-----------|-----------|------|
| Leader / Follower | 主从节点 | 主节点处理读写，从节点同步数据 |
| Quorum | 多数派 | 分布式一致性的法定节点数量（如 3 节点集群需 2 节点确认）|
| Failover | 故障切换 | 主节点故障后自动切换到从节点 |
| Split-Brain | 脑裂 | 网络分区导致多个节点都认为自己是主节点 |
| Consensus | 共识算法 | 如 Raft、Paxos，用于分布式一致性的算法 |

---

## 监控与运维

| 术语（EN）| 术语（CN）| 定义 |
|-----------|-----------|------|
| Throughput | 吞吐量 | 单位时间内处理的消息数量（msg/s）|
| Latency | 延迟 | 消息从发送到消费的时间差 |
| Lag | 消费滞后 | 消费者处理速度跟不上生产速度，堆积的消息数量 |
| Backpressure | 背压 | 消费者处理能力不足时，向生产者反馈限速信号 |
