# 1.1 项目概述

## 项目背景

随着分布式系统和微服务架构的普及，系统间通信的可靠性和解耦性变得越来越重要。消息中间件作为分布式系统的核心基础设施，承担着异步通信、服务解耦、流量削峰等关键职责。

本项目旨在设计一个功能完整、性能优异的消息中间件数据库，为开发者提供从零构建消息中间件的知识图谱和工程实践指南。

## 核心定位

| 维度 | 说明 |
|------|------|
| **消息模式** | 支持点对点（Queue）和发布订阅（Topic）两种模式 |
| **协议兼容** | 兼容 AMQP 0-9-1、MQTT 5.0，支持协议网关扩展 |
| **存储方式** | 支持持久化消息和内存消息两种模式 |
| **集群能力** | 支持主从复制、分片集群，适配高可用场景 |
| **目标用户** | 需要理解消息中间件内部原理的开发者 |

## 主要特性

- ✅ 持久化消息存储，支持消息追溯
- ✅ 多协议接入（AMQP / MQTT / HTTP）
- ✅ 死信队列（DLX）异常消息处理
- ✅ 延迟消息 / 定时消息
- ✅ 消费确认（ACK）与消息重投
- ✅ 消费者分组与负载均衡
- ✅ 基于 ACL 的细粒度权限控制
- ✅ 完整的监控指标暴露（Prometheus 兼容）

## 名词对照

| 英文 | 中文 | 说明 |
|------|------|------|
| Broker | 代理节点 | 消息中间件的服务实例 |
| Producer | 生产者 | 发送消息的客户端 |
| Consumer | 消费者 | 接收并处理消息的客户端 |
| Queue | 队列 | 存储消息的容器（点对点模式）|
| Topic | 主题 | 发布订阅模式下的消息分类 |
| Exchange | 交换机 | 消息路由的枢纽 |
| Binding | 绑定 | 交换机与队列的关联关系 |
| Message | 消息 | 业务数据的载体 |
| Consumer Group | 消费者分组 | 多个消费者协同消费同一队列 |
| Offset | 消费偏移量 | 记录消费者消费到哪条消息 |
| DLX | 死信交换机 | 处理失败消息的交换机 |
| ACK | 确认 | 消费者确认消息处理成功 |
