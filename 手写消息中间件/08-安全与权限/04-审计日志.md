# 8.4 审计日志

## 概述

审计日志记录所有管理操作和安全相关事件，满足合规要求和事后追溯。

## 数据库设计

### 表结构：audit_log（审计日志表）

```sql
CREATE TABLE audit_log (
    log_id             BIGINT            NOT NULL AUTO_INCREMENT COMMENT '日志ID',
    
    -- 操作者
    user_id            VARCHAR(64)       DEFAULT NULL COMMENT '操作用户ID',
    username           VARCHAR(128)      DEFAULT NULL COMMENT '操作用户名',
    client_ip          VARCHAR(45)       DEFAULT NULL COMMENT '客户端IP',
    session_id         VARCHAR(64)       DEFAULT NULL COMMENT '会话ID',
    
    -- 操作
    operation_type     ENUM('create', 'update', 'delete', 'read', 'config_change', 
                           'auth_success', 'auth_failure', 'acl_check', 'admin_action') 
                      NOT NULL COMMENT '操作类型',
    resource_type      VARCHAR(64)       NOT NULL COMMENT '资源类型',
    resource_name      VARCHAR(255)      DEFAULT NULL COMMENT '资源名称',
    resource_id        VARCHAR(64)       DEFAULT NULL COMMENT '资源ID',
    
    -- 操作详情
    request_method     VARCHAR(16)       DEFAULT NULL COMMENT 'HTTP方法',
    request_path       VARCHAR(1024)     DEFAULT NULL COMMENT '请求路径',
    request_params     JSON              DEFAULT NULL COMMENT '请求参数(脱敏)',
    response_status    INT               DEFAULT NULL COMMENT '响应状态码',
    error_message      TEXT              DEFAULT NULL COMMENT '错误信息',
    
    -- 结果
    success            BOOLEAN           NOT NULL DEFAULT TRUE COMMENT '是否成功',
    
    -- 时间
    event_time         DATETIME(3)       NOT NULL COMMENT '事件时间',
    duration_ms        INT               DEFAULT NULL COMMENT '操作耗时(毫秒)',
    
    -- 归档
    archived           BOOLEAN           NOT NULL DEFAULT FALSE COMMENT '是否已归档',
    
    PRIMARY KEY (log_id),
    INDEX idx_user_time (user_id, event_time),
    INDEX idx_resource (resource_type, resource_id),
    INDEX idx_type (operation_type),
    INDEX idx_event_time (event_time),
    INDEX idx_ip (client_ip),
    INDEX idx_archived (archived, event_time)
) ENGINE = InnoDB DEFAULT CHARSET = utf8mb4 COLLATE = utf8mb4_unicode_ci
  COMMENT = '审计日志表';
```

---

## 审计事件分类

| 类别 | 操作类型 | 说明 |
|------|---------|------|
| 认证 | auth_success, auth_failure | 登录成功/失败 |
| 资源管理 | create, update, delete | 队列/交换机/用户管理 |
| 权限检查 | acl_check | 权限验证记录 |
| 管理操作 | admin_action | 强制关闭、集群变更等 |
| 配置变更 | config_change | 策略、参数修改 |

---

## 日志保留策略

| 日志类型 | 保留期 | 归档策略 |
|---------|--------|---------|
| 认证日志 | 180天 | 热存储 → 冷存储 |
| 资源操作日志 | 365天 | 热存储 → 冷存储 |
| 管理操作日志 | 730天 | 热存储 → 冷存储 → 归档 |
