# 5.2 消费模式

## 概述

消费模式定义了消息从队列到消费者的分发策略。不同的消费模式适用于不同的业务场景，直接影响消息的可靠性和吞吐量。

## 消费模式分类

### 1. Pull（拉取模式）

消费者主动向 Broker 请求消息。

```
Consumer ──▶ Basic.Get ──▶ Broker ──▶ 返回消息/空
Consumer ──▶ Basic.Get ──▶ Broker ──▶ 返回消息/空
Consumer ──▶ Basic.Get ──▶ Broker ──▶ 返回消息/空
```

**特点：**
- 消费者完全控制消费速率
- 适合批处理、定时消费
- 无消息时产生空轮询开销

### 2. Push（推送模式）

Broker 主动将消息推送给消费者。

```
Broker ──▶ Deliver(msg1) ──▶ Consumer
Broker ──▶ Deliver(msg2) ──▶ Consumer
Consumer ──▶ ACK(msg1) ──▶ Broker
Broker ──▶ Deliver(msg3) ──▶ Consumer
```

**特点：**
- 低延迟，消息到达即刻投递
- 通过 prefetch_count 控制推送速率
- 适合实时处理场景

### 3. Streaming（流式消费）

基于偏移量的持续消费，支持回溯和重播。

```
Consumer ──▶ Subscribe(offset=0) ──▶ Broker
Broker ──▶ msg1(offset=0) ──▶ Consumer
Broker ──▶ msg2(offset=1) ──▶ Consumer
Consumer ──▶ Commit(offset=1) ──▶ Broker
Broker ──▶ msg3(offset=2) ──▶ Consumer
```

**特点：**
- 支持从任意位置消费
- 消费进度持久化
- 适合事件溯源、日志分析

## 数据库设计

### 表结构：consume_mode_config（消费模式配置表）

```sql
CREATE TABLE consume_mode_config (
    config_id          VARCHAR(64)       NOT NULL COMMENT '配置ID',
    consumer_id        VARCHAR(64)       NOT NULL COMMENT '消费者ID',
    queue_id           VARCHAR(64)       NOT NULL COMMENT '队列ID',
    
    -- 消费模式
    consume_mode       ENUM('pull', 'push', 'streaming') NOT NULL COMMENT '消费模式',
    
    -- Pull 模式配置
    pull_batch_size    INT               DEFAULT 10 COMMENT '拉取批量大小',
    pull_max_wait_ms   INT               DEFAULT 5000 COMMENT '最大等待时间(毫秒)',
    pull_timeout_ms    INT               DEFAULT 30000 COMMENT '拉取超时(毫秒)',
    
    -- Push 模式配置
    push_prefetch_count INT              DEFAULT 10 COMMENT '预取数量',
    push_prefetch_size  INT              DEFAULT 0 COMMENT '预取大小(字节,0=无限制)',
    
    -- Streaming 模式配置
    stream_start_offset BIGINT           DEFAULT NULL COMMENT '起始偏移量',
    stream_commit_mode ENUM('auto', 'manual') DEFAULT 'auto' COMMENT '提交模式',
    stream_commit_interval_ms BIGINT     DEFAULT 5000 COMMENT '自动提交间隔',
    
    -- 确认模式
    ack_mode           ENUM('auto', 'manual', 'transactional') DEFAULT 'manual' COMMENT '确认模式',
    
    -- 审计
    created_at         DATETIME(3)       NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    updated_at         DATETIME(3)       NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3),
    
    PRIMARY KEY (config_id),
    UNIQUE KEY uk_consumer_queue (consumer_id, queue_id)
) ENGINE = InnoDB DEFAULT CHARSET = utf8mb4 COLLATE = utf8mb4_unicode_ci
  COMMENT = '消费模式配置表';
```

## 模式选择指南

| 场景 | 推荐模式 | 原因 |
|------|---------|------|
| 实时订单处理 | Push | 低延迟，消息即时投递 |
| 批量数据同步 | Pull | 消费者控制节奏，避免过载 |
| 日志/事件分析 | Streaming | 支持回溯和重消费 |
| 定时报表生成 | Pull + 定时触发 | 精确控制消费时间 |
| IoT 设备数据 | Push | 高吞吐，设备端轻量 |
