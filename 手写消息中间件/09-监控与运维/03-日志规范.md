# 9.3 日志规范

## 概述

统一日志规范是分布式追踪、问题排查和安全审计的基础。本规范覆盖日志级别、格式、存储和保留策略。

## 日志分类

| 日志类型 | 文件后缀 | 说明 | 级别 |
|---------|---------|------|------|
| 应用日志 | `.log` | 业务操作日志 | INFO/DEBUG |
| 错误日志 | `.err` | 错误和异常 | ERROR |
| GC 日志 | `gc.log` | 垃圾回收详情 | DEBUG |
| 慢查询日志 | `slow.log` | 慢操作追踪 | WARN |
| 安全日志 | `security.log` | 认证/鉴权/审计 | INFO |
| 审计日志 | `audit.log` | 合规性操作 | INFO |
| 访问日志 | `access.log` | HTTP/AMQP 请求 | INFO |
| 追踪日志 | `trace.log` | 分布式追踪 | DEBUG |

## 日志格式标准

### 结构化日志格式（JSON）

```json
{
  "timestamp": "2026-03-29T20:08:00.123+08:00",
  "level": "INFO",
  "service": "message-broker",
  "node": "broker-01",
  "thread": "pool-3-thread-15",
  "trace_id": "abc123-def456",
  "span_id": "xyz789",
  "logger": "com.broker.queue.QueueManager",
  "message": "Message enqueued successfully",
  "context": {
    "queue_id": "queue-uuid-001",
    "queue_name": "order.pay",
    "message_id": "msg-uuid-123",
    "vhost": "/",
    "latency_ms": 2
  }
}
```

### 传统日志格式（Plain Text）

```
%d{yyyy-MM-dd HH:mm:ss.SSS} %-5level [%thread/%X{trace_id}] %logger{36} - %msg%n

输出示例:
2026-03-29 20:08:00.123 INFO  [pool-3-thread-15/abc123-def456] c.b.q.QueueManager - Message enqueued successfully queue_id=queue-uuid-001 message_id=msg-uuid-123 latency_ms=2
```

## 日志级别规范

| 级别 | 使用场景 | 示例 |
|------|---------|------|
| ERROR | 系统故障，需要立即处理 | DB连接失败、OOM、节点宕机 |
| WARN | 异常但不阻塞 | 重试成功、限流触发、配额超限 |
| INFO | 正常业务流程 | 队列创建、连接建立、消息发送成功 |
| DEBUG | 开发调试 | 方法入口/出口、参数详情、循环次数 |
| TRACE | 最详细追踪 | 每行代码执行、字节级别细节 |

## 敏感信息过滤

### 需脱敏字段

| 字段 | 脱敏规则 | 示例 |
|------|---------|------|
| password | `***` | `password=***` |
| token | 前3后3，中间`***` | `token=abc***xyz` |
| credit_card | 仅保留后4位 | `card=****1234` |
| mobile | 中间4位`*` | `mobile=138****5678` |
| email | `@`前3后1 | `email=Use***@example.com` |
| Authorization | Bearer token | `Authorization=***` |

### 脱敏实现示例

```java
// Logback 脱敏配置
<configuration>
  <encoder class="ch.qos.logback.core.encoder.JsonEncoder">
    <providers>
      <patternProvider>
        <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} %-5level %msg</pattern>
        <masks>
          <mask field="password" replacement="***" />
          <mask field="token" replacement="***" pattern="(.{3}).*(.{3})" />
          <mask field="authorization" replacement="***" />
        </masks>
      </patternProvider>
    </providers>
  </encoder>
</configuration>
```

## 日志保留策略

| 日志类型 | 保留时间 | 存储位置 | 压缩 |
|---------|---------|---------|------|
| 应用日志 | 30天 | `/var/log/broker/` | 7天后 gzip |
| 错误日志 | 90天 | `/var/log/broker/` | 立即压缩 |
| GC 日志 | 7天 | `/var/log/broker/` | 不压缩 |
| 慢查询日志 | 90天 | `/var/log/broker/` | 30天后压缩 |
| 安全日志 | 180天 | `/var/log/broker/security/` | 30天后压缩 |
| 审计日志 | 365天 | `/var/log/broker/audit/` | 90天后压缩 |
| 访问日志 | 30天 | `/var/log/broker/` | 7天后 gzip |

## 日志存储与轮转

### Logrotate 配置

```bash
# /etc/logrotate.d/message-broker

/var/log/broker/*.log {
    daily              # 每日轮转
    rotate 30          # 保留30份
    compress           # 压缩旧日志
    delaycompress      # 延迟一天压缩
    notifempty         # 空文件不轮转
    missingok           # 缺失不报错
    create 0640 broker broker  # 新日志权限
    
    postrotate
        /bin/kill -HUP `cat /var/run/broker.pid 2>/dev/null` 2>/dev/null || true
    endscript
}

/var/log/broker/gc.log {
    daily
    rotate 7
    notifempty
    missingok
    compress
    copytruncate       # 复制后截断（避免重启）
}
```
