# 9.4 运维操作手册

## 概述

本手册提供消息中间件的日常运维操作指南，包括启动停止、扩缩容、故障处理等标准操作流程。

## 日常操作命令

### 服务管理

```bash
# 查看服务状态
openclaw gateway status

# 启动服务
openclaw gateway start

# 停止服务
openclaw gateway stop

# 重启服务
openclaw gateway restart

# 查看实时日志
openclaw logs --follow --tail 100

# 查看错误日志
openclaw logs --level error --since 1h
```

### 节点管理

```bash
# 查看集群节点列表
openclaw cluster nodes list

# 查看节点健康状态
openclaw cluster nodes status --node broker-01

# 从集群移除节点（优雅下线）
openclaw cluster nodes drain --node broker-03 --timeout 300s

# 添加节点到集群
openclaw cluster nodes join --node broker-04 --role follower

# 手动触发 Leader 转移
openclaw cluster leader transfer --to broker-02
```

### 队列管理

```bash
# 列出所有队列
openclaw queue list --vhost /

# 查看队列详情
openclaw queue stats --name order.pay --vhost /

# 清空队列消息（慎用）
openclaw queue purge --name order.pay --vhost /

# 删除队列
openclaw queue delete --name order.pay --vhost / --if-empty

# 迁移队列到其他节点
openclaw queue migrate --name order.pay --to-node broker-02
```

### 连接与会话管理

```bash
# 列出活跃连接
openclaw connection list

# 强制关闭连接
openclaw connection close --id conn-uuid-xxx --reason "maintenance"

# 列出活跃通道
openclaw channel list --connection-id conn-uuid-xxx

# 查看用户会话
openclaw session list --user admin
```

## 标准运维流程

### 流程 1：优雅重启

```
目的：Rolling Update / 配置变更后重启

步骤：
1. 确认节点健康状态
   $ openclaw cluster nodes status
   
2. 标记节点为维护模式
   $ openclaw cluster nodes maintain --node broker-01
   
3. 等待消费完成（最多30分钟）
   $ openclaw cluster nodes drain --node broker-01
   
4. 执行重启
   $ openclaw gateway restart
   
5. 确认服务恢复
   $ openclaw gateway status
   $ openclaw cluster nodes status --node broker-01
   
6. 取消维护模式
   $ openclaw cluster nodes online --node broker-01
```

### 流程 2：节点扩缩容

```
目的：水平扩缩容

扩容步骤：
1. 准备新节点（安装、配置网络）
2. 将节点加入集群
   $ openclaw cluster nodes join --node broker-04
   
3. 等待数据同步
   $ openclaw cluster rebalance status
   
4. 验证节点状态
   $ openclaw cluster nodes status

缩容步骤：
1. 标记节点为待下线
   $ openclaw cluster nodes drain --node broker-04 --timeout 600s
   
2. 等待数据迁移完成
   $ openclaw cluster rebalance status
   
3. 确认队列均衡
   $ openclaw queue list --sort-by node
   
4. 从集群移除
   $ openclaw cluster nodes remove --node broker-04
```

### 流程 3：故障切换

```
触发条件：Leader 节点不可用

自动切换（默认）：
1. Follower 检测到 Leader 超时
2. Raft 发起新一轮选举
3. 新的 Leader 选出（通常 < 30s）
4. 客户端自动重连

手动切换：
$ openclaw cluster leader transfer --to broker-02
```

### 流程 4：数据迁移

```
目的：将队列从节点A迁移到节点B

1. 创建目标队列
   $ openclaw queue create --name order.pay --vhost / --node broker-02

2. 暂停生产者投递
   $ openclaw exchange pause-publishing --name order.exchange

3. 等待消费完成
   $ watch openclaw queue stats --name order.pay --field message_count

4. 原子切换 Binding
   $ openclaw binding update --exchange order.exchange --queue order.pay --target-node broker-02

5. 恢复生产
   $ openclaw exchange resume-publishing --name order.exchange

6. 清理源队列
   $ openclaw queue delete --name order.pay.bak --node broker-01
```

## 常见故障处理

### 故障 1：节点无法启动

```
症状：gateway start 失败

排查步骤：
1. 检查端口占用
   $ netstat -tlnp | grep 5672
   
2. 检查数据目录权限
   $ ls -la /data/broker/
   
3. 检查磁盘空间
   $ df -h /data/
   
4. 检查内存
   $ free -h
   
5. 查看启动日志
   $ openclaw logs --level error --since 10m
```

### 故障 2：消息堆积

```
症状：队列消息数持续增长

排查步骤：
1. 查看消费者状态
   $ openclaw queue stats --name order.pay
   
2. 查看消费速率
   $ openclaw metrics get --queue order.pay --metric consumer_rate
   
3. 检查消费者连接
   $ openclaw connection list --queue order.pay
   
4. 常见原因：
   - 消费者崩溃 → 重启消费者
   - 消费慢 → 增加消费者数量 / 优化处理逻辑
   - 限流触发 → 调整限流阈值

处理：
$ openclaw queue throttle --name order.pay --reset
```

### 故障 3：存储空间不足

```
排查：
$ df -h /data/broker/messages/

处理方案（按紧急程度）：
1. 紧急：清理过期消息
   $ openclaw queue purge --name order.pay --older-than 7d
   
2. 扩容存储
   $ openclaw storage expand --device /dev/sdb
   
3. 调整消息保留策略
   $ openclaw queue update --name order.pay --set "x-max-length=50000"
   
4. 启用消息压缩
   $ openclaw cluster config set --key compression.enabled --value true
```

### 故障 4：集群脑裂

```
症状：多个节点同时是 Leader

紧急处理：
1. 确认分区状态
   $ openclaw cluster partition status
   
2. 手动触发恢复（优先保护数据）
   $ openclaw cluster partition resolve --action auto
   
3. 选择保留节点（数据最新优先）
   $ openclaw cluster partition resolve --keep-node broker-01 --shutdown-nodes broker-02,broker-03
   
4. 检查数据一致性
   $ openclaw cluster check integrity
   
5. 重启被关闭节点
   $ openclaw cluster nodes rejoin --node broker-02
```

## 巡检清单

### 每日巡检

```
[ ] 所有节点状态正常 (openclaw cluster nodes status)
[ ] 无 ERROR 级别日志 (openclaw logs --level error --since 24h)
[ ] 队列消息数无异常堆积
[ ] 磁盘空间使用率 < 80%
[ ] 内存使用率 < 85%
[ ] 集群 Leader 正常
[ ] 复制延迟 < 1s
[ ] 备份任务成功执行
```

### 每周巡检

```
[ ] 审查安全日志（失败登录尝试）
[ ] 检查慢查询日志 (> 100ms)
[ ] 验证备份可恢复性
[ ] 审查配额使用情况
[ ] 更新证书有效期
[ ] 性能基线对比
```
