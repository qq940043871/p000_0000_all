# 10.2 索引设计

## 概述

高效的索引设计是快速消息查询的关键。索引需要在查询性能和写入性能之间取得平衡。

## 索引类型

### 1. 消息ID索引

```sql
-- 快速通过消息ID定位消息
CREATE INDEX idx_message_id ON message_storage(message_id);
```

### 2. 序列号索引

```sql
-- 按序列号范围查询(消费者拉取)
CREATE INDEX idx_sequence ON message_storage(queue_id, partition_id, sequence_num);
```

### 3. 时间索引

```sql
-- 按时间范围查询(消息回溯)
CREATE INDEX idx_timestamp ON message_storage(queue_id, message_timestamp);
```

### 4. 路由键索引

```sql
-- 按路由键模糊匹配
CREATE INDEX idx_routing_key ON message_storage(routing_key);
```

### 5. 复合索引

```sql
-- 常用查询组合优化
CREATE INDEX idx_queue_status_time ON message(queue_id, message_status, message_timestamp);
```

## 数据库设计

### 表结构：index_config（索引配置表）

```sql
CREATE TABLE index_config (
    index_id           VARCHAR(64)       NOT NULL COMMENT '索引ID',
    table_name         VARCHAR(128)      NOT NULL COMMENT '表名',
    index_name         VARCHAR(128)      NOT NULL COMMENT '索引名称',
    
    -- 索引定义
    index_type         ENUM('btree', 'hash', 'bitmap', 'fulltext', 'spatial') NOT NULL COMMENT '索引类型',
    columns            JSON             NOT NULL COMMENT '索引列定义',
    unique             BOOLEAN           NOT NULL DEFAULT FALSE COMMENT '是否唯一索引',
    
    -- 状态
    status             ENUM('building', 'active', 'disabled', 'rebuilding') DEFAULT 'active',
    
    -- 性能
    size_bytes         BIGINT            DEFAULT NULL COMMENT '索引大小(字节)',
    row_count          BIGINT            DEFAULT NULL COMMENT '索引行数',
    
    -- 审计
    created_at         DATETIME(3)       NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    last_rebuilt_at     DATETIME(3)      DEFAULT NULL COMMENT '上次重建时间',
    
    PRIMARY KEY (index_id),
    UNIQUE KEY uk_table_name (table_name, index_name)
) ENGINE = InnoDB DEFAULT CHARSET = utf8mb4 COLLATE = utf8mb4_unicode_ci
  COMMENT = '索引配置表';
```

## 索引优化原则

| 原则 | 说明 |
|------|------|
| 高基数列优先 | 选择唯一值多的列做索引 |
| 覆盖索引 | 索引包含查询所需全部列 |
| 最左前缀 | 组合索引注意列顺序 |
| 避免过度索引 | 每个索引增加写入开销 |
| 定期重建 | 索引碎片化影响性能 |
