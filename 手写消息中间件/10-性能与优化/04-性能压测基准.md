# 10.4 性能压测基准

## 概述

性能压测是评估消息中间件能力、发现瓶颈和验证优化的核心手段。本章节提供完整的压测方法论、工具配置和基准数据。

## 压测方法论

### 压测类型

| 类型 | 目标 | 持续时间 | 指标 |
|------|------|---------|------|
| 基准测试 | 获取单节点最大性能 | 10-30min | TPS/Latency |
| 容量测试 | 确定系统容量上限 | 1-4h | 最大TPS/最优并发 |
| 稳定性测试 | 长时间运行稳定性 | 24-72h | 错误率/资源趋势 |
| 峰值测试 | 模拟突发流量 | 30-60min | 峰值处理能力 |
| 破坏性测试 | 极端条件表现 | 1-2h | 容错/恢复能力 |

### 自定义 Go 压测客户端

```go
package main

import (
    "sync/atomic"
    "time"
    "fmt"
)

type BenchmarkResult struct {
    Published  int64
    Confirmed  int64
    Consumed   int64
    Errors     int64
    MinLatency int64
    MaxLatency int64
    AvgLatency int64
    P50Latency int64
    P95Latency int64
    P99Latency int64
}

func RunBenchmark(cfg BenchmarkConfig) *BenchmarkResult {
    // 并发生产者
    for i := 0; i < cfg.ProducerCount; i++ {
        go producer(i, cfg)
    }
    
    // 并发消费者
    for i := 0; i < cfg.ConsumerCount; i++ {
        go consumer(i, cfg)
    }
    
    // 采集指标
    ticker := time.NewTicker(1 * time.Second)
    for range ticker.C {
        printStats(result)
    }
    
    return result
}
```

## 基准测试场景

### 场景 1：纯发送吞吐（Baseline）

```
目标：测量最大发送速率
配置：
  - 生产者：10-100 个
  - 消息大小：1KB / 10KB / 100KB
  - 持久化：是/否
  - 确认模式：无确认 / Publisher Confirms
  - 交换机：Direct

指标：
  - 消息发送速率 (msg/s)
  - 字节吞吐 (MB/s)
  - CPU 使用率
  - 网络带宽
```

### 场景 2：纯消费吞吐

```
目标：测量最大消费速率
配置：
  - 消费者：10-100 个
  - 预取数：1 / 10 / 100 / 1000
  - ACK 模式：自动 / 手动单条 / 手动批量
  - 消息大小：1KB

指标：
  - 消息消费速率 (msg/s)
  - 端到端延迟
  - 消费者 lag
  - 处理时间分布
```

### 场景 3：端到端延迟

```
目标：测量消息从发送到消费的平均延迟
配置：
  - 生产者：1 个
  - 消费者：1 个
  - 消息大小：1KB
  - 队列深度：1 / 100 / 1000 / 10000

测量方法：
  latency = consumer_receive_time - producer_send_time
  
指标：
  - 平均延迟 (Avg)
  - 中位延迟 (P50)
  - 95分位延迟 (P95)
  - 99分位延迟 (P99)
  - 最大延迟 (Max)
```

### 场景 4：混合读写

```
目标：模拟真实业务负载
配置：
  - 生产者:消费者 = 1:1 到 5:1
  - 突发流量：周期性的发送峰值
  - 消息大小：随机 0.5KB-2KB

典型比例：
  - 轻量负载：10% 发送，90% 消费
  - 均衡负载：50% 发送，50% 消费
  - 重度负载：80% 发送，20% 消费
```

## 压测报告模板

```markdown
# 性能压测报告

## 测试信息
- 测试时间：2026-03-29 20:00-22:00
- 测试版本：v1.0.0
- 测试环境：4核8G / 3节点集群

## 测试配置
- 消息大小：1KB
- 持久化：是
- 确认模式：Publisher Confirms

## 结果汇总
| 指标 | 10并发 | 50并发 | 100并发 |
|------|--------|--------|--------|
| 发送 TPS | 15,000 | 75,000 | 120,000 |
| 消费 TPS | 20,000 | 95,000 | 150,000 |
| Avg 延迟 | 0.5ms | 1.2ms | 2.8ms |
| P99 延迟 | 2ms | 8ms | 25ms |

## 瓶颈分析
1. CPU 成为主要瓶颈（>85%）
2. 网络带宽 1Gbps 接近上限
3. GC 暂停影响 P99

## 优化建议
1. 增加 CPU 核心
2. 升级网络到 10Gbps
3. 切换 ZGC 减少暂停
```
