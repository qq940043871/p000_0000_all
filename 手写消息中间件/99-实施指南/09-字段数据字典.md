# 字段数据字典

> 本文档是所有数据库表字段的完整字典，按表分组，便于开发查阅。

---

## 一、公共字段约定

所有表均包含以下审计字段（如适用）：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` / `{entity}_id` | VARCHAR(64) | 主键，使用 UUID v4 |
| `created_at` | DATETIME(3) | 创建时间，精确到毫秒 |
| `updated_at` | DATETIME(3) | 更新时间，精确到毫秒 |
| `deleted_at` | DATETIME(3) | 软删除时间（支持回收）|
| `{entity}_name` | VARCHAR(255) | 实体名称 |

## 二、queue_info（队列元数据表）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| queue_id | VARCHAR(64) | ✅ | 队列唯一标识，UUID |
| queue_name | VARCHAR(255) | ✅ | 队列名称，支持中英文 |
| vhost | VARCHAR(128) | ✅ | 所属虚拟主机，默认为 `/` |
| exchange_name | VARCHAR(255) | ✅ | 绑定的交换机名称 |
| binding_key | VARCHAR(255) | ✅ | 绑定键，支持通配符 |
| queue_type | ENUM | ✅ | 队列类型：classic / quorum / stream |
| durable | BOOLEAN | ✅ | 是否持久化，重启后保留 |
| auto_delete | BOOLEAN | ✅ | 无消费者时是否自动删除 |
| exclusive | BOOLEAN | ✅ | 是否独占（仅一个连接可用）|
| max_length | BIGINT | | 队列最大消息数，NULL=无限制 |
| max_bytes | BIGINT | | 队列最大字节数，NULL=无限制 |
| overflow_policy | ENUM | | 溢出策略：reject-publish / drop-head / flow-control |
| dead_letter_exchange | VARCHAR(255) | | 死信交换机名称 |
| dead_letter_routing_key | VARCHAR(255) | | 死信路由键 |
| message_ttl | BIGINT | | 队列默认 TTL（毫秒）|
| status | ENUM | ✅ | 状态：running / suspending / deleted |
| message_count | BIGINT | ✅ | 当前消息数量（实时更新）|
| consumer_count | INT | ✅ | 当前消费者数量 |
