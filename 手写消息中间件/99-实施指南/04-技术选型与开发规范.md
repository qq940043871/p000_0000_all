# 技术选型与开发规范

## 一、技术栈建议

### 1. 核心语言与框架

| 层级 | 推荐技术 | 备选 | 说明 |
|------|---------|------|------|
| **网关/协议层** | Netty | Reactor-Netty | 高性能异步网络框架 |
| **消息路由** | Java / Go | Rust | Java 生态成熟，Go 性能好 |
| **持久化存储** | RocksDB | PostgreSQL | RocksDB 适合追加写场景 |
| **元数据库** | PostgreSQL | MySQL 8.0 | 事务支持好 |
| **缓存** | Redis Cluster | 内存 MapDB | Offset 缓存热点数据 |
| **消息队列** | 自研（基于本文档）| RocketMQ | 深度定制场景 |
| **配置中心** | Apollo | etcd | 多环境配置管理 |
| **服务注册** | etcd | Consul | 集群节点发现 |

## 二、代码命名规范

### 1. 数据库对象命名

| 对象 | 命名规范 | 示例 |
|------|---------|------|
| 数据库 | 小写下划线 | `message_broker` |
| 表 | 小写下划线，单数名词 | `queue_info` |
| 主键 | `{table_singular}_id` | `queue_id` |
| 外键 | `{ref_table}_id` | `consumer_id` |
| 索引 | `idx_{table}_{column}` | `idx_queue_status` |
| 唯一键 | `uk_{table}_{column}` | `uk_queue_name` |

## 三、Git 提交规范

```
feat: 新功能
fix: Bug 修复
docs: 文档变更
style: 代码格式（不影响功能）
refactor: 重构
perf: 性能优化
test: 测试相关
chore: 构建/工具变更

示例:
feat(queue): 添加队列优先级支持
fix(dlq): 修复死信队列统计错误
perf(storage): 优化 Commit Log 写入性能
docs(api): 更新 REST API 文档
```

## 四、接口版本管理

所有对外 API 必须包含版本号：

```
/api/v1/{resource}
/api/v2/{resource}  (Breaking Changes 时升级)
```

版本升级规则：
- 添加可选参数 → 向后兼容，不升级版本
- 添加必填参数 → 向后不兼容，升级版本
- 删除参数 → 向后不兼容，升级版本
- 修改参数类型 → 向后不兼容，升级版本

## 五、错误码规范

| 前缀 | 范围 | 说明 |
|------|------|------|
| `QUEUE_` | 1000-1999 | 队列相关错误 |
| `EXCHANGE_` | 2000-2999 | 交换机相关错误 |
| `MESSAGE_` | 3000-3999 | 消息相关错误 |
| `CONSUMER_` | 4000-4999 | 消费者相关错误 |
| `PRODUCER_` | 5000-5999 | 生产者相关错误 |
| `AUTH_` | 6000-6999 | 认证授权错误 |
| `CLUSTER_` | 7000-7999 | 集群相关错误 |
| `STORAGE_` | 8000-8999 | 存储相关错误 |
| `SYSTEM_` | 9000-9999 | 系统级错误 |

错误响应示例：
```json
{
  "error": {
    "code": "QUEUE_1001",
    "message": "队列已存在",
    "detail": "queue_name: order.queue 已存在于 vhost: /",
    "trace_id": "abc123"
  }
}
```

## 六、配置规范

### 配置文件格式 (config.yaml)

```yaml
server:
  host: "0.0.0.0"
  port: 5672
  management_port: 15672
  
storage:
  engine: "rocksdb"
  data_dir: "/data/broker"
  max_segment_size_mb: 512
  
cluster:
  enabled: true
  node_name: "broker-01"
  cluster_id: "prod-cluster"
  discovery:
    type: "static"
    seed_nodes:
      - "broker-01:5672"
      - "broker-02:5672"
      
security:
  tls:
    enabled: false
    cert_file: "/etc/broker/tls.crt"
    key_file: "/etc/broker/tls.key"
  auth:
    enabled: true
    type: "builtin"
    
monitoring:
  metrics:
    enabled: true
    port: 9090
  logging:
    level: "info"
    format: "json"
```
