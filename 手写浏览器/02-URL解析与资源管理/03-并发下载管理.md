## 一、并发控制

```
浏览器并发限制：
  每个域名：6个并发连接（HTTP/1.1）
  代理连接池：每个代理服务器N个连接
  
HTTP/2改进：
  多路复用：单连接并行请求
  流控制：防止发送方压垮接收方
  
资源优先级：
  1. HTML文档 - 最高
  2. CSS/字体 - 高
  3. JS（同步）- 中高
  4. JS（异步/defer）- 中
  5. 图片/视频 - 低
```

## 二、下载调度器

```cpp
class ResourceScheduler {
private:
    std::map<std::string, std::queue<ResourceRequest>> pending;
    std::map<std::string, std::set<ResourceRequest*>> active;
    std::mutex mutex;
    const size_t maxConnectionsPerHost = 6;
    
public:
    void schedule(ResourceRequest* request) {
        std::lock_guard<std::mutex> lock(mutex);
        
        std::string host = request->url().host();
        pending[host].push(*request);
        
        tryStartDownload(host);
    }
    
    void onResourceComplete(const std::string& host, ResourceRequest* req) {
        std::lock_guard<std::mutex> lock(mutex);
        
        active[host].erase(req);
        
        // 继续下载下一个
        tryStartDownload(host);
    }
    
private:
    void tryStartDownload(const std::string& host) {
        if (active[host].size() >= maxConnectionsPerHost) {
            return;
        }
        
        if (pending[host].empty()) {
            return;
        }
        
        ResourceRequest req = pending[host].front();
        pending[host].pop();
        
        active[host].insert(&req);
        req.start();
    }
};
```
