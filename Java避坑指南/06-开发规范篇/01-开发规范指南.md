## 一、命名规范

### 1. 包命名
```
com.公司名.业务模块.子模块

示例：
com.alibaba.user.service
com.alibaba.order.controller
com.alibaba.product.dao
```

### 2. 类命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 普通类 | UpperCamelCase | UserService |
| 工具类 | UpperCamelCase + Util | StringUtil |
| 异常类 | UpperCamelCase + Exception | BusinessException |
| 枚举类 | UpperCamelCase | OrderStatus |
| 实体类 | UpperCamelCase | User |
| VO/DTO | UpperCamelCase + 后缀 | UserVO |
| DAO接口 | I开头 | IUserDao |
| Service接口 | I开头 | IUserService |
| Service实现 | 普通类名 + Impl | UserServiceImpl |

### 3. 方法命名

| 操作 | 规范 | 示例 |
|------|------|------|
| 查询 | find/select/get | findById |
| 新增 | save/insert/add | saveUser |
| 修改 | update/modify | updateUser |
| 删除 | delete/remove | deleteById |
| 统计 | count | countAll |

### 4. 变量命名
```java
// 推荐
private UserService userService;
private int pageSize;
private List<User> userList;
private Map<String, User> userMap;
private Set<String> userIdSet;

// 不推荐
private UserService u;
private List<User> list;
```

## 二、代码风格

### 1. 大括号
```java
// 推荐：左大括号不换行
public void method() {
    if (condition) {
        doSomething();
    }
}

// 不推荐
public void method()
{
    if (condition)
    {
        doSomething();
    }
}
```

### 2. 空格
```java
// 运算符两侧加空格
int a = b + c;      // 推荐
int a=b+c;           // 不推荐

// 关键字和括号之间加空格
if (condition) {     // 推荐
if(condition){        // 不推荐
```

### 3. 行长度
```java
// 每行不超过120个字符
// 过长时换行
String sql = "SELECT id, name, age, email, phone, address " +
            "FROM user WHERE status = 1";
```

## 四、方法规范

### 1. 方法长度
```java
// 推荐：方法不超过50行
// 如果过长，拆分

// 不推荐
public void process() {
    // 100行代码...
}

// 推荐
public void process() {
    step1();
    step2();
    step3();
}
```

### 2. 参数数量
```java
// 推荐：参数不超过5个
// 如果过多，使用DTO封装

// 不推荐
public void createUser(String name, int age, String email, 
    String phone, String address, String city) {
}

// 推荐
public void createUser(UserDTO userDTO) {
}
```

## 五、异常处理规范

### 1. 异常分类
```java
// 业务异常：业务逻辑错误
public class BusinessException extends RuntimeException {
}

// 系统异常：程序错误
public class SystemException extends RuntimeException {
}

// 参数异常：参数校验失败
public class ParamException extends RuntimeException {
}
```

### 2. 统一异常处理
```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    
    @ExceptionHandler(BusinessException.class)
    public Result<Void> handleBusinessException(BusinessException e) {
        log.warn("业务异常", e);
        return Result.error(e.getCode(), e.getMessage());
    }
    
    @ExceptionHandler(Exception.class)
    public Result<Void> handleException(Exception e) {
        log.error("系统异常", e);
        return Result.error(500, "系统异常");
    }
}
```

## 六、日志规范

### 1. 日志级别
```java
// DEBUG：开发调试
log.debug("查询参数: {}", params);

// INFO：正常流程
log.info("用户登录: userId={}", userId);

// WARN：警告
log.warn("库存不足: productId={}, stock={}", productId, stock);

// ERROR：错误
log.error("数据库异常", e);
```

### 2. 日志内容
```java
// 推荐：使用占位符
log.info("用户 {} 购买了商品 {}", userId, productId);

// 不推荐：字符串拼接
log.info("用户" + userId + "购买了商品" + productId);
```

### 3. 敏感信息
```java
// 不推荐：打印敏感信息
log.info("密码: {}", password);
log.info("手机号: {}", phone);

// 推荐：脱敏打印
log.info("手机号: {}", DesensitizeUtil.phone(phone));
```

## 七、接口规范

### 1. RESTful风格
```java
// GET - 查询
@GetMapping("/users/{id}")
public User getUser(@PathVariable Long id) {}

// POST - 新增
@PostMapping("/users")
public Long createUser(@RequestBody @Valid User user) {}

// PUT - 更新
@PutMapping("/users/{id}")
public void updateUser(@PathVariable Long id, @RequestBody User user) {}

// DELETE - 删除
@DeleteMapping("/users/{id}")
public void deleteUser(@PathVariable Long id) {}
```

### 2. 统一响应格式
```java
@Data
public class Result<T> {
    private int code;
    private String message;
    private T data;
    
    public static <T> Result<T> success(T data) {
        Result<T> result = new Result<>();
        result.setCode(200);
        result.setMessage("success");
        result.setData(data);
        return result;
    }
}
```

## 八、数据库规范

### 1. 表命名
```sql
-- 表名使用小写，单词间用下划线
-- 模块名前缀
-- t_user（用户表）
-- t_order（订单表）
-- t_order_item（订单明细表）
```

### 2. 字段命名
```sql
-- 所有字段使用小写
-- 创建时间：create_time
-- 更新时间：update_time
-- 逻辑删除：deleted
-- 乐观锁：version
```

### 3. SQL编写
```sql
-- 使用明确的字段列表
SELECT id, name, age FROM user WHERE status = 1;

-- 使用LIMIT限制返回条数
SELECT * FROM user LIMIT 100;

-- 避免SELECT *
-- 添加适当的索引
```

## 十、安全规范

### 1. 认证授权
```java
// 使用Spring Security
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/public/**").permitAll()
                .anyRequest().authenticated()
            );
        return http.build();
    }
}
```

### 2. 敏感数据
```java
// 密码加密存储
@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();
}

// 敏感数据脱敏
public String maskPhone(String phone) {
    if (phone == null) return null;
    return phone.substring(0, 3) + "****" + phone.substring(7);
}
```
