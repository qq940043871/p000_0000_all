## 一、内存查看命令

### free

```bash
# 基础用法
free                       # 默认KB
free -h                    # 人性化显示(G/M/K)
free -m                    # MB为单位
free -g                    # GB为单位
free -s 1                  # 每秒刷新
free -c 10                 # 刷新10次

# 输出解释
#               total        used        free      shared  buff/cache   available
# Mem:      32768000    15000000     5000000     100000    12768000    15000000
# Swap:      8000000           0     8000000

# Mem行: 物理内存
# Swap行: 交换空间

# 重要字段:
# total:   总物理内存
# used:    已使用
# free:    完全空闲
# available: 实际可用(含缓存)
```

### vmstat

```bash
# 基础用法
vmstat                        # 一次性输出
vmstat 1                      # 每秒刷新
vmstat 1 10                   # 每秒刷新,共10次
vmstat -s                     # 显示详细统计
vmstat -d                     # 显示磁盘统计

# 输出解释
# procs -----------memory---------- -swap- -----io---- -system- --------cpu--------
#  r  b   swpd   free   buff  cache   si   so    bi    bo   in   cs  us  sy  id  wa  st
#  2  0      0 5000000 100000 12768000    0    0     0     0    0    0   5   2  93   0   0

# r: 运行中进程数
# b: 阻塞进程数
# swpd: 使用Swap大小
# free: 空闲内存
# buff: 缓冲区
# cache: 缓存
# si/so: Swap进/出
# bi/bo: 磁盘块进/出
# in: 中断数
# cs: 上下文切换
# us/sy/id/wa/st: CPU各状态占比
```

## 二、进程内存分析

### ps查看内存

```bash
# 按内存排序
ps aux --sort=-%mem          # 从高到低
ps aux --sort=%mem           # 从低到高
ps aux | sort -k4 -nr        # 同上

# 只显示前10个
ps aux --sort=-%mem | head -11

# 显示内存字段
ps -eo pid,vsz,rss,pmem,pcpu,comm --sort=-rss

# 字段说明
# VSZ: 虚拟内存大小(KB)
# RSS: 实际物理内存大小(KB)
# %MEM: 内存占比
```

### pmap查看内存映射

```bash
# 基础用法
pmap PID                      # 查看进程内存映射
pmap -x PID                   # 显示详细格式
pmap -X PID                   # 更详细
pmap -d PID                   # 显示设备映射

# 输出解释
# 00007f8c90000000   52384K rw-pa-- ./java
# 地址          大小   权限  映射文件

# 权限: r(read) w(write) x(execute) p(private) s(shared)

# 统计内存
pmap PID | tail -1

# 查看具体段
pmap PID | grep heap
pmap PID | grep stack
```

### /proc/PID/status

```bash
cat /proc/PID/status

# 关键字段
# Name:        进程名
# State:       进程状态
# Pid:         进程ID
# PPid:        父进程ID
# Threads:     线程数
# VmPeak:      虚拟内存峰值
# VmSize:      虚拟内存大小
# VmLck:       锁住的内存
# VmPin:       Pin内存
# VmRSS:       物理内存(RSS)
# RssAnon:     匿名页
# RssFile:     文件映射页
# RssShmem:    共享内存
# VmData:      数据段大小
# VmStk:       栈大小
# VmExe:       代码段大小
# VmLib:       共享库大小
# VmPTE:       页表大小
# VmSwap:      Swap大小
```

## 三、smaps详解

```bash
# 查看详细内存分布
cat /proc/PID/smaps

# 查看特定段的内存
cat /proc/PID/smaps | grep -A 20 "\[heap\]"
cat /proc/PID/smaps | grep -A 20 "\[stack\]"

# 统计各状态内存总和
cat /proc/PID/smaps | awk '/Size|Rss|Pss|Shared_Clean|Shared_Dirty|Private_Clean|Private_Dirty/{split($1,a,":");sum[a[1]]+=$2}END{for(k in sum)print k,sum[k]}'

# 查看Java堆内存
cat /proc/PID/smaps | grep -A 10 "\[heap\]" | head -20
```

## 四、内存分析工具

### strace跟踪内存分配

```bash
# 跟踪系统调用
strace -e trace=brk,mmap,munmap -p PID

# 跟踪内存相关调用
strace -e memory -p PID
```

### valgrind检测内存泄漏

```bash
# 安装
apt install valgrind

# 检测内存泄漏
valgrind --leak-check=full --show-leak-kinds=all --track-origins=yes ./program

# 输出说明
# definitely lost: 确定泄漏
# indirectly lost: 间接泄漏
# possibly lost: 可能泄漏
# still reachable: 未释放但仍可达
```

### gdb分析内存

```bash
# 附加到进程
gdb -p PID

# 查看内存
(gdb) x/100x 0x12345678      # 十六进制显示
(gdb) x/s 0x12345678         # 字符串显示
(gdb) x/i 0x12345678         # 指令显示

# 设置观察点
(gdb) watch *(int*)0x12345678
(gdb) info watchpoints
```

## 五、内存问题排查

### 内存泄漏排查

```bash
# 1. 观察内存增长趋势
while true; do
    ps -o pid,vsz,rss,comm -p PID
    sleep 5
done

# 2. 查看进程的内存分布
pmap -x PID | tail -1

# 3. 多次dump smaps对比
cat /proc/PID/smaps > /tmp/smaps_1.txt
sleep 60
cat /proc/PID/smaps > /tmp/smaps_2.txt
diff /tmp/smaps_1.txt /tmp/smaps_2.txt

# 4. 使用gdb附加观察
gdb -p PID
(gdb) set pagination off
(gdb) monitor info mem
```

### 内存不足排查

```bash
# 1. 检查系统内存
free -h
cat /proc/meminfo

# 2. 找出内存消耗大户
ps aux --sort=-%mem | head -10

# 3. 检查缓存
free && sync && echo 3 > /proc/sys/vm/drop_caches && free

# 4. 检查OOM日志
dmesg | grep -i oom
journalctl -k | grep -i oom

# 5. 检查Swap使用
swapon -s
vmstat 1
```

## 六、内存优化

### 系统级优化

```bash
# 调整水位线
sysctl -w vm.min_free_kbytes=65536

# 调整缓存策略
sysctl -w vm.vfs_cache_pressure=200   # 优先回收缓存
sysctl -w vm.swappiness=10            # 减少Swap使用

# 开启内存大页
sysctl -w vm.nr_hugepages=256

# 持久化配置
cat >> /etc/sysctl.conf << EOF
vm.min_free_kbytes=65536
vm.vfs_cache_pressure=200
vm.swappiness=10
EOF
sysctl -p
```
