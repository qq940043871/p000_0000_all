## 一、进程控制命令

### kill

```bash
# ====== 发送信号 ======
kill PID                    # 发送SIGTERM（优雅终止）
kill -9 PID                 # 发送SIGKILL（强制杀死）
kill -15 PID                # 显式发送SIGTERM
kill -l                     # 显示所有信号

# 常用信号
# 1) SIGHUP   - 挂起，重读配置
# 9) SIGKILL  - 强制杀死
# 15) SIGTERM - 优雅终止（默认）
# 18) SIGCONT - 继续运行
# 19) SIGSTOP - 暂停
# 20) SIGTSTP - 终端暂停

# ====== killall ======
killall java                 # 按名字杀死所有java进程
killall -9 nginx             # 强制杀死
killall -u username nginx    # 杀死用户的所有nginx进程

# ====== pkill ======
pkill -9 java                # 按名字杀死
pkill -u username            # 杀死用户进程
pkill -f "java -jar"         # 按完整命令匹配
```

### 进程前后台切换

```bash
# 后台运行
./script.sh &                # 后台运行
nohup ./script.sh &          # 不挂断后台运行

# 前台运行
fg                            # 切回前台
fg %1                         # 切回指定后台任务

# 后台任务
jobs                          # 列出后台任务
bg %1                         # 后台继续运行
Ctrl+Z                        # 挂起当前任务
```

## 二、进程优先级

### nice/renice

```bash
# nice: 启动时设置优先级
# 范围: -20(最高) ~ 19(最低)
# 默认: 0

nice -n 10 ./script.sh       # 以低优先级启动
nice -n -10 ./script.sh      # 以高优先级启动（需要root）

# renice: 修改运行中进程优先级
renice 10 PID                 # 修改进程优先级
renice -n 5 -u username       # 修改用户所有进程优先级
renice 5 -p PID1 PID2         # 修改多个进程

# 查看优先级
ps -eo pid,ni,cmd            # 显示NI列
```

### nohup/screen

```bash
# nohup: 不挂断运行
nohup ./script.sh > output.log 2>&1 &
# > output.log: 输出重定向
# 2>&1: 错误输出重定向到标准输出
# &: 后台运行

# screen: 终端复用
screen -S mysession           # 创建命名会话
screen -ls                    # 列出所有会话
screen -r mysession           # 恢复会话
screen -X -S mysession quit   # 删除会话

# 在screen中
# Ctrl+a d: 分离会话
# Ctrl+a c: 创建新窗口
# Ctrl+a n/p: 下一个/上一个窗口
# Ctrl+a [: 进入复制模式
```

## 三、守护进程管理

### systemd

```bash
# ====== 服务管理 ======
systemctl start nginx         # 启动服务
systemctl stop nginx          # 停止服务
systemctl restart nginx       # 重启服务
systemctl reload nginx        # 重载配置
systemctl status nginx        # 查看状态
systemctl enable nginx        # 开机自启
systemctl disable nginx       # 取消开机自启
systemctl is-enabled nginx    # 检查是否自启

# ====== 查看服务 ======
systemctl list-units --type=service           # 列出所有服务
systemctl list-units --type=service --state=running  # 运行中的服务
systemctl list-dependencies nginx              # 服务依赖

# ====== 服务日志 ======
journalctl -u nginx                  # 查看服务日志
journalctl -u nginx -f                # 实时跟踪
journalctl -u nginx --since "1 hour ago"
journalctl -u nginx -p err            # 错误级别
journalctl --vacuum-time=7d           # 清理7天前日志
```

### 服务配置文件

```ini
# /etc/systemd/system/nginx.service
[Unit]
Description=nginx http server
Documentation=https://nginx.org/en/docs/
After=network.target

[Service]
Type=forking
PIDFile=/run/nginx.pid
ExecStartPre=/usr/sbin/nginx -t
ExecStart=/usr/sbin/nginx
ExecReload=/bin/kill -s HUP $MAINPID
ExecStop=/bin/kill -s QUIT $MAINPID
PrivateTmp=true

[Install]
WantedBy=multi-user.target
```

## 四、进程监控

### 实时监控

```bash
# top
top -c                          # 显示完整命令
top -H                          # 显示线程
top -p PID                      # 监控特定进程
top -d 1 -n 10                  # 采样10次

# htop
htop                            # 交互式
htop -u username                # 只显示用户
htop -p PID1,PID2               # 监控指定进程

# atop
atop                            # 更详细的资源监控
atop -r                         # 回顾历史

# glances
glances                         # 跨平台监控
glances -t 2                    # 2秒刷新
```

### 进程详细信息

```bash
# 查看进程树
pstree -ap PID                  # 显示PID的进程树

# 查看进程打开的文件
lsof -p PID                     # 进程打开的所有文件
lsof -c java                    # java进程打开的文件
lsof -i :8080                  # 占用端口的进程
lsof -i tcp                    # 所有TCP连接

# 查看进程的命令行
cat /proc/PID/cmdline | tr '\0' ' '

# 查看进程环境变量
cat /proc/PID/environ | tr '\0' '\n'
```

## 五、进程资源限制

### ulimit

```bash
# 查看限制
ulimit -a                       # 所有限制
ulimit -n                       # 文件描述符数量
ulimit -u                       # 最大用户进程数

# 设置限制
ulimit -n 65535                 # 设置文件描述符
ulimit -u 4096                 # 设置最大进程数

# 永久设置：/etc/security/limits.conf
* soft nofile 65535
* hard nofile 65535
* soft nproc 4096
* hard nproc 4096
```

## 六、常见问题处理

### 进程CPU占用高

```bash
# 1. 找出CPU高的进程
top -c

# 2. 找出CPU高的线程
top -H -p PID

# 3. 获取线程堆栈
kill -3 PID    # 发送SIGQUIT，输出到stderr

# 4. 用perf采样
perf top -p PID
```

### 进程内存占用高

```bash
# 1. 找出内存高的进程
ps aux --sort=-%mem | head

# 2. 查看进程详细内存
cat /proc/PID/status | grep -E "VmRSS|VmSize|VmPeak"

# 3. 查看内存映射
cat /proc/PID/smaps

# 4. 使用pmap
pmap -x PID
```

### 僵死进程

```bash
# 1. 查找僵尸进程
ps aux | grep -w Z
top -b1 | grep -w Z

# 2. 查找僵尸进程父进程
ps -eo pid,ppid,stat,cmd | grep -w Z

# 3. 解决方案：杀死父进程
kill -9 PPID
```
