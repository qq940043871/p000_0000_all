## 一、进程基础概念

### 什么是进程

```
进程 = 程序(代码) + 运行环境(内存/文件/信号) + PCB(进程控制块)

PCB(进程控制块)包含：
- 进程ID(PID)
- 进程状态
- 程序计数器(PC)
- 寄存器上下文
- 内存管理信息
- I/O状态信息
- 调度信息
```

## 二、进程状态机

```
                    ┌──────────────┐
                    │   创建态     │
                    └──────┬───────┘
                           │ fork()
                           ▼
                    ┌──────────────┐
                    │   就绪态     │
                    │  (Ready)    │
                    └──────┬───────┘
                           │ 调度器选中
                           ▼
                    ┌──────────────┐
                    │   运行态     │
                    │ (Running)   │
                    └──────┬───────┘
                           │ 时间片用完/抢占
              ┌────────────┼────────────┐
              ▼            ▼            ▼
       ┌──────────┐  ┌──────────┐  ┌──────────┐
       │  就绪态   │  │  阻塞态   │  │   退出    │
       └──────────┘  └──────┬───┘  └──────────┘
                            │ I/O完成
                            ▼
                      ┌──────────┐
                      │  就绪态   │
                      └──────────┘

进程状态：
R = Running (运行或就绪)
S = Sleeping (可中断睡眠)
D = Disk Sleep (不可中断睡眠)
T = Stopped (暂停)
Z = Zombie (僵尸)
I = Idle (空闲)
```

## 三、进程创建

### fork()

```c
#include <unistd.h>
#include <stdio.h>
#include <sys/types.h>

int main() {
    pid_t pid = fork();  // 创建子进程
    
    if (pid < 0) {
        // 创建失败
        perror("fork failed");
    } else if (pid == 0) {
        // 子进程
        printf("Child: PID=%d, Parent PID=%d\n", getpid(), getppid());
    } else {
        // 父进程
        printf("Parent: PID=%d, Child PID=%d\n", getpid(), pid);
    }
    
    return 0;
}
```

### exec()

```c
#include <unistd.h>

int main() {
    // 用新程序替换当前进程映像
    execlp("ls", "ls", "-la", NULL);
    
    // 如果execlp返回，说明执行失败
    perror("execlp");
    return 1;
}
```

## 四、进程标识

```bash
# PID: 进程ID
# PPID: 父进程ID
# UID/GID: 用户/组ID
# SID: 会话ID
# PGID: 进程组ID

# 查看PID信息
cat /proc/self/status | grep -E "^Pid|^PPid|^Uid|^Gid"

# 进程树
pstree                       # 树形显示
pstree -p                    # 显示PID
pstree -u username          # 按用户分组

# 查看进程父子关系
ps -ef                       # 标准格式
ps -ejH                      # 显示进程树
```

## 五、进程相关命令

### ps命令

```bash
# 基础用法
ps                          # 当前终端进程
ps -e                        # 所有进程
ps -f                        # 完整格式
ps -ef                       # 完整格式所有进程
ps aux                       # BSD格式所有进程（最常用）

# 输出解释
# USER    PID   %CPU  %MEM   VSZ    RSS  TTY   STAT  START  TIME  COMMAND
# user   1234   0.1   0.5  123456  7890  ?    S     10:00  0:01  java -jar app.jar

# STAT状态说明
# R: 运行
# S: 睡眠
# D: 不可中断睡眠
# T: 暂停/跟踪
# Z: 僵尸
# s: 会话首进程
# l: 多线程
# +: 前台进程组
```

### top命令

```bash
# 基础
top                          # 交互式查看
top -d 1                     # 每秒刷新
top -p 1234                  # 监控特定进程
top -u username              # 只显示用户进程

# 交互命令
# P: 按CPU排序
# M: 按内存排序
# N: 按PID排序
# T: 按时间排序
# q: 退出
# k: 杀死进程
# r: 调整优先级
# 1: 显示所有CPU核心
# f: 选择显示字段

# top输出说明
top - 10:30:00 up 5 days, 2:15, 2 users, load average: 0.52, 0.48, 0.45
#                        运行时间     登录用户  1min  5min  15min 负载

Tasks: 245 total, 1 running, 244 sleeping, 0 stopped, 0 zombie
# 任务统计

%Cpu(s): 25.0 us, 5.0 sy, 0.0 ni, 68.0 id, 1.0 wa, 0.0 hi, 1.0 si, 0.0 st
# CPU使用率
# us: 用户态
# sy: 系统态
# ni: nice调整
# id: 空闲
# wa: I/O等待
# hi: 硬件中断
# si: 软件中断
# st: 被偷走(虚拟化)

KiB Mem: 32768000 total, 12345678 free, 15000000 used, 5422322 buff/cache
# 内存统计
```

## 六、/proc目录

```bash
# /proc/PID/ 重要文件
/proc/PID/cmdline        # 命令行参数
/proc/PID/environ        # 环境变量
/proc/PID/status         # 进程状态（简洁）
/proc/PID/statm           # 内存状态
/proc/PID/smaps          # 详细内存映射
/proc/PID/maps           # 内存映射
/proc/PID/fd/            # 文件描述符
/proc/PID/fdinfo/        # fd详情
/proc/PID/cwd            # 当前工作目录
/proc/PID/exe           # 可执行文件链接

# 示例：查看Java进程内存详情
cat /proc/$(pgrep java)/smaps | grep -A 5 "heap"

# 示例：查看进程打开的文件
ls -la /proc/PID/fd
```
