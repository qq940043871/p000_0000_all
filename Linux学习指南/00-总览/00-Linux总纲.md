## 🎯 核心知识点速查

| 场景 | 命令 | 说明 |
|------|------|------|
| **CPU飙升** | `top -H -p <pid>` | 查看线程CPU |
| | `perf top -p <pid>` | 热点函数 |
| **内存不足** | `free -h` | 内存概览 |
| | `cat /proc/<pid>/smaps` | 进程内存详情 |
| **进程消失** | `dmesg | grep -i oom` | OOM日志 |
| | `journalctl -u <service>` | 服务日志 |
| **网络问题** | `ss -tlnp` | 端口监听 |
| | `tcpdump -i eth0 port 80` | 抓包 |
| **磁盘IO高** | `iostat -x 1` | IO统计 |
| | `iotop` | 进程IO |
| **负载高** | `uptime` | 负载均值 |
| | `vmstat 1` | 综合统计 |

## ⚡ 线上排查快速手册

### 1. 系统整体状态（第一步必看）

```bash
# 系统负载
uptime

# 内存概览
free -h

# CPU概览
top -bn1 | head -20

# 磁盘IO
iostat -x 1 3

# 网络连接
ss -s
```

### 2. 快速定位问题

```bash
# CPU飙升 → 找进程 → 找线程 → 找代码
top -c                          # 找CPU高的进程
top -H -p <pid>                 # 找CPU高的线程
printf "%x\n" <tid>             # 线程ID转16进制
jstack <pid> | grep <hex_tid>   # Java找堆栈

# 内存不足 → 找进程 → 分析内存
ps aux --sort=-%mem | head      # 找内存大的进程
cat /proc/<pid>/status          # 进程内存详情
dmesg | grep -i oom             # OOM日志

# 进程消失 → 查日志 → 查OOM
journalctl -u <service> -n 100  # 服务日志
dmesg | tail -50                # 内核日志
```
