## 一、排查流程

```
┌─────────────────────────────────────────────┐
│  1. free -h - 查看系统内存使用               │
│  2. ps aux --sort=-%mem - 找出内存高的进程  │
│  3. cat /proc/PID/status - 查看进程内存      │
│  4. pmap -x PID - 查看内存映射              │
│  5. dmesg | grep -i oom - 查看OOM日志       │
└─────────────────────────────────────────────┘
```

## 二、内存分析

### 第一步：系统内存分析

```bash
# 查看内存使用
free -h

# 查看详细内存信息
cat /proc/meminfo

# 关键指标
# MemAvailable: 可用内存
# AnonPages: 匿名页(堆/栈)
# Cached: 文件缓存
# Buffers: 块设备缓存
# SwapTotal/SwapFree: Swap使用

# 查看内存趋势
vmstat 1 10
```

### 第二步：进程内存分析

```bash
# 按内存排序
ps aux --sort=-%mem | head -20

# 查看进程内存详情
ps -o pid,vsz,rss,pmem,pcpu,comm -p PID

# /proc/PID/status关键字段
# VmPeak: 虚拟内存峰值
# VmSize: 虚拟内存大小
# VmRSS: 物理内存(实际使用)
# VmData: 数据段大小
# VmStk: 栈大小

cat /proc/PID/status | grep -E "VmPeak|VmSize|VmRSS|VmData"
```

### 第三步：内存映射分析

```bash
# 查看内存映射
pmap -x PID

# 查看堆
pmap PID | grep "\[heap\]"

# 详细分析
cat /proc/PID/smaps | grep -A 10 "\[heap\]"

# 统计
pmap -X PID | tail -1
```

## 三、OOM分析

### OOM Killer原理

```
内存不足 → 内核尝试回收 → 失败 → OOM Killer启动 → 选择进程杀死

选择依据:
- oom_score (基于内存使用量和CPU时间)
- oom_score_adj (可调整: -1000到1000)
```

### 查看OOM日志

```bash
# 查看内核日志
dmesg | grep -i oom | tail -50
dmesg -T | grep -i oom              # 时间戳格式

# journalctl
journalctl -k | grep -i oom

# OOM统计
cat /proc/vmstat | grep -i oom

# 查看进程OOM分数
cat /proc/PID/oom_score
cat /proc/PID/oom_score_adj
```

### 防止OOM杀死关键进程

```bash
# 禁止杀死
echo -1000 > /proc/PID/oom_score_adj

# 更容易杀死
echo 1000 > /proc/PID/oom_score_adj

# 持久化配置
# /etc/security/limits.conf
```

## 四、内存泄漏排查

### 检测方法

```bash
# 1. 多次采样对比
cat /proc/PID/status | grep VmRSS
sleep 60
cat /proc/PID/status | grep VmRSS
sleep 60
cat /proc/PID/status | grep VmRSS

# 2. 写脚本监控
while true; do
    echo "$(date) $(cat /proc/PID/status | grep VmRSS)"
    sleep 10
done >> /tmp/mem.log

# 3. pmap持续观察
while true; do
    pmap PID | tail -1
    sleep 10
done
```

### Java内存泄漏

```bash
# jmap生成堆转储
jmap -dump:format=b,file=/tmp/heap.hprof PID

# jmap查看内存
jmap -heap PID
jmap -histo:live PID | head -30

# MAT分析
# https://eclipse.org/mat/
# 导入heap.hprof分析
```

### C/C++内存泄漏

```bash
# valgrind
valgrind --leak-check=full --show-leak-kinds=all ./program

# mtrace
# 在代码中添加
#include <mcheck.h>
mtrace();

# 运行后查看
# export MALLOC_TRACE=/tmp/trace
# ./program
# mtrace /tmp/trace
```

## 五、内存优化

### 系统级

```bash
# 减少缓存占用
sync && echo 3 > /proc/sys/vm/drop_caches

# 调整Swap倾向
sysctl -w vm.swappiness=10

# 调整水位
sysctl -w vm.min_free_kbytes=65536

# 查看并调整
cat /proc/sys/vm/swappiness
cat /proc/sys/vm/min_free_kbytes
```

### 应用级

```bash
# Java堆大小
# -Xms2g -Xmx2g
# 初始堆和最大堆设置为一样,避免动态调整

# 查看Java堆
jmap -heap PID

# 调整GC
# -XX:+UseG1GC
# -XX:MaxGCPauseMillis=200
```
