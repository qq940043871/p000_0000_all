# 进程消失排查手册

> 模块：性能排查
> 更新时间：2026-03-29

---

## 一、进程消失常见原因

```
1. OOM Killer杀死
2. 手动kill/kill -9
3. 代码调用exit()
4. 段错误/崩溃
5. 资源耗尽(exit code != 0)
6. 外部监控脚本杀死
```

---

## 二、排查步骤

### 第一步：查看系统日志

```bash
# 内核日志
dmesg | grep -i "killed process"
dmesg | grep -i oom
dmesg | grep -i segfault
dmesg -T | tail -100

# journalctl
journalctl -k | grep -i killed
journalctl -xe | tail -100
```

### 第二步：查看应用日志

```bash
# systemd服务日志
journalctl -u service-name --since "1 hour ago"
journalctl -u service-name -n 100 --no-pager

# 应用日志
tail -f /var/log/app.log
grep -i error /var/log/app.log

# /var/log/messages
grep "process" /var/log/messages
```

### 第三步：检查退出状态

```bash
# 查看历史进程
last | grep username
last | head -50

# 查看进程退出记录
grep PID /var/log/syslog

# 检查crontab
crontab -l -u username
grep username /var/log/cron
```

## 三、具体场景

### 场景1：OOM被杀

```
日志特征:
[ 1234.567890] Out of memory: Killed process 5678 (java) total-vm:12345678kB, anon-rss:2345678kB

排查:
1. 查看OOM前的内存使用
2. 检查进程内存配置
3. 调整OOM参数或增加内存
```

### 场景2：段错误崩溃

```
日志特征:
Segmentation fault (core dumped)
[12345.678901] java[12345]: segfault at 7f1234567890 ip 00007f1234567890 sp 00007fff12345678 error 4

排查:
1. 查看core文件
ls -la /proc/PID/cwd
coredumpctl list
coredumpctl info PID
gdb /path/to/binary /path/to/core
```

### 场景3：被杀脚本控制

```
排查:
1. 检查crontab
crontab -l
crontab -u username -l

2. 检查systemd定时任务
systemctl list-timers

3. 检查监控脚本
ps aux | grep monitor
```

## 四、持久化日志

### 启用进程审计

```bash
# 安装auditd
apt install auditd / yum install audit

# 启动
systemctl start auditd

# 添加规则监控进程
auditctl -w /usr/bin/kill -p x
auditctl -w /usr/bin/pkill -p x

# 查看审计日志
ausearch -k killed
ausearch -sc kill
```

### core文件配置

```bash
# 查看core配置
ulimit -c
cat /proc/sys/kernel/core_pattern

# 设置core文件大小
ulimit -c unlimited

# 设置core文件路径
echo "/tmp/core-%e-%p-%t" > /proc/sys/kernel/core_pattern

# 永久配置 /etc/sysctl.conf
kernel.core_pattern = /tmp/core-%e-%p-%t
kernel.core_uses_pid = 1
```

## 五、定位元凶

### 查看谁杀死了进程

```bash
# 审计日志
ausearch -sc kill | head -20

# strace跟踪
strace -p PID -f

# 父进程检查
ps -ef | grep PPID
```
