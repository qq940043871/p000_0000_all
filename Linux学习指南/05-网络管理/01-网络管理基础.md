## 一、网络协议栈

```

│              用户空间                          │

│              协议无关层                          │
│         Socket层 (sock, inet_sock)              │
└────────────────┬───────────────────────────────┘
                 │
┌────────────────▼───────────────────────────────┐
│              协议栈层                            │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐│
│  │TCP/UDP │ │  SCTP   │ │  DCCP   │ │ Raw IP  ││
│  └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘│
│       └───────────┴───────────┴───────────┘     │

                 │

                 │
┌────────────────▼───────────────────────────────┐
│              物理网卡                           │
└───────────────────────────────────────────────┘
```

## 二、网络配置命令

### ip命令

```bash
# ====== IP地址管理 ======
ip addr                        # 查看所有地址
ip addr show eth0              # 查看特定网卡
ip addr add 192.168.1.100/24 dev eth0   # 添加IP
ip addr del 192.168.1.100/24 dev eth0    # 删除IP
ip addr flush dev eth0         # 清除所有IP

# ====== 网卡管理 ======
ip link show                   # 查看网卡
ip link set eth0 up           # 启用网卡
ip link set eth0 down         # 禁用网卡
ip link set eth0 mtu 9000     # 设置MTU
ip link set eth0 promisc on   # 混杂模式
ip link set eth0 name eth1    # 重命名

# ====== 路由管理 ======
ip route                       # 查看路由表
ip route show                  # 同上
ip route add default via 192.168.1.1 dev eth0   # 添加默认路由
ip route add 192.168.2.0/24 via 192.168.1.1     # 添加静态路由
ip route del default                           # 删除默认路由

# ====== neighbor表 ======
ip neigh                       # 查看ARP表
ip neigh show                 # 同上
ip neigh add 192.168.1.1 lladdr aa:bb:cc:dd:ee:ff dev eth0  # 静态ARP
ip neigh del 192.168.1.1 dev eth0                        # 删除ARP
```

### ifconfig (老命令)

```bash
ifconfig                       # 查看所有网卡
ifconfig eth0                  # 查看特定网卡
ifconfig eth0 192.168.1.100 netmask 255.255.255.0  # 设置IP
ifconfig eth0 up               # 启用
ifconfig eth0 down             # 禁用
ifconfig eth0:0 10.0.0.1       # 添加子接口
```

## 三、网络诊断命令

### netstat

```bash
# 基础用法
netstat -a                     # 所有连接
netstat -t                     # TCP连接
netstat -u                     # UDP连接
netstat -l                     # 监听端口
netstat -p                     # 显示进程
netstat -n                     # 数字地址
netstat -r                     # 路由表

# 常用组合
netstat -tulnp                 # 监听端口+进程
netstat -anp                   # 所有连接+进程
netstat -rn                    # 路由表(数字)
netstat -i                     # 网卡统计
netstat -s                     # 协议统计

# 输出解释
# Proto: 协议(tcp/udp/unix)
# Recv-Q: 接收队列
# Send-Q: 发送队列
# Local Address: 本地地址
# Foreign Address: 远程地址
# State: 状态(LISTEN/ESTABLISHED/TIME_WAIT等)
```

### ss命令 (推荐替代netstat)

```bash
# 基础用法
ss -a                         # 所有连接
ss -t                         # TCP连接
ss -u                         # UDP连接
ss -l                         # 监听端口
ss -p                         # 显示进程
ss -n                         # 数字地址

# 常用组合
ss -tulnp                     # 监听端口+进程
ss -tuln                      # 监听端口(不解析)
ss -tan state established     # 已建立连接
ss -tan state listening       # 监听端口
ss -tan state time-wait       # TIME_WAIT

# Socket缓存
ss -m                         # 显示内存使用
ss -s                         # 汇总统计

# 高级过滤
ss dst 192.168.1.100          # 目标地址过滤
ss src :80                    # 源端口过滤
ss sport gt :1024              # 源端口>1024
ss dport eq :443              # 目标端口=443

# 输出解释
# State: 连接状态
# Recv-Q: 接收队列(未应用字节数)
# Send-Q: 发送队列(未确认字节数)
# Local Address:Port  本地地址
# Peer Address:Port   远程地址
```

## 四、连接状态分析

### TCP连接状态

```
状态说明:
LISTEN:        服务端监听中
SYN_SENT:      客户端发送SYN中
SYN_RECV:      收到SYN,发送SYN+ACK
ESTABLISHED:   连接已建立
FIN_WAIT_1:    主动关闭,发FIN
FIN_WAIT_2:    收到ACK,等待FIN
TIME_WAIT:     等待2MSL后关闭
CLOSE:         连接已关闭
CLOSE_WAIT:    被动关闭,等待关闭
LAST_ACK:      最后ACK
CLOSING:       同时关闭
```

### 常见问题分析

```bash
# ====== 大量TIME_WAIT ======
# 原因: 短连接过多
# 解决: 调整内核参数
sysctl -w net.ipv4.tcp_tw_reuse=1        # 复用TIME_WAIT
sysctl -w net.ipv4.tcp_tw_recycle=1      # 快速回收
sysctl -w net.ipv4.tcp_fin_timeout=30    # 缩短超时

# ====== 大量CLOSE_WAIT ======
# 原因: 对方关闭连接,本地未正确关闭socket
# 解决: 检查代码,确保正确关闭连接

# ====== 大量SYN_RECV ======
# 原因: 可能SYN Flood攻击
# 解决: 检查连接数,限制源IP
ss -tan state syn-recv | wc -l
```

## 五、网络统计

```bash
# ====== 网卡统计 ======
ip -s link show eth0          # 网卡统计
netstat -i                     # 网卡接口
cat /proc/net/dev              # 原始数据

# 字段说明:
# rx: 接收
# tx: 发送
# bytes: 字节数
# packets: 包数
# errs: 错误数
# drop: 丢弃数
# colls: 冲突数

# ====== 协议统计 ======
netstat -s                     # 所有协议统计
netstat -st                    # TCP统计
netstat -su                    # UDP统计
ss -s                          # Socket汇总

# ====== 连接统计 ======
cat /proc/net/sockstat         # Socket统计
cat /proc/net/tcp             # TCP连接详情
cat /proc/net/udp             # UDP连接详情
```

## 六、网络故障排查

```bash
# ====== 连通性测试 ======
ping 8.8.8.8                  # Ping测试
ping -c 4 8.8.8.8            # 4次Ping
ping -i 0.5 8.8.8.8          # 0.5秒间隔
ping -s 1472 8.8.8.8         # 指定包大小

# ====== 路由追踪 ======
traceroute 8.8.8.8           # 路由追踪(UDP)
traceroute -I 8.8.8.8        # ICMP追踪
traceroute -T 8.8.8.8        # TCP追踪
traceroute -n 8.8.8.8        # 不解析域名

tracert 8.8.8.8               # Windows traceroute

# ====== DNS查询 ======
nslookup example.com          # DNS查询
dig example.com               # 详细DNS信息
host example.com              # 简单查询
getent hosts example.com      # 系统hosts查询

# ====== 端口测试 ======
telnet 192.168.1.100 8080    # 端口连通性
nc -zv 192.168.1.100 8080     # 测试端口
nc -zv 192.168.1.100 1-1000  # 范围测试

# ====== MTU测试 ======
ping -M do -s 1400 8.8.8.8   # 测试MTU
```
