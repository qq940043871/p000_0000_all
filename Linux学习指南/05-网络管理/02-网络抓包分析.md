## 一、tcpdump基础

### 基本语法

```bash
# 基础用法
tcpdump                           # 抓包(需要root)
tcpdump -i eth0                   # 指定网卡
tcpdump -i any                    # 所有网卡
tcpdump -c 100                    # 抓100个包
tcpdump -w output.pcap            # 保存到文件
tcpdump -r output.pcap            # 读取文件

# 表达式
tcpdump host 192.168.1.100         # 特定主机
tcpdump port 80                    # 特定端口
tcpdump net 192.168.1.0/24        # 网段
tcpdump src 192.168.1.100         # 源地址
tcpdump dst 192.168.1.100         # 目标地址
```

### 协议过滤

```bash
# TCP/UDP/ICMP
tcpdump tcp                        # TCP
tcpdump udp                        # UDP
tcpdump icmp                       # ICMP
tcpdump arp                        # ARP
tcpdump port 443                   # HTTPS

# 端口范围
tcpdump "port 80 or port 443"
tcpdump "port 8080 or port 8443"

# 组合条件
tcpdump -i eth0 tcp and host 192.168.1.100
tcpdump -i eth0 tcp and "port 80 or port 443"
tcpdump -i eth0 "tcp[tcpflags] & tcp-syn != 0"
tcpdump -i eth0 "tcp[tcpflags] & tcp-ack != 0"
```

### 输出格式

```bash
# 详细输出
tcpdump -n -vv                     # 不解析域名,非常详细
tcpdump -nn                        # 不解析域名和端口
tcpdump -X                        # 显示十六进制
tcpdump -XX                       # 显示链路层头

# 时间戳
tcpdump -tttt                     # 人类可读时间戳
tcpdump -t                        # 不显示时间
tcpdump -ttttttt                  # 微秒精度

# 包内容
tcpdump -A                        # ASCII
tcpdump -X                        # 十六进制+ASCII
tcpdump -XX                       # 带链路层
```

## 二、tcpdump实战

### 常用场景

```bash
# HTTP请求
tcpdump -i eth0 -nn -A "tcp port 80 and (((ip[2:2] - ((ip[0]&0xf)<<2)) - ((tcp[12]&0xf0)>>2)) != 0)"

# HTTP GET请求
tcpdump -i eth0 -nn -A 'tcp[((tcp[12:1] & 0xf0) >> 2):4]' = 0x47455420

# HTTPS(SSL/TLS)连接
tcpdump -i eth0 -nn -w ssl.pcap "tcp port 443"

# 特定IP通信
tcpdump -i eth0 host 192.168.1.100 and port 80

# SYN包(连接建立)
tcpdump -i eth0 'tcp[tcpflags] & tcp-syn == tcp-syn'

# FIN包(连接关闭)
tcpdump -i eth0 'tcp[tcpflags] & tcp-fin == tcp-fin'

# RST包(重置连接)
tcpdump -i eth0 'tcp[tcpflags] & tcp-rst == tcp-rst'

# 超过100字节的包
tcpdump -i eth0 greater 100

# 监控多网卡
tcpdump -i eth0 -i eth1
```

## 三、Wireshark分析

### 过滤器语法

```
# 捕获过滤器(在tcpdump/wireshark捕获时使用)
tcp port 80
host 192.168.1.100
net 192.168.1.0/24
tcp[0:2] > 1024

# 显示过滤器(在wireshark查看时使用)
# 协议过滤
tcp
udp
icmp
dns

# IP过滤
ip.addr == 192.168.1.100
ip.src == 192.168.1.100
ip.dst == 192.168.1.100

# 端口过滤
tcp.port == 80
tcp.srcport == 80
tcp.dstport == 443

# TCP状态过滤
tcp.flags.syn == 1
tcp.flags.ack == 1
tcp.flags.fin == 1
tcp.flags.reset == 1

# HTTP过滤
http.request.method == "GET"
http.response.code == 200
http.host == "example.com"

# 组合过滤
ip.addr == 192.168.1.100 and tcp.port == 80
```

### 常用操作

```
1. 统计 → 协议分级        # 查看协议分布
2. 统计 → 对话            # 查看通信双方
3. 统计 → 端点            # 查看IP端点
4. 追踪流 → TCP流        # 追踪完整TCP会话
5. 专家信息               # 查看异常信息
```

## 四、流量分析

### HTTP分析

```bash
# 抓取HTTP请求
tcpdump -i eth0 -nn -A "tcp port 80 and tcp[((tcp[12] & 0xf0) >> 2)] = 0x47455420"

# curl模拟请求
curl -v http://example.com

# 查看完整HTTP会话
tcpdump -i eth0 -nn -A port 80 | grep -A 20 "GET\|POST"
```

### 连接问题分析

```bash
# 查看TCP状态分布
ss -tan | awk '{print $1}' | sort | uniq -c | sort -rn

# 连接数统计
ss -tan | awk '{print $4}' | cut -d: -f2 | sort | uniq -c | sort -rn

# TIME_WAIT过多
ss -tan | awk '/TIME-WAIT/{print $4}' | cut -d: -f2 | sort | uniq -c | sort -rn

# SYN_RECV攻击检测
ss -tan | awk '/SYN-RECV/{print $5}' | cut -d: -f1 | sort | uniq -c | sort -rn | head -10
```

## 五、网络性能分析

### 带宽测试

```bash
# iperf服务端
iperf -s                          # 启动服务端
iperf -s -p 5001                  # 指定端口

# iperf客户端
iperf -c server_ip                # 测试带宽
iperf -c server_ip -t 30          # 30秒测试
iperf -c server_ip -P 4          # 4并行流
iperf -c server_ip -u            # UDP测试
iperf -c server_ip -R            # 反向测试(下载)

# 其他带宽测试
speedtest-cli                     # 互联网测速
wget -O /dev/null http://speed.test.file
```

### 网络延迟分析

```bash
# TCP连接延迟
curl -o /dev/null -w "DNS: %{time_namelookup}s\nTCP: %{time_connect}s\nTTFB: %{time_starttransfer}s\nTotal: %{time_total}s\n" http://example.com

# TCP握手时间
tcpdump -i eth0 host 192.168.1.100 and port 80 -w handshake.pcap
# 在Wireshark中查看三次握手时间

# 查看重传
cat /proc/net/netstat | grep TcpExt | awk '{print $13, $14}'  # 数据包丢失
```
