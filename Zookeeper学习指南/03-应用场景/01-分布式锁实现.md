## 一、分布式锁原理

### 1.1 为什么需要分布式锁

```
分布式系统中的并发问题:

┌──────────┐  ┌──────────┐  ┌──────────┐
│  Node 1  │  │  Node 2  │  │  Node 3  │
│  ┌─────┐ │  │  ┌─────┐ │  │  ┌─────┐ │
│  │App  │ │  │  │App  │ │  │  │App  │ │
│  └──┬──┘ │  │  └──┬──┘ │  │  └──┬──┘ │
└─────┼────┘  └─────┼────┘  └─────┼────┘
      │             │             │
      └─────────────┼─────────────┘
                    │
              ┌─────▼─────┐
              │  共享资源  │
              │ (DB/Cache)│
              └───────────┘

问题: 多节点同时操作共享资源，需要协调
解决: 使用分布式锁保证互斥访问
```

### 1.2 ZooKeeper实现原理

```
ZooKeeper分布式锁原理:

1. 创建临时顺序节点
2. 判断自己是否是最小序号节点
3. 是最小节点 → 获得锁
4. 不是最小节点 → 监听前一个节点
5. 前一个节点删除 → 被唤醒，再次判断
```

## 二、简单分布式锁

### 2.1 基本实现

```java
import org.apache.zookeeper.*;
import org.apache.zookeeper.data.Stat;
import java.util.Collections;
import java.util.List;
import java.util.concurrent.CountDownLatch;

public class SimpleDistributedLock {
    private ZooKeeper zk;
    private String lockPath = "/locks";
    private String currentLock;
    
    public SimpleDistributedLock(ZooKeeper zk) {
        this.zk = zk;
        try {
            Stat stat = zk.exists(lockPath, false);
            if (stat == null) {
                zk.create(lockPath, new byte[0], 
                    ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT);
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
    
    public void lock() throws Exception {
        currentLock = zk.create(lockPath + "/lock-", 
            new byte[0], ZooDefs.Ids.OPEN_ACL_UNSAFE, 
            CreateMode.EPHEMERAL_SEQUENTIAL);
        
        while (true) {
            List<String> children = zk.getChildren(lockPath, false);
            Collections.sort(children);
            
            String currentNode = currentLock.substring(lockPath.length() + 1);
            int index = children.indexOf(currentNode);
            
            if (index == 0) {
                return;
            }
            
            String prevNode = lockPath + "/" + children.get(index - 1);
            CountDownLatch latch = new CountDownLatch(1);
            
            Stat stat = zk.exists(prevNode, event -> {
                if (event.getType() == Watcher.Event.EventType.NodeDeleted) {
                    latch.countDown();
                }
            });
            
            if (stat != null) {
                latch.await();
            }
        }
    }
    
    public void unlock() throws Exception {
        zk.delete(currentLock, -1);
    }
}
```

## 三、Curator分布式锁

### 3.1 可重入锁

```java
import org.apache.curator.framework.CuratorFramework;
import org.apache.curator.framework.CuratorFrameworkFactory;
import org.apache.curator.framework.recipes.locks.InterProcessMutex;
import org.apache.curator.retry.ExponentialBackoffRetry;

public class CuratorLockDemo {
    public static void main(String[] args) throws Exception {
        CuratorFramework client = CuratorFrameworkFactory.builder()
            .connectString("localhost:2181")
            .retryPolicy(new ExponentialBackoffRetry(1000, 3))
            .build();
        client.start();
        
        InterProcessMutex lock = new InterProcessMutex(client, "/locks/my-lock");
        
        try {
            System.out.println("尝试获取锁...");
            lock.acquire();
            System.out.println("获得锁，执行业务逻辑");
            
            Thread.sleep(5000);
            
        } finally {
            lock.release();
            System.out.println("释放锁");
        }
        
        client.close();
    }
}
```

## 四、分布式锁最佳实践

### 4.1 锁粒度

```
锁粒度选择:

粗粒度锁:
- 锁定整个资源
- 简单但并发度低

细粒度锁:
- 锁定资源的一部分
- 复杂但并发度高

示例:
- 粗粒度: /locks/order
- 细粒度: /locks/order/user_123
```

### 4.2 锁超时

```java
public class LockWithTimeout {
    private InterProcessMutex lock;
    private long timeout;
    
    public boolean tryLock() {
        try {
            return lock.acquire(timeout, TimeUnit.SECONDS);
        } catch (Exception e) {
            return false;
        }
    }
    
    public void unlock() {
        try {
            lock.release();
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

### 4.3 锁重入

```java
public class ReentrantLockDemo {
    private InterProcessMutex lock;
    
    public void method1() throws Exception {
        lock.acquire();
        try {
            method2();
        } finally {
            lock.release();
        }
    }
    
    public void method2() throws Exception {
        lock.acquire();
        try {
            System.out.println("可重入锁，再次获得锁");
        } finally {
            lock.release();
        }
    }
}
```

### 4.4 异常处理

```java
public void safeLock() {
    try {
        if (lock.acquire(10, TimeUnit.SECONDS)) {
            try {
                doBusiness();
            } catch (Exception e) {
                log.error("业务执行异常", e);
            } finally {
                try {
                    lock.release();
                } catch (Exception e) {
                    log.error("释放锁异常", e);
                }
            }
        }
    } catch (Exception e) {
        log.error("获取锁异常", e);
    }
}
```

## 五、分布式锁对比

| 实现方式 | 优点 | 缺点 |
|----------|------|------|
| ZooKeeper | 可靠、支持临时节点自动释放 | 性能较低、依赖ZK集群 |
| Redis | 性能高、实现简单 | 需要处理锁续期、主从切换问题 |
| 数据库 | 简单、易理解 | 性能差、存在单点问题 |
