## 一、缓存策略

### 1.1 缓存模式

```
常见缓存模式:

┌─────────────────────────────────────────────────────────────┐
│  1. Cache Aside (旁路缓存)                                  │
│     - 读: 先查缓存，miss则查DB并写入缓存                    │
│     - 写: 先更新DB，再删除缓存                              │
│                                                             │
│  2. Read Through (读穿透)                                   │
│     - 缓存层负责读取DB                                      │
│     - 应用只与缓存交互                                      │
│                                                             │
│  3. Write Through (写穿透)                                  │
│     - 写入缓存后，缓存层同步写入DB                          │
│                                                             │
│  4. Write Behind (异步写入)                                 │
│     - 写入缓存后，异步写入DB                                │
│     - 性能高但可能丢数据                                    │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 Cache Aside实现

```java
public String get(String key) {
    String value = jedis.get(key);
    if (value == null) {
        value = db.query(key);
        if (value != null) {
            jedis.setex(key, 3600, value);
        }
    }
    return value;
}

public void set(String key, String value) {
    db.update(key, value);
    jedis.del(key);
}
```

### 1.3 缓存穿透

```
问题: 查询不存在的数据，缓存和DB都没有

解决方案:
┌─────────────────────────────────────────────────────────────┐
│  1. 缓存空值                                                │
│     - 查询为空时缓存null值                                  │
│     - 设置较短过期时间                                      │
│                                                             │
│  2. 布隆过滤器                                              │
│     - 预先加载所有可能的数据到布隆过滤器                    │
│     - 查询前先检查布隆过滤器                                │
└─────────────────────────────────────────────────────────────┘
```

```java
public String get(String key) {
    String value = jedis.get(key);
    if (value == null) {
        value = db.query(key);
        if (value != null) {
            jedis.setex(key, 3600, value);
        } else {
            jedis.setex(key, 60, "NULL");
        }
    }
    if ("NULL".equals(value)) {
        return null;
    }
    return value;
}
```

### 1.4 缓存击穿

```
问题: 热点key过期，大量请求同时查询DB

解决方案:
┌─────────────────────────────────────────────────────────────┐
│  1. 互斥锁                                                  │
│     - 只允许一个线程查询DB                                  │
│     - 其他线程等待缓存刷新                                  │
│                                                             │
│  2. 逻辑过期                                                │
│     - 缓存不设置过期时间                                    │
│     - 数据中包含逻辑过期时间                                │
│     - 后台异步刷新                                          │
└─────────────────────────────────────────────────────────────┘
```

```java
public String getWithLock(String key) {
    String value = jedis.get(key);
    if (value == null) {
        String lockKey = "lock:" + key;
        if (jedis.setnx(lockKey, "1") == 1) {
            jedis.expire(lockKey, 10);
            try {
                value = db.query(key);
                jedis.setex(key, 3600, value);
            } finally {
                jedis.del(lockKey);
            }
        } else {
            try {
                Thread.sleep(50);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
            return getWithLock(key);
        }
    }
    return value;
}
```

### 1.5 缓存雪崩

```
问题: 大量缓存同时过期，请求全部打到DB

解决方案:
┌─────────────────────────────────────────────────────────────┐
│  1. 随机过期时间                                            │
│     - 过期时间加随机值，避免同时过期                        │
│                                                             │
│  2. 多级缓存                                                │
│     - 本地缓存 + Redis缓存                                  │
│                                                             │
│  3. 熔断降级                                                │
│     - DB压力过大时熔断                                      │
│     - 返回默认值或降级页面                                  │
└─────────────────────────────────────────────────────────────┘
```

```java
int baseExpire = 3600;
int randomExpire = new Random().nextInt(300);
jedis.setex(key, baseExpire + randomExpire, value);
```

## 二、分布式锁

### 2.1 分布式锁要求

```
分布式锁必须满足:
┌─────────────────────────────────────────────────────────────┐
│  1. 互斥性     - 任意时刻只有一个客户端持有锁              │
│  2. 防死锁     - 必须有超时机制，避免死锁                  │
│  3. 安全性     - 只能释放自己持有的锁                      │
│  4. 高可用     - 锁服务高可用                              │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 简单实现

```java
public boolean tryLock(String key, String value, int expireSeconds) {
    return jedis.setnx(key, value) == 1 && jedis.expire(key, expireSeconds) == 1;
}

public boolean releaseLock(String key, String value) {
    String script = 
        "if redis.call('GET', KEYS[1]) == ARGV[1] then " +
        "    return redis.call('DEL', KEYS[1]) " +
        "else " +
        "    return 0 " +
        "end";
    return (Long) jedis.eval(script, 
        Collections.singletonList(key), 
        Collections.singletonList(value)) == 1;
}
```

### 2.4 Redisson分布式锁

```java
import org.redisson.Redisson;
import org.redisson.api.RLock;
import org.redisson.api.RedissonClient;
import org.redisson.config.Config;

Config config = new Config();
config.useSingleServer()
    .setAddress("redis://localhost:6379");

RedissonClient redisson = Redisson.create(config);

RLock lock = redisson.getLock("myLock");

try {
    if (lock.tryLock(10, 30, TimeUnit.SECONDS)) {
        try {
            System.out.println("执行业务逻辑");
        } finally {
            lock.unlock();
        }
    } else {
        System.out.println("获取锁失败");
    }
} catch (InterruptedException e) {
    Thread.currentThread().interrupt();
}
```

### 2.5 Redisson看门狗

```
Redisson看门狗机制:

┌─────────────────────────────────────────────────────────────┐
│  1. 默认锁过期时间30秒                                      │
│  2. 后台线程每10秒检查一次                                  │
│  3. 如果锁还被持有，自动续期                                │
│  4. 客户端宕机，锁自动过期                                  │
└─────────────────────────────────────────────────────────────┘

使用方式:
RLock lock = redisson.getLock("myLock");
lock.lock();  // 使用看门狗自动续期
```

## 三、分布式限流

### 3.1 计数器限流

```java
public boolean allowRequest(String key, int limit, int windowSeconds) {
    long count = jedis.incr(key);
    if (count == 1) {
        jedis.expire(key, windowSeconds);
    }
    return count <= limit;
}
```

### 3.2 滑动窗口限流

```java
public boolean allowRequest(String key, int limit, int windowSeconds) {
    long now = System.currentTimeMillis();
    long windowStart = now - windowSeconds * 1000;
    
    jedis.zremrangeByScore(key, 0, windowStart);
    
    long count = jedis.zcard(key);
    if (count >= limit) {
        return false;
    }
    
    jedis.zadd(key, now, UUID.randomUUID().toString());
    jedis.expire(key, windowSeconds);
    return true;
}
```

### 3.3 令牌桶限流

```java
public boolean allowRequest(String key, int rate, int capacity) {
    String script = 
        "local tokens = redis.call('GET', KEYS[1]) " +
        "local last_time = redis.call('GET', KEYS[2]) " +
        "local now = tonumber(ARGV[1]) " +
        "local rate = tonumber(ARGV[2]) " +
        "local capacity = tonumber(ARGV[3]) " +
        "local requested = tonumber(ARGV[4]) " +
        "if tokens == false then tokens = capacity end " +
        "if last_time == false then last_time = now end " +
        "local delta = math.max(0, now - last_time) " +
        "local filled_tokens = math.min(capacity, tokens + delta * rate) " +
        "local allowed = filled_tokens >= requested " +
        "if allowed then filled_tokens = filled_tokens - requested end " +
        "redis.call('SET', KEYS[1], filled_tokens) " +
        "redis.call('SET', KEYS[2], now) " +
        "return allowed";
    
    return (Long) jedis.eval(script, 
        Arrays.asList(key + ":tokens", key + ":last_time"),
        Arrays.asList(
            String.valueOf(System.currentTimeMillis() / 1000.0),
            String.valueOf(rate),
            String.valueOf(capacity),
            "1"
        )) == 1;
}
```

## 四、消息队列

### 4.1 List实现队列

```java
public void produce(String queue, String message) {
    jedis.lpush(queue, message);
}

public String consume(String queue) {
    return jedis.rpop(queue);
}

public String blockConsume(String queue, int timeout) {
    List<String> result = jedis.brpop(timeout, queue);
    return result != null ? result.get(1) : null;
}
```

### 4.2 延迟队列

```java
public void produceDelay(String queue, String message, long delaySeconds) {
    long executeTime = System.currentTimeMillis() / 1000 + delaySeconds;
    jedis.zadd(queue, executeTime, message);
}

public List<String> consumeDelay(String queue) {
    long now = System.currentTimeMillis() / 1000;
    Set<String> messages = jedis.zrangeByScore(queue, 0, now, 0, 10);
    if (!messages.isEmpty()) {
        for (String message : messages) {
            jedis.zrem(queue, message);
        }
    }
    return new ArrayList<>(messages);
}
```

### 4.3 Stream消息队列

```java
public void produceStream(String stream, Map<String, String> message) {
    jedis.xadd(stream, new StreamEntryID(), message);
}

public List<StreamEntry> consumeStream(String stream, String group, String consumer) {
    List<Map.Entry<String, List<StreamEntry>>> result = 
        jedis.xreadGroup(group, consumer, 1, 0, false, 
            StreamEntryID.UNRECEIVED_ENTRY, stream);
    if (result != null && !result.isEmpty()) {
        return result.get(0).getValue();
    }
    return Collections.emptyList();
}

public void ackStream(String stream, String group, String id) {
    jedis.xack(stream, group, new StreamEntryID(id));
}
```

## 五、最佳实践总结

### 5.1 缓存最佳实践

```
1. 合理设置过期时间
   - 热点数据设置较长过期时间
   - 加随机值避免雪崩

2. 缓存穿透防护
   - 缓存空值
   - 使用布隆过滤器

3. 缓存击穿防护
   - 使用互斥锁
   - 逻辑过期

4. 缓存雪崩防护
   - 随机过期时间
   - 多级缓存
   - 熔断降级
```

### 5.2 分布式锁最佳实践

```
1. 使用Redisson
   - 内置看门狗机制
   - 支持可重入锁
   - 支持公平锁、读写锁

2. 设置合理超时
   - 业务执行时间 + 缓冲时间
   - 避免锁提前释放

3. 异常处理
   - finally中释放锁
   - 处理中断异常

4. 锁粒度
   - 细粒度锁提高并发
   - 避免大锁
```
