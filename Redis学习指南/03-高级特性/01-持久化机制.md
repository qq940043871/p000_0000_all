# 持久化机制

> 模块：高级特性
> 更新时间：2026-04-06

---

## 一、持久化概述

### 1.1 为什么需要持久化

```
Redis是内存数据库，重启后数据会丢失

持久化目的:
┌─────────────────────────────────────────────────────────────┐
│  ┌─────────────┐    持久化    ┌─────────────┐              │
│  │   内存数据   │ ──────────▶ │    磁盘     │              │
│  │  (易丢失)   │             │  (持久化)   │              │
│  └─────────────┘             └─────────────┘              │
│                                                             │
│  重启后从磁盘恢复数据，保证数据不丢失                       │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 持久化方式

| 方式 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| RDB | 快照持久化 | 文件小、恢复快 | 可能丢失数据 |
| AOF | 日志持久化 | 数据安全、可读 | 文件大、恢复慢 |
| 混合 | RDB+AOF | 兼具两者优点 | 配置复杂 |

## 二、RDB持久化

### 2.1 RDB原理

```
RDB (Redis Database) 是某个时间点的数据快照

生成过程:
┌─────────────────────────────────────────────────────────────┐
│  1. Redis主进程fork子进程                                   │
│  2. 子进程将内存数据写入临时文件                            │
│  3. 写入完成后替换旧的RDB文件                               │
│  4. 主进程继续处理请求                                      │
└─────────────────────────────────────────────────────────────┘

特点:
- fork时会有短暂阻塞
- 生成紧凑的二进制文件
- 适合备份和灾难恢复
```

### 2.2 RDB配置

```bash
save 900 1
save 300 10
save 60 10000

save ""

dbfilename dump.rdb

dir /var/lib/redis

stop-writes-on-bgsave-error yes

rdbcompression yes

rdbchecksum yes
```

### 2.3 手动触发RDB

```bash
BGSAVE

SAVE

LASTSAVE
```

### 2.4 RDB文件结构

```
RDB文件格式:

┌─────────────────────────────────────────────────────────────┐
│ REDIS   │ 版本 │ 数据库 │ 数据   │ EOF  │ 校验和          │
│ (魔数)  │      │ 选择器 │        │      │                 │
└─────────────────────────────────────────────────────────────┘

- 魔数: "REDIS"
- 版本: RDB版本号
- 数据库选择器: 数据库编号
- 数据: 键值对数据
- EOF: 结束标志
- 校验和: CRC64校验
```

## 三、AOF持久化

### 3.1 AOF原理

```
AOF (Append Only File) 记录所有写操作命令

工作流程:
┌─────────────────────────────────────────────────────────────┐
│  ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐  │
│  │ 写命令  │───▶│ AOF缓冲 │───▶│ AOF文件 │───▶│  磁盘   │  │
│  │         │    │  区     │    │         │    │         │  │
│  └─────────┘    └─────────┘    └─────────┘    └─────────┘  │
│                                                             │
│  1. 执行写命令                                              │
│  2. 命令追加到AOF缓冲区                                     │
│  3. 根据策略写入AOF文件                                     │
│  4. AOF重写压缩文件                                         │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 AOF配置

```bash
appendonly yes

appendfilename "appendonly.aof"

appendfsync always
appendfsync everysec
appendfsync no

no-appendfsync-on-rewrite no

auto-aof-rewrite-percentage 100
auto-aof-rewrite-min-size 64mb

aof-load-truncated yes

aof-use-rdb-preamble yes
```

### 3.3 AOF重写

```
AOF重写原理:

原AOF文件:
SET key1 value1
SET key1 value2
SET key1 value3
INCR counter
INCR counter
INCR counter

重写后:
SET key1 value3
SET counter 3

触发条件:
1. 手动触发: BGREWRITEAOF
2. 自动触发: 
   - AOF文件大小超过上次重写后大小的100%
   - AOF文件大小超过64MB
```

### 3.4 AOF文件修复

```bash
redis-check-aof --fix appendonly.aof
```

## 四、混合持久化

### 4.1 混合持久化原理

```
Redis 4.0+ 支持混合持久化

混合持久化 = RDB头部 + AOF尾部

┌─────────────────────────────────────────────────────────────┐
│                    混合AOF文件                              │
│  ┌─────────────────────┐  ┌─────────────────────┐          │
│  │     RDB格式数据     │  │     AOF格式数据     │          │
│  │   (全量快照)        │  │   (增量命令)        │          │
│  └─────────────────────┘  └─────────────────────┘          │
└─────────────────────────────────────────────────────────────┘

优点:
- RDB恢复速度快
- AOF数据完整性好
- 文件大小适中
```

### 4.2 混合持久化配置

```bash
appendonly yes
aof-use-rdb-preamble yes
```

## 五、持久化策略选择

### 5.1 场景对比

| 场景 | 推荐策略 | 说明 |
|------|----------|------|
| 允许分钟级丢失 | RDB | 性能最好 |
| 允许秒级丢失 | AOF(everysec) | 平衡选择 |
| 不允许丢失 | AOF(always) | 性能较差 |
| 大数据量恢复 | 混合持久化 | 综合最优 |

### 5.2 配置建议

```bash
appendonly yes
appendfsync everysec
aof-use-rdb-preamble yes

save 900 1
save 300 10
save 60 10000
```

## 七、备份恢复

### 7.1 RDB恢复

```bash
cp /backup/dump.rdb /var/lib/redis/dump.rdb
redis-server /etc/redis/redis.conf
```

### 7.2 AOF恢复

```bash
cp /backup/appendonly.aof /var/lib/redis/appendonly.aof
redis-check-aof --fix /var/lib/redis/appendonly.aof
redis-server /etc/redis/redis.conf
```

### 7.3 备份脚本

```bash
#!/bin/bash
BACKUP_DIR="/backup/redis"
DATE=$(date +%Y%m%d_%H%M%S)

redis-cli BGSAVE
sleep 5

cp /var/lib/redis/dump.rdb ${BACKUP_DIR}/dump_${DATE}.rdb
cp /var/lib/redis/appendonly.aof ${BACKUP_DIR}/appendonly_${DATE}.aof

find ${BACKUP_DIR} -name "*.rdb" -mtime +7 -delete
find ${BACKUP_DIR} -name "*.aof" -mtime +7 -delete
```
