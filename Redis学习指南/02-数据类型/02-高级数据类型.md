## 一、Bitmap（位图）

### 1.1 概述

```
Bitmap是基于String类型的位操作

特点:
- 每个bit只能是0或1
- 极度节省空间
- 适合二值状态统计
```

### 1.2 基本命令

```bash
SETBIT key offset 1
SETBIT key offset 0

GETBIT key offset

BITCOUNT key
BITCOUNT key 0 100

BITPOS key 1
BITPOS key 1 start end

BITOP AND dest key1 key2
BITOP OR dest key1 key2
BITOP XOR dest key1 key2
BITOP NOT dest key
```

### 1.3 应用场景

```bash
SETBIT user:signin:20260101 1001 1
GETBIT user:signin:20260101 1001

SETBIT user:signin:20260101 1001 1
SETBIT user:signin:20260102 1001 1
SETBIT user:signin:20260103 1001 1
BITCOUNT user:signin:20260101

BITOP AND result user:signin:20260101 user:signin:20260102
BITCOUNT result

SETBIT online:users 1001 1
BITCOUNT online:users
```

### 1.4 Java操作

```java
jedis.setbit("bitmap", 0, true);
jedis.setbit("bitmap", 1, false);

boolean bit = jedis.getbit("bitmap", 0);

long count = jedis.bitcount("bitmap");
long count = jedis.bitcount("bitmap", 0, 100);

long pos = jedis.bitpos("bitmap", true);

jedis.bitop(BitOP.AND, "result", "bitmap1", "bitmap2");
jedis.bitop(BitOP.OR, "result", "bitmap1", "bitmap2");
```

## 二、HyperLogLog

### 2.1 概述

```
HyperLogLog是基数统计的数据结构

特点:
- 极小内存占用(12KB)
- 统计不重复元素数量
- 存在约0.81%误差
```

### 2.2 基本命令

```bash
PFADD key element1 element2

PFCOUNT key

PFMERGE dest key1 key2
```

### 2.3 应用场景

```bash
PFADD page:home:uv user1 user2 user3
PFCOUNT page:home:uv

PFADD page:home:uv:20260101 user1 user2
PFADD page:home:uv:20260102 user2 user3
PFMERGE page:home:uv:week page:home:uv:20260101 page:home:uv:20260102
PFCOUNT page:home:uv:week
```

### 2.4 Java操作

```java
jedis.pfadd("hll", "a", "b", "c");

long count = jedis.pfcount("hll");

jedis.pfmerge("dest", "hll1", "hll2");
```

## 三、Geo（地理位置）

### 3.1 概述

```
Geo用于存储地理位置信息

特点:
- 基于Sorted Set实现
- 支持距离计算
- 支持范围查询
```

### 3.2 基本命令

```bash
GEOADD key longitude latitude member
GEOADD key 116.404 39.915 beijing
GEOADD key 121.474 31.230 shanghai

GEOPOS key member

GEODIST key member1 member2
GEODIST key member1 member2 km
GEODIST key member1 member2 m

GEORADIUS key longitude latitude radius km
GEORADIUS key 116.404 39.915 100 km WITHDIST
GEORADIUS key 116.404 39.915 100 km WITHCOORD

GEORADIUSBYMEMBER key member radius km
GEORADIUSBYMEMBER key beijing 100 km

GEOHASH key member
```

### 3.3 应用场景

```bash
GEOADD locations 116.404 39.915 beijing
GEOADD locations 121.474 31.230 shanghai
GEOADD locations 113.264 23.129 guangzhou

GEORADIUS locations 116.404 39.915 500 km WITHDIST

GEORADIUSBYMEMBER locations beijing 500 km

GEODIST locations beijing shanghai km
```

### 3.4 Java操作

```java
jedis.geoadd("locations", 116.404, 39.915, "beijing");
jedis.geoadd("locations", 121.474, 31.230, "shanghai");

List<GeoCoordinate> pos = jedis.geopos("locations", "beijing");

Double dist = jedis.geodist("locations", "beijing", "shanghai", GeoUnit.KM);

List<GeoRadiusResponse> radius = jedis.georadius("locations", 
    116.404, 39.915, 500, GeoUnit.KM);

List<GeoRadiusResponse> radiusByMember = jedis.georadiusByMember("locations", 
    "beijing", 500, GeoUnit.KM);
```

## 四、Stream（流）

### 4.1 概述

```
Stream是Redis 5.0引入的数据类型

特点:
- 类似消息队列
- 支持消费组
- 支持持久化
- 支持ACK确认
```

### 4.2 基本命令

```bash
XADD key * field value
XADD key * name "张三" age 25

XLEN key

XRANGE key - +
XRANGE key - + COUNT 10
XREVRANGE key + -

XREAD COUNT 10 STREAMS key $
XREAD BLOCK 5000 STREAMS key $

XGROUP CREATE key groupname $
XGROUP CREATE key group1 $ MKSTREAM

XREADGROUP GROUP group consumer COUNT 1 STREAMS key >
XREADGROUP GROUP group consumer BLOCK 5000 STREAMS key >

XACK key group ID
XACK mystream group1 1526569495631-0

XDEL key ID
XTRIM key MAXLEN 1000
XINFO STREAM key
```

### 4.3 应用场景

```bash
XADD orders * order_id 1001 user_id 2001 amount 99.9

XGROUP CREATE orders order_group $ MKSTREAM

XREADGROUP GROUP order_group consumer1 COUNT 1 STREAMS orders >

XACK orders order_group 1712304000000-0

XINFO STREAM orders
```

### 4.4 Java操作

```java
StreamEntryID id = jedis.xadd("mystream", 
    new StreamEntryID(), 
    Map.of("field", "value"));

long len = jedis.xlen("mystream");

List<StreamEntry> entries = jedis.xrange("mystream", 
    new StreamEntryID(), new StreamEntryID(), 10);

jedis.xgroupCreate("mystream", "mygroup", new StreamEntryID(), true);

List<Map.Entry<String, List<StreamEntry>>> result = 
    jedis.xreadGroup("mygroup", "consumer1", 
        1, 5000, false, new StreamEntryID().UNRECEIVED_ENTRY, "mystream");

jedis.xack("mystream", "mygroup", new StreamEntryID("1712304000000-0"));
```

## 五、Bitfield

### 5.1 概述

```
Bitfield用于对String进行位域操作

特点:
- 支持有符号/无符号整数
- 支持原子操作
- 支持溢出控制
```

### 5.2 基本命令

```bash
BITFIELD key GET u8 0
BITFIELD key GET i16 0
BITFIELD key SET u8 0 100
BITFIELD key INCRBY u8 0 1

BITFIELD key SET u8 0 100 GET u8 0

BITFIELD key OVERFLOW WRAP INCRBY u8 0 1
BITFIELD key OVERFLOW SAT INCRBY u8 0 1
BITFIELD key OVERFLOW FAIL INCRBY u8 0 1
```

### 5.3 应用场景

```bash
BITFIELD counter SET u32 0 0
BITFIELD counter INCRBY u32 0 1

BITFIELD user:1001 SET u8 0 25
BITFIELD user:1001 SET u8 8 1
BITFIELD user:1001 GET u8 0
```

## 六、数据类型选择指南

| 场景 | 推荐类型 | 说明 |
|------|----------|------|
| 简单缓存 | String | 最基础类型 |
| 对象存储 | Hash | 字段级别操作 |
| 消息队列 | List/Stream | Stream更强大 |
| 标签系统 | Set | 集合运算 |
| 排行榜 | Sorted Set | 自动排序 |
| 签到统计 | Bitmap | 节省空间 |
| UV统计 | HyperLogLog | 大数据量 |
| 附近的人 | Geo | 地理计算 |
| 消息系统 | Stream | 消费组支持 |
