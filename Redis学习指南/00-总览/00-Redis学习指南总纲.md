## 一、Redis概述

### 1.1 什么是Redis

```
Redis (Remote Dictionary Server) 是一个开源的内存数据结构存储系统

核心特点:
┌─────────────────────────────────────────────────────────────┐
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │   内存存储   │  │  数据结构   │  │   持久化    │         │
│  │  高性能读写  │  │  丰富类型   │  │  RDB/AOF   │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │   高可用    │  │   集群      │  │   单线程    │         │
│  │  主从复制   │  │  分布式     │  │  IO多路复用 │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 Redis应用场景

| 场景 | 说明 | 数据结构 |
|------|------|----------|
| 缓存 | 热点数据缓存 | String |
| 会话存储 | 分布式Session | String |
| 计数器 | 点赞、浏览量 | String |
| 排行榜 | 实时排名 | Sorted Set |
| 消息队列 | 简单队列 | List/Stream |
| 分布式锁 | 并发控制 | String/Lua |
| 社交网络 | 关注、粉丝 | Set |
| 地理位置 | 附近的人 | Geo |

### 1.3 Redis优势

```
1. 高性能
   - 内存操作，读写速度极快
   - 单机QPS可达10万+

2. 丰富数据类型
   - String、List、Set、Hash、Sorted Set
   - Bitmap、HyperLogLog、Geo、Stream

3. 持久化
   - RDB快照
   - AOF日志
   - 混合持久化

4. 高可用
   - 主从复制
   - Sentinel哨兵
   - Cluster集群

5. 功能丰富
   - 事务
   - Lua脚本
   - Pub/Sub
   - Pipeline
```

## 三、快速开始

### 3.1 安装Redis

```bash
# Ubuntu/Debian
apt-get update
apt-get install redis-server

# CentOS/RHEL
yum install redis

# macOS
brew install redis

# Docker
docker run -d --name redis -p 6379:6379 redis:7
```

### 3.2 启动Redis

```bash
# 前台启动
redis-server

# 指定配置文件
redis-server /etc/redis/redis.conf

# 后台启动
redis-server --daemonize yes

# 检查状态
redis-cli ping
# PONG
```

### 3.3 基本操作

```bash
# 连接Redis
redis-cli
redis-cli -h 127.0.0.1 -p 6379 -a password

# 基本命令
SET key value
GET key
DEL key
EXISTS key
EXPIRE key 60
TTL key

# 查看信息
INFO
DBSIZE
CONFIG GET *
```

## 四、核心概念

### 4.1 单线程模型

```
Redis为什么单线程还这么快?

┌─────────────────────────────────────────────────────────────┐
│  1. 纯内存操作                                              │
│     - 内存访问速度极快(纳秒级)                              │
│                                                             │
│  2. IO多路复用                                              │
│     - epoll/kqueue高效处理大量连接                          │
│     - 非阻塞IO                                              │
│                                                             │
│  3. 单线程避免开销                                          │
│     - 无多线程上下文切换                                    │
│     - 无锁竞争开销                                          │
│     - 无CPU缓存失效                                         │
└─────────────────────────────────────────────────────────────┘

注意: Redis 6.0+ 引入多线程处理网络IO
```

### 4.2 内存管理

```
内存淘汰策略:

1. noeviction      - 不淘汰，内存满时返回错误
2. allkeys-lru     - 所有key中LRU淘汰
3. volatile-lru    - 设置过期时间的key中LRU淘汰
4. allkeys-random  - 所有key中随机淘汰
5. volatile-random - 设置过期时间的key中随机淘汰
6. volatile-ttl    - 淘汰即将过期的key
7. allkeys-lfu     - 所有key中LFU淘汰(Redis 4.0+)
8. volatile-lfu    - 设置过期时间的key中LFU淘汰

配置:
maxmemory 1gb
maxmemory-policy allkeys-lru
```

### 4.3 过期策略

```
Redis过期策略:

1. 惰性删除
   - 访问key时检查是否过期
   - 过期则删除

2. 定期删除
   - 定期随机检查部分key
   - 删除过期的key

3. 内存淘汰
   - 内存不足时触发淘汰策略
```

## 五、性能指标

### 5.1 性能基准

```
Redis性能参考(单实例):

操作类型        QPS         延迟
─────────────────────────────────
GET/SET        10万+       < 1ms
INCR           10万+       < 1ms
LPUSH/RPUSH    10万+       < 1ms
HSET/HGET      10万+       < 1ms
SADD           10万+       < 1ms
ZADD           8万+        < 1ms
```

### 5.2 性能优化建议

```
1. 避免大Key
   - 单个Key的Value不要超过10KB
   - 使用Hash存储多个小字段

2. 合理使用数据类型
   - 选择合适的数据类型
   - 避免使用复杂操作

3. 批量操作
   - 使用Pipeline减少网络往返
   - 使用Lua脚本保证原子性

4. 连接池
   - 复用连接减少开销
   - 合理设置连接数

5. 内存优化
   - 设置合理的过期时间
   - 选择合适的淘汰策略
```
