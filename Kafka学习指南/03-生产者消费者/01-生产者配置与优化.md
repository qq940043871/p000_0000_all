## 二、消息发送方式

### 2.1 发后即忘(Fire-and-Forget)

```java
Producer<String, String> producer = new KafkaProducer<>(props);

producer.send(new ProducerRecord<>("topic", "key", "value"));

producer.close();
```

### 2.2 同步发送(Sync)

```java
try {
    RecordMetadata metadata = producer.send(
        new ProducerRecord<>("topic", "key", "value")
    ).get();
    
    System.out.printf("发送成功: partition=%d, offset=%d%n",
        metadata.partition(), metadata.offset());
} catch (Exception e) {
    e.printStackTrace();
}
```

### 2.3 异步发送(Async)

```java
producer.send(new ProducerRecord<>("topic", "key", "value"), 
    new Callback() {
        @Override
        public void onCompletion(RecordMetadata metadata, Exception exception) {
            if (exception != null) {
                System.err.println("发送失败: " + exception.getMessage());
            } else {
                System.out.printf("发送成功: partition=%d, offset=%d%n",
                    metadata.partition(), metadata.offset());
            }
        }
    }
);
```

### 2.4 Lambda方式

```java
producer.send(new ProducerRecord<>("topic", "key", "value"), 
    (metadata, exception) -> {
        if (exception != null) {
            log.error("发送失败", exception);
        } else {
            log.info("发送成功: partition={}, offset={}", 
                metadata.partition(), metadata.offset());
        }
    }
);
```

## 三、生产者核心机制

### 3.1 消息批次

```
Producer内部流程:

┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   Record    │───▶│   Buffer    │───▶│   Batch     │
│  (消息)     │    │  (缓冲区)   │    │  (批次)     │
└─────────────┘    └─────────────┘    └──────┬──────┘
                                             │
                   ┌─────────────┐           │
                   │   Sender    │◀──────────┘
                   │   Thread    │
                   └──────┬──────┘
                          │
                   ┌──────▼──────┐
                   │   Broker    │
                   └─────────────┘

批次触发条件:
1. batch.size 达到阈值
2. linger.ms 超时
3. 缓冲区满
```

### 3.2 压缩算法对比

| 算法 | 压缩率 | CPU消耗 | 适用场景 |
|------|--------|---------|----------|
| none | 1:1 | 最低 | 网络带宽充足 |
| gzip | 高 | 高 | 带宽受限 |
| snappy | 中 | 低 | 平衡场景 |
| lz4 | 中 | 最低 | 高吞吐场景 |
| zstd | 高 | 中 | Kafka 2.1+ |

### 3.3 幂等性实现

```java
props.put("enable.idempotence", true);

props.put("enable.idempotence", true);
props.put("acks", "all");
props.put("retries", Integer.MAX_VALUE);
props.put("max.in.flight.requests.per.connection", 5);
```

## 四、生产者配置详解

### 4.1 完整配置示例

```java
Properties props = new Properties();

props.put("bootstrap.servers", "kafka1:9092,kafka2:9092,kafka3:9092");
props.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
props.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");

props.put("acks", "all");
props.put("retries", 3);
props.put("retry.backoff.ms", 100);
props.put("enable.idempotence", true);
props.put("max.in.flight.requests.per.connection", 5);

props.put("batch.size", 32768);
props.put("linger.ms", 10);
props.put("buffer.memory", 67108864);
props.put("compression.type", "lz4");

props.put("max.block.ms", 60000);
props.put("request.timeout.ms", 30000);
props.put("delivery.timeout.ms", 120000);

props.put("client.id", "order-service-producer");
props.put("connections.max.idle.ms", 540000);

KafkaProducer<String, String> producer = new KafkaProducer<>(props);
```

### 4.2 参数说明表

| 参数 | 默认值 | 说明 |
|------|--------|------|
| acks | all | 确认机制(0/1/all/-1) |
| retries | 2147483647 | 重试次数 |
| retry.backoff.ms | 100 | 重试间隔 |
| batch.size | 16384 | 批次大小(字节) |
| linger.ms | 0 | 等待时间(毫秒) |
| buffer.memory | 33554432 | 缓冲区大小 |
| compression.type | none | 压缩算法 |
| max.block.ms | 60000 | 最大阻塞时间 |
| request.timeout.ms | 30000 | 请求超时 |
| delivery.timeout.ms | 120000 | 投递超时 |

## 五、生产者监控

### 5.1 关键指标

```java
import org.apache.kafka.common.Metric;
import org.apache.kafka.common.MetricName;

Map<MetricName, ? extends Metric> metrics = producer.metrics();

for (Map.Entry<MetricName, ? extends Metric> entry : metrics.entrySet()) {
    System.out.println(entry.getKey().name() + ": " + entry.getValue().metricValue());
}
```

### 5.2 重要监控项

| 指标 | 说明 |
|------|------|
| record-send-rate | 消息发送速率 |
| record-error-rate | 消息错误速率 |
| record-queue-time-avg | 消息排队时间 |
| request-latency-avg | 请求延迟 |
| batch-size-avg | 平均批次大小 |
| compression-rate-avg | 压缩率 |
| buffer-available-bytes | 可用缓冲区 |
| io-wait-time-ns-avg | IO等待时间 |
