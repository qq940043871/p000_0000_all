## 一、消息偏移量(Offset)

### 1.1 Offset概念

```
Offset是消息在分区中的唯一标识

Partition 0:
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│  0  │  1  │  2  │  3  │  4  │  5  │  6  │
├─────┼─────┼─────┼─────┼─────┼─────┼─────┤
│msg1 │msg2 │msg3 │msg4 │msg5 │msg6 │msg7 │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┘
                      ↑
              Consumer Offset = 3
              (已消费到offset 2，下一条是3)
```

### 1.2 Offset特点

- **单调递增**：每个分区内的Offset从0开始递增
- **分区独立**：不同分区的Offset独立
- **持久化**：Offset存储在`__consumer_offsets`主题中
- **可重置**：可以手动重置消费位置

### 1.3 Offset提交模式

```java
Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("group.id", "test-group");
props.put("key.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");
props.put("value.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");

props.put("enable.auto.commit", "true");
props.put("auto.commit.interval.ms", "5000");

props.put("enable.auto.commit", "false");

KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props);
consumer.subscribe(Arrays.asList("my-topic"));

while (true) {
    ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));
    
    for (ConsumerRecord<String, String> record : records) {
        System.out.printf("offset=%d, value=%s%n", record.offset(), record.value());
    }
    
    consumer.commitSync();
    
    consumer.commitAsync();
}
```

## 二、消费组(Consumer Group)

### 2.1 消费组概念

```
消费组是消费者的逻辑分组

┌─────────────────────────────────────────────────────────────┐
│                    Consumer Group: group1                   │
│                                                             │
│  Topic: my-topic (3 partitions)                            │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ Partition 0 │  │ Partition 1 │  │ Partition 2 │         │
│  └──────┬──────┘  └──────┬──────┘  └──────┬──────┘         │
│         │                │                │                 │
│         ▼                ▼                ▼                 │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ Consumer 1  │  │ Consumer 2  │  │ Consumer 3  │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
└─────────────────────────────────────────────────────────────┘

规则：一个分区只能被同一消费组中的一个消费者消费
```

### 2.2 消费组特性

```
1. 消息广播：不同消费组独立消费同一消息
2. 负载均衡：同组消费者分担分区消费
3. 容错能力：消费者故障时自动Rebalance
4. 偏移量管理：每个消费组独立维护Offset
```

### 2.3 消费者数量与分区关系

```
场景1：消费者数 = 分区数
┌───────────┐  ┌───────────┐  ┌───────────┐
│Partition 0│  │Partition 1│  │Partition 2│
└─────┬─────┘  └─────┬─────┘  └─────┬─────┘
      │              │              │
      ▼              ▼              ▼
┌───────────┐  ┌───────────┐  ┌───────────┐
│Consumer 1 │  │Consumer 2 │  │Consumer 3 │
└───────────┘  └───────────┘  └───────────┘

场景2：消费者数 > 分区数
┌───────────┐  ┌───────────┐
│Partition 0│  │Partition 1│
└─────┬─────┘  └─────┬─────┘
      │              │
      ▼              ▼
┌───────────┐  ┌───────────┐  ┌───────────┐
│Consumer 1 │  │Consumer 2 │  │Consumer 3 │
└───────────┘  └───────────┘  │  (空闲)   │
                              └───────────┘

场景3：消费者数 < 分区数
┌───────────┐  ┌───────────┐  ┌───────────┐  ┌───────────┐
│Partition 0│  │Partition 1│  │Partition 2│  │Partition 3│
└─────┬─────┘  └─────┬─────┘  └─────┬─────┘  └─────┬─────┘
      │              │              │              │
      └──────────────┼──────────────┼──────────────┘
                     │              │
                     ▼              ▼
              ┌───────────┐  ┌───────────┐
              │Consumer 1 │  │Consumer 2 │
              │  (P0,P2)  │  │  (P1,P3)  │
              └───────────┘  └───────────┘
```

## 三、Rebalance机制

### 3.1 Rebalance触发条件

```
1. 消费者加入消费组
2. 消费者离开消费组（主动/故障）
3. Topic分区数变化
4. 消费者订阅的Topic变化
5. 会话超时（session.timeout.ms）
6. 心跳超时（heartbeat.timeout.ms）
```

### 3.2 Rebalance过程

```
1. 所有消费者停止消费
2. 消费者发送JoinGroup请求
3. 选择Group Leader
4. Leader分配分区
5. 分发分配结果
6. 消费者开始消费
```

### 3.3 Rebalance配置优化

```java
Properties props = new Properties();

props.put("session.timeout.ms", "10000");
props.put("heartbeat.interval.ms", "3000");
props.put("max.poll.interval.ms", "300000");
props.put("max.poll.records", "500");

props.put("partition.assignment.strategy", 
    "org.apache.kafka.clients.consumer.RangeAssignor");
props.put("partition.assignment.strategy", 
    "org.apache.kafka.clients.consumer.RoundRobinAssignor");
props.put("partition.assignment.strategy", 
    "org.apache.kafka.clients.consumer.StickyAssignor");
props.put("partition.assignment.strategy", 
    "org.apache.kafka.clients.consumer.CooperativeStickyAssignor");
```

## 四、Offset管理

### 4.1 查看消费组Offset

```bash
kafka-consumer-groups.sh --list --bootstrap-server localhost:9092

kafka-consumer-groups.sh --describe \
  --group my-group \
  --bootstrap-server localhost:9092

kafka-consumer-groups.sh --describe \
  --group my-group \
  --state \
  --bootstrap-server localhost:9092
```

### 4.2 重置Offset

```bash
kafka-consumer-groups.sh --reset-offsets \
  --group my-group \
  --topic my-topic \
  --to-earliest \
  --bootstrap-server localhost:9092 \
  --dry-run

kafka-consumer-groups.sh --reset-offsets \
  --group my-group \
  --topic my-topic \
  --to-latest \
  --bootstrap-server localhost:9092 \
  --execute

kafka-consumer-groups.sh --reset-offsets \
  --group my-group \
  --topic my-topic \
  --to-offset 100 \
  --bootstrap-server localhost:9092 \
  --execute

kafka-consumer-groups.sh --reset-offsets \
  --group my-group \
  --topic my-topic \
  --to-datetime 2026-01-01T00:00:00.000 \
  --bootstrap-server localhost:9092 \
  --execute

kafka-consumer-groups.sh --reset-offsets \
  --group my-group \
  --topic my-topic \
  --shift-by -10 \
  --bootstrap-server localhost:9092 \
  --execute
```

### 4.3 Java代码重置Offset

```java
consumer.subscribe(Arrays.asList("my-topic"));

Set<TopicPartition> partitions = consumer.assignment();
for (TopicPartition partition : partitions) {
    consumer.seek(partition, 0);
}

consumer.seekToBeginning(partitions);

consumer.seekToEnd(partitions);

TopicPartition partition = new TopicPartition("my-topic", 0);
consumer.seek(partition, 100);
```

## 五、消费语义

### 5.1 三种语义

| 语义 | 说明 | 实现方式 |
|------|------|----------|
| At Most Once | 最多一次，可能丢失 | 先提交offset再处理 |
| At Least Once | 至少一次，可能重复 | 先处理再提交offset |
| Exactly Once | 精确一次 | 事务+幂等 |

### 5.2 At Least Once实现

```java
while (true) {
    ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));
    
    for (ConsumerRecord<String, String> record : records) {
        try {
            processMessage(record);
            consumer.commitSync(Collections.singletonMap(
                new TopicPartition(record.topic(), record.partition()),
                new OffsetAndMetadata(record.offset() + 1)
            ));
        } catch (Exception e) {
            log.error("处理失败", e);
        }
    }
}
```

### 5.3 Exactly Once实现

```java
KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props);
KafkaProducer<String, String> producer = new KafkaProducer<>(producerProps);

consumer.subscribe(Arrays.asList("input-topic"));

while (true) {
    ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));
    
    producer.beginTransaction();
    
    try {
        for (ConsumerRecord<String, String> record : records) {
            producer.send(new ProducerRecord<>("output-topic", record.key(), record.value()));
        }
        
        producer.sendOffsetsToTransaction(
            getOffsets(records), 
            consumer.groupMetadata()
        );
        
        producer.commitTransaction();
    } catch (Exception e) {
        producer.abortTransaction();
    }
}
```
