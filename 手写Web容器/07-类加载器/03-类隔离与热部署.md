# 类隔离与热部署

> 模块：类加载器
> 更新时间：2026-03-29

## 一、类隔离原理

```
问题：
  - 应用A依赖 commons-lang 3.0
  - 应用B依赖 commons-lang 2.0
  - 两个版本不能共存

解决：
  - 每个应用使用独立的ClassLoader
  - 类加载器不同，即使类名相同也是不同的类
  - 实现应用级隔离
```

## 二、热部署实现

```java
public class HotDeployManager {
    private Map<String, Long> lastModified = new ConcurrentHashMap<>();
    private Context context;
    
    public void checkAndReload() {
        File webInf = new File(context.getDocBase(), "WEB-INF");
        File classesDir = new File(webInf, "classes");
        File libDir = new File(webInf, "lib");
        
        boolean needReload = false;
        
        // 检查classes目录
        needReload |= checkDirectory(classesDir);
        
        // 检查lib目录
        if (libDir.exists()) {
            File[] jars = libDir.listFiles();
            if (jars != null) {
                for (File jar : jars) {
                    needReload |= checkFile(jar);
                }
            }
        }
        
        if (needReload) {
            reload();
        }
    }
    
    private boolean checkDirectory(File dir) {
        if (!dir.exists()) return false;
        
        File[] files = dir.listFiles();
        if (files == null) return false;
        
        boolean changed = false;
        for (File file : files) {
            if (file.isDirectory()) {
                changed |= checkDirectory(file);
            } else {
                changed |= checkFile(file);
            }
        }
        return changed;
    }
    
    private boolean checkFile(File file) {
        String path = file.getAbsolutePath();
        long last = lastModified.getOrDefault(path, 0L);
        long current = file.lastModified();
        
        if (current > last) {
            lastModified.put(path, current);
            return true;
        }
        return false;
    }
    
    private void reload() {
        System.out.println("Reloading context: " + context.getPath());
        
        // 1. 停止当前应用
        context.stop();
        
        // 2. 创建新的ClassLoader
        WebAppClassLoader newLoader = new WebAppClassLoader(
            Thread.currentThread().getContextClassLoader(), context
        );
        
        // 3. 更新Context的ClassLoader
        context.setClassLoader(newLoader);
        
        // 4. 重新启动应用
        context.start();
    }
}
```
