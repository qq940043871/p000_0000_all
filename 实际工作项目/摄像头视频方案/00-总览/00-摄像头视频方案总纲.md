## ⚡ 快速部署

```bash
# 1. 启动MinIO
docker run -d --name minio -p 9000:9000 -p 9001:9001 \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=minioadmin \
  -v /data/minio:/data minio/minio

# 2. 启动ZLM
docker run -d --name zlm -p 1935:1935 -p 8080:80 -p 8554:554 \
  -v /data/zlm:/data panhome/zlmediakit

# 3. FFmpeg拉流推流
ffmpeg -rtsp_transport tcp -i rtsp://camera/live \
  -c copy -f flv rtmp://zlm/live/stream
```

## 📊 完整架构

```
┌──────────────┐
│   摄像头      │
│  (RTSP)      │
└──────┬───────┘
       │ RTSP
       ▼
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   FFmpeg     │────▶│     ZLM      │────▶│   播放器     │
│  (拉流)      │     │ (流媒体服务) │     │ (FLV/HLS)    │
└──────┬───────┘     └──────────────┘     └──────────────┘
       │                    │
       │ MP4分段            │ MP4/HLS
       ▼                    ▼
┌──────────────┐     ┌──────────────┐
│   MinIO      │     │   录制文件    │
│  (对象存储)   │     │   (本地)     │
└──────────────┘     └──────────────┘
```

---

## 🔥 核心流程

```
1. 拉流: FFmpeg rtsp://camera → 本地
2. 推流: FFmpeg → rtmp://zlm/live/stream
3. 分发: ZLM → RTMP/HLS/FLV/WebRTC
4. 录制: ZLM API → MP4文件 → MinIO
5. 回放: 播放器 → MinIO/HLS URL
```
