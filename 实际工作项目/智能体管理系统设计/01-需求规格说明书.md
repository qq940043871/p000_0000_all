# 智能体管理系统 - 需求规格说明书

> 版本：v1.0
> 日期：2026-03-30
> 技术栈：Spring Boot + Vue + UniApp

---

## 一、系统概述

### 1.1 项目背景
构建一套完整的智能体（AI Agent）管理平台，支持多租户、权限控制、知识库管理、工作流编排等功能。后端对接开源版 Deep Back API，支持 Llama / 豆包 Coding Plan 等大模型。

### 1.2 目标用户
- **平台管理员**：系统配置、租户管理
- **租户管理员**：用户管理、权限分配、资源配额
- **普通用户**：创建智能体、配置工作流、使用对话
- **开发者**：API 接入、二次开发

### 1.3 核心功能模块

```
┌─────────────────────────────────────────────────────────────┐
│                      智能体管理系统                           │
├─────────────┬─────────────┬─────────────┬─────────────────────┤
│  权限管理    │  用户管理    │  知识库管理  │    智能体管理        │
│  Permission │    User     │  Knowledge  │     Agent           │
├─────────────┼─────────────┼─────────────┼─────────────────────┤
│  工作流管理  │  文件资源    │  对话上下文  │    模型管理          │
│  Workflow   │    File     │   Context   │     Model           │
└─────────────┴─────────────┴─────────────┴─────────────────────┘
```

## 二、功能需求详述

### 2.1 权限管理模块

#### 2.1.1 功能列表
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 角色管理 | 定义角色（超级管理员、租户管理员、普通用户、访客） | P0 |
| 菜单权限 | 控制页面访问权限 | P0 |
| 按钮权限 | 控制操作权限（增删改查） | P0 |
| 数据权限 | 控制数据可见范围（全部/本部门/本人） | P1 |
| API权限 | 控制接口访问权限 | P1 |

#### 2.1.2 权限模型
采用 RBAC（Role-Based Access Control）模型：
- 用户 ↔ 角色 ↔ 权限 ↔ 资源

### 2.2 用户管理模块

#### 2.2.1 功能列表
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 用户注册/登录 | 支持账号密码、手机号、邮箱登录 | P0 |
| 用户CRUD | 增删改查用户基本信息 | P0 |
| 部门管理 | 组织架构管理 | P1 |
| 用户组管理 | 批量授权 | P1 |
| 登录日志 | 记录登录时间、IP、设备 | P2 |
| 操作日志 | 记录关键操作审计 | P2 |

### 2.3 知识库管理模块

#### 2.3.1 功能列表
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 知识库CRUD | 创建、编辑、删除知识库 | P0 |
| 文档上传 | 支持 PDF、Word、TXT、Markdown | P0 |
| 文档解析 | 自动分块、向量化存储 | P0 |
| 向量检索 | 相似度搜索、RAG 召回 | P0 |
| 知识库授权 | 共享给指定用户/部门 | P1 |
| 版本管理 | 文档版本控制 | P2 |

#### 2.3.2 支持的文档格式
- PDF（含扫描件 OCR）
- Word/Excel/PPT
- TXT/Markdown
- 网页 URL 抓取
- 数据库连接同步

### 2.4 智能体管理模块

#### 2.4.1 功能列表
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 智能体CRUD | 创建、配置、发布、删除智能体 | P0 |
| 提示词配置 | System Prompt、User Prompt 模板 | P0 |
| 模型选择 | 绑定 Llama / 豆包等模型 | P0 |
| 知识库绑定 | 关联知识库实现 RAG | P0 |

| 使用统计 | 调用次数、Token 消耗 | P2 |

#### 2.4.2 智能体配置参数
```yaml

  name: "客服助手"
  description: "企业内部客服智能体"
  model: "doubao-coding-plan"
  temperature: 0.7
  max_tokens: 4096
  system_prompt: "你是一个专业的客服助手..."
  knowledge_bases: ["kb_001", "kb_002"]
  tools: ["search", "calculator"]
  workflow: "wf_001"
```

## 三、非功能需求

### 3.1 性能需求
| 指标 | 目标值 |
|------|--------|
| 接口响应时间 | P99 < 500ms |
| 并发用户数 | 支持 1000+ 并发 |
| 文件上传速度 | 支持 100MB/s |
| 向量检索延迟 | < 100ms |

### 3.2 安全需求
- 数据传输：HTTPS/TLS 1.3
- 密码存储：BCrypt 加密
- API 安全：JWT Token + Refresh Token
- 敏感数据：AES-256 加密存储
- 审计日志：完整操作追溯

### 3.3 兼容性需求
- 浏览器：Chrome 90+、Firefox 88+、Edge 90+
- 移动端：iOS 12+、Android 8+
- 屏幕适配：PC（1920px）、平板（768px）、手机（375px）

## 五、接口规范

### 5.1 RESTful API 规范
```
GET    /api/v1/users          # 查询列表
GET    /api/v1/users/{id}     # 查询详情
POST   /api/v1/users          # 创建
PUT    /api/v1/users/{id}     # 更新
DELETE /api/v1/users/{id}     # 删除
```

### 5.2 统一响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1711800000000,
  "traceId": "xxx-xxx-xxx"
}
```

### 5.3 Deep Back API 对接
```java
// 模型调用接口
POST /v1/chat/completions
{
  "model": "llama-3-70b",
  "messages": [
    {"role": "system", "content": "..."},
    {"role": "user", "content": "..."}
  ],
  "temperature": 0.7,
  "stream": true
}
```
