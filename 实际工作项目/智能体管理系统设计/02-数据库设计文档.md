## 一、设计规范

### 1.1 命名规范
- 表名：`模块名_表名`，如 `sys_user`、`agent_bot`
- 字段名：小写下划线，如 `create_time`、`update_by`
- 主键：`id`，BigInt，自增
- 逻辑删除：`deleted` (0-未删除, 1-已删除)
- 时间字段：`create_time`、`update_time`
- 租户字段：`tenant_id`（多租户支持）

### 1.2 字段类型规范
| 数据类型 | 使用场景 |
|----------|----------|
| BIGINT | 主键、外键、大整数 |
| VARCHAR(64) | 用户名、编码等短字符串 |
| VARCHAR(255) | 名称、标题等 |
| VARCHAR(500) | URL、路径等 |
| TEXT | 描述、内容等长文本 |
| LONGTEXT | JSON、大文本 |
| INT | 状态、计数、排序 |
| TINYINT | 布尔值、小范围枚举 |
| DECIMAL(3,2) | 温度等小数 |
| DATETIME | 时间戳 |
| JSON | 配置、扩展字段 |
