## 一、垃圾回收概述

垃圾回收（Garbage Collection）是JVM自动内存管理的核心，负责回收不再使用的对象，释放内存空间。

### 垃圾判定算法

```java
// 1. 引用计数算法（存在循环引用问题）
public class ReferenceCountingGC {
    
    private int refCount = 0;
    
    public void addRef() {
        refCount++;
    }
    
    public void release() {
        refCount--;
        if (refCount == 0) {
            collect();
        }
    }
    
    // 问题：循环引用导致内存泄漏
    // A引用B，B引用A，但A和B都不再被使用时，refCount都不为0
}

// 2. 可达性分析算法（Java采用）
public class GCRootTracing {
    
    // GC Roots
    private static Set<Object> gcRoots = new HashSet<>();
    
    public static Set<Object> findGarbage() {
        Set<Object> reachable = new HashSet<>();
        Queue<Object> queue = new LinkedList<>();
        
        // 从GC Root开始遍历
        for (Object root : gcRoots) {
            queue.add(root);
            reachable.add(root);
        }
        
        // BFS遍历所有可达对象
        while (!queue.isEmpty()) {
            Object obj = queue.poll();
            for (Object ref : obj.getReferences()) {
                if (!reachable.contains(ref)) {
                    reachable.add(ref);
                    queue.add(ref);
                }
            }
        }
        
        // 不可达的对象就是垃圾
        Set<Object> allObjects = getAllObjects();
        allObjects.removeAll(reachable);
        return allObjects;
    }
}
```

## 二、GC Root对象

```java
// GC Root包括：
// 1. 虚拟机栈中的引用
// 2. 方法区中静态属性引用的对象
// 3. 方法区中常量引用的对象
// 4. 本地方法栈中JNI引用的对象
// 5. 活跃的线程
// 6. 内部对象（ClassLoader等）

public class GCRootSet {
    
    public static Set<Object> buildGCRoots() {
        Set<Object> roots = new HashSet<>();
        
        // 1. 虚拟机栈中的引用
        for (Frame frame : getAllStackFrames()) {
            for (Object local : frame.getLocalVariables()) {
                if (local != null) {
                    roots.add(local);
                }
            }
        }
        
        // 2. 方法区中的静态属性
        for (Class<?> clazz : getLoadedClasses()) {
            for (Field field : clazz.getStaticFields()) {
                Object value = field.get(null); // 获取静态字段值
                if (value != null) {
                    roots.add(value);
                }
            }
        }
        
        // 3. 方法区中的常量
        for (Class<?> clazz : getLoadedClasses()) {
            for (Object constant : clazz.getConstantPool()) {
                if (constant instanceof Object) {
                    roots.add(constant);
                }
            }
        }
        
        // 4. 本地方法栈中的引用
        for (NativeFrame frame : getNativeFrames()) {
            for (Object ref : frame.getReferences()) {
                roots.add(ref);
            }
        }
        
        return roots;
    }
}
```

## 四、垃圾回收算法

### 1. 标记-清除算法

```java
public class MarkSweepGC {
    
    public void collect() {
        // 1. 标记阶段
        Set<Object> garbage = findGarbage();
        
        // 2. 清除阶段
        for (Object obj : garbage) {
            freeMemory(obj);
        }
    }
    
    // 问题：产生内存碎片
}

// 2. 复制算法
public class CopyingGC {
    
    private Object[] fromSpace;
    private Object[] toSpace;
    private int toOffset;
    
    public void collect() {
        int free = 0;
        
        // 复制存活对象到toSpace
        for (Object obj : fromSpace) {
            if (obj != null && isLive(obj)) {
                toSpace[free++] = obj;
            }
        }
        
        // 交换fromSpace和toSpace
        swap(fromSpace, toSpace);
        
        // 清空原来的fromSpace
        clear(fromSpace);
    }
    
    // 优点：无内存碎片
    // 缺点：内存利用率低（只能使用一半）
}

// 3. 标记-整理算法
public class MarkCompactGC {
    
    public void collect() {
        // 1. 标记存活对象
        Set<Object> liveObjects = findLiveObjects();
        
        // 2. 整理：将存活对象向一端移动
        int compact = 0;
        for (Object obj : getHeapObjects()) {
            if (isLive(obj)) {
                moveObject(obj, compact++);
            }
        }
        
        // 3. 清理边界外的对象
        clearBeyond(compact);
    }
}
```

## 五、垃圾收集器

### Serial收集器

```java
public class SerialGC extends GarbageCollector {
    
    // 单线程执行，适合Client模式
    // 使用复制算法进行Young GC
    // 使用标记-整理算法进行Old GC
    
    public void collectGeneration(Generation generation) {
        if (generation instanceof YoungGeneration) {
            minorGC();
        } else {
            majorGC();
        }
    }
}

// ParNew收集器 - Serial的多线程版本
public class ParNewGC extends SerialGC {
    
    private int threads;
    
    public ParNewGC(int threads) {
        this.threads = threads;
    }
    
    @Override
    public void minorGC() {
        // 多线程并行执行Minor GC
        parallelCollect(threads);
    }
}

// CMS收集器
public class CMSGC extends GarbageCollector {
    
    // 并发标记清除收集器
    // 步骤：初始标记 → 并发标记 → 重新标记 → 并发清除
    
    public void collect() {
        // 1. 初始标记（Stop The World）
        initialMark();
        
        // 2. 并发标记
        concurrentMark();
        
        // 3. 重新标记（Stop The World）
        remark();
        
        // 4. 并发清除
        concurrentSweep();
    }
}

// G1收集器
public class G1GC extends GarbageCollector {
    
    // 将堆划分为多个Region
    // 跟踪各Region的垃圾堆积程度
    // 优先收集垃圾最多的Region
    
    private Region[] heapRegions;
    
    @Override
    public void collect() {
        // G1的Young GC或Mixed GC
        String which = ucms_mode ? "concurrent" : "partial";
        do {
            // 回收价值最大的Region
            Region maxGarbage = findMaxGarbageRegion();
            collectRegion(maxGarbage);
            
        } while (!shouldTerminate());
    }
}
```

## 六、总结

垃圾回收是JVM自动内存管理的核心，不同的GC算法和收集器有不同的特点和适用场景。

**学习要点**：
1. 理解可达性分析算法
2. 掌握GC Root的作用
3. 理解分代收集理论
4. 了解各种垃圾收集器的特点
