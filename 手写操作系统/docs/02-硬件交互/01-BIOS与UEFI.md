# 第五章：BIOS与UEFI

## 5.1 BIOS基础

### BIOS的作用

BIOS（Basic Input/Output System）是计算机启动时运行的第一个程序。

**主要功能：**
1. **硬件检测**：检测CPU、内存、磁盘等硬件
2. **初始化硬件**：初始化各种硬件设备
3. **加载引导程序**：从磁盘加载MBR到内存
4. **提供中断服务**：提供INT 0x10、INT 0x13等服务

### BIOS启动流程

```
上电 → POST（自检） → 初始化硬件 → 加载MBR → 执行MBR代码
```

### BIOS中断服务

#### INT 0x10 - 视频服务
```asm
; 设置视频模式
mov ah, 0x00
mov al, 0x03        ; 80x25文本模式
int 0x10

; 打印字符
mov ah, 0x0E
mov al, 'A'
mov bh, 0
mov bl, 0x07
int 0x10
```

#### INT 0x13 - 磁盘服务
```asm
; 读磁盘
mov ah, 0x02
mov al, 1           ; 读1个扇区
mov ch, 0           ; 柱面
mov cl, 2           ; 扇区
mov dh, 0           ; 磁头
mov dl, 0x80        ; 驱动器（0x80=第一个硬盘）
mov es:bx, 0x7e00   ; 缓冲区地址
int 0x13
```

#### INT 0x16 - 键盘服务
```asm
; 读取按键
mov ah, 0x00
int 0x16            ; AL中返回按键ASCII码
```

## 5.2 UEFI基础

### UEFI的优势

UEFI（Unified Extensible Firmware Interface）是BIOS的现代替代品。

**优势：**
1. **支持大磁盘**：支持超过2TB的磁盘
2. **图形界面**：提供图形化启动菜单
3. **安全启动**：支持Secure Boot
4. **模块化**：驱动程序可以动态加载
5. **网络启动**：支持PXE网络启动

### UEFI启动流程

```
上电 → SEC阶段 → PEI阶段 → DXE阶段 → BDS阶段 → 加载操作系统
```

## 5.3 MBR与GPT

### MBR（Master Boot Record）

MBR是磁盘的第一个扇区（512字节）。

**结构：**
```
0-445字节：引导代码
446-461字节：分区表项1
462-477字节：分区表项2
478-493字节：分区表项3
494-509字节：分区表项4
510-511字节：启动签名（0x55AA）
```

**分区表项结构：**
```
0字节：启动标志（0x80=可启动）
1-3字节：CHS地址
4字节：分区类型
5-7字节：CHS地址
8-11字节：LBA地址
12-15字节：分区大小
```

### GPT（GUID Partition Table）

GPT是现代分区表格式，支持更大的磁盘和更多分区。

**优势：**
- 支持超过2TB的磁盘
- 支持128个分区
- 分区表备份
- CRC校验

## 5.4 实模式到保护模式

### 实模式

**特点：**
- 16位寻址
- 最大1MB内存
- 无内存保护
- 兼容8086

### 保护模式

**特点：**
- 32位寻址
- 最大4GB内存
- 内存保护
- 特权级别

### 转换过程

```asm
; 1. 关闭中断
cli

; 2. 加载GDT
lgdt [gdt_ptr]

; 3. 设置CR0的PE位
mov eax, cr0
or eax, 1
mov cr0, eax

; 4. 长跳转到保护模式
jmp 0x08:protected_mode
```

## 5.5 全局描述符表（GDT）

### GDT结构

```
GDT表 ┌─────────────────────┐
      │  空描述符（8字节）  │
      ├─────────────────────┤
      │  代码段描述符       │
      ├─────────────────────┤
      │  数据段描述符       │
      ├─────────────────────┤
      │  ...                │
      └─────────────────────┘
```

### 描述符结构

```
63-56: 基址高8位
55:    粒度标志
54:    默认操作数大小
53:    保留
52:    可用
51-48: 段界限高4位
47:    P（存在位）
46-45: DPL（特权级）
44:    S（描述符类型）
43-40: 类型
39-32: 基址中间8位
31-16: 基址低16位
15-0:  段界限低16位
```

## 5.6 中断描述符表（IDT）

### IDT结构

```
IDT表 ┌─────────────────────┐
      │  中断门0            │
      ├─────────────────────┤
      │  中断门1            │
      ├─────────────────────┤
      │  ...                │
      ├─────────────────────┤
      │  中断门255          │
      └─────────────────────┘
```

### 中断门结构

```
63-48: 处理程序地址高16位
47-46: DPL（特权级）
45:    P（存在位）
44-40: 类型（0x0E=中断门）
39-32: 保留
31-16: 处理程序地址低16位
15-0:  段选择子
```
