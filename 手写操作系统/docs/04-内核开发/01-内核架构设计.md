# 第七章：内核架构设计

## 7.1 内核的基本结构

### 单体内核（Monolithic Kernel）

所有功能都在内核中实现。

**优点：**
- 性能高
- 实现简单

**缺点：**
- 代码复杂
- 难以维护
- 一个模块崩溃会导致整个系统崩溃

**例子：** Linux、Windows

### 微内核（Microkernel）

只在内核中实现最基本的功能，其他功能由用户态进程实现。

**优点：**
- 模块化
- 易于维护
- 稳定性高

**缺点：**
- 性能较低
- 进程间通信开销大

**例子：** MINIX、QNX

### 混合内核（Hybrid Kernel）

结合单体内核和微内核的优点。

**例子：** Windows NT、macOS

## 7.3 内核初始化流程

```
MBR → Bootloader → 内核入口 → 初始化GDT/IDT → 初始化内存 → 初始化进程 → 启动Shell
```

### 内核入口代码

```asm
[bits 32]

global kernel_main

kernel_main:
    ; 设置栈
    mov esp, 0x90000
    
    ; 调用C函数
    call kernel_init
    
    ; 无限循环
    jmp $
```

### 内核初始化函数

```c
void kernel_init() {
    // 初始化GDT
    init_gdt();
    
    // 初始化IDT
    init_idt();
    
    // 初始化内存管理
    init_memory();
    
    // 初始化进程管理
    init_process();
    
    // 初始化设备驱动
    init_drivers();
    
    // 启用中断
    enable_interrupts();
    
    // 启动第一个进程
    start_first_process();
}
```

## 7.4 系统调用接口

### 系统调用的作用

系统调用是用户程序与内核通信的接口。

### 常见系统调用

```c
// 进程管理
int fork();
int exec(const char *filename);
int exit(int status);
int wait(int *status);

// 文件操作
int open(const char *filename, int flags);
int close(int fd);
int read(int fd, void *buffer, int size);
int write(int fd, void *buffer, int size);

// 内存管理
void *malloc(int size);
void free(void *ptr);

// 设备操作
int ioctl(int fd, int cmd, void *arg);
```

### 系统调用的实现

```c
// 系统调用表
typedef int (*syscall_handler)(void);

syscall_handler syscall_table[] = {
    (syscall_handler)sys_fork,
    (syscall_handler)sys_exec,
    (syscall_handler)sys_exit,
    (syscall_handler)sys_wait,
    (syscall_handler)sys_open,
    (syscall_handler)sys_close,
    (syscall_handler)sys_read,
    (syscall_handler)sys_write,
    // ...
};

// 系统调用处理
int syscall_handler(int syscall_num, void *args) {
    if (syscall_num >= SYSCALL_COUNT) {
        return -1;
    }
    return syscall_table[syscall_num](args);
}
```
