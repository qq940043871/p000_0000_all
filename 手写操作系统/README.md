## 🎯 学习路线

### 第一阶段：基础知识（1-2周）

### 第二阶段：硬件交互（2-3周）

- MBR引导扇区
- GRUB引导加载器
- 实模式到保护模式

- 内核初始化

### 第五阶段：核心模块（4-6周）
- 文件系统实现
- 进程管理
- 内存管理
- 中断异常处理

### 第六阶段：设备驱动（2-3周）
- 驱动程序基础
- 磁盘驱动
- 网络驱动
- 字符设备驱动

### 第七阶段：实战项目（2-3周）
- 项目规划与设计
- 完整实现
- 测试与调试
- 性能优化

## 📖 核心概念

### 1. 引导过程
```
BIOS → MBR → Bootloader → 内核 → Shell
```

### 2. 内存布局
```
0x00000000 - 0x000FFFFF: 实模式内存（1MB）
0x00100000 - 0xFFFFFFFF: 保护模式内存
```

### 3. 中断向量表
```
0x00-0x1F: CPU异常
0x20-0x2F: 硬件中断
0x30-0xFF: 软件中断
```

### 4. 进程状态
```
就绪 → 运行 → 等待 → 就绪
```

## 🚀 快速开始

### 1. 克隆项目
```bash
git clone <repo-url>
cd 自制操作系统
```

### 2. 安装依赖
```bash
# Ubuntu/Debian
sudo apt-get install build-essential nasm qemu-system-x86

# macOS
brew install nasm qemu
```

### 3. 编译内核
```bash
cd code
make all
```

### 4. 运行系统
```bash
./run.sh
```

### 5. 调试系统
```bash
./debug.sh
```

## 💡 学习建议

1. **循序渐进**：从简单到复杂，不要跳过基础
2. **动手实践**：每个模块都要编写代码并测试
3. **理解原理**：不仅要知道怎么做，更要理解为什么
4. **调试技巧**：学会使用GDB和QEMU进行调试
5. **代码阅读**：阅读Linux等开源OS的源码
6. **文档记录**：记录学习过程和遇到的问题
