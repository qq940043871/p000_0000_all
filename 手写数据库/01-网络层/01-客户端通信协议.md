## 一、MySQL协议概述

```
MySQL协议特点：
  - 基于TCP/IP
  - 客户端-服务端模式
  - 变长包格式
  - 支持prepared statements
```

## 二、协议报文格式

```
MySQL数据包结构：
┌────────────────────────────────────────┐
│  包头（Packet Header）                   │
├────────────────────────────────────────┤
│  长度（3字节）：数据长度                  │
│  序号（1字节）：包序列号                  │
├────────────────────────────────────────┤
│  数据（可变长度）                         │
└────────────────────────────────────────┘

特点：
  - 最大16MB（16777215字节）
  - 超过则分包传输
  - 序列号用于校验顺序
```

## 三、Java实现

```java
public class MysqlPacket {
    private byte[] header = new byte[4];  // 3字节长度 + 1字节序号
    private ByteBuffer payload;
    
    // 读取包
    public static MysqlPacket read(SocketChannel channel) throws IOException {
        MysqlPacket packet = new MysqlPacket();
        
        // 读取包头
        ByteBuffer header = ByteBuffer.allocate(4);
        channel.read(header);
        header.flip();
        
        int length = ((header.get(0) & 0xFF) << 16) | 
                     ((header.get(1) & 0xFF) << 8) | 
                     (header.get(2) & 0xFF);
        byte sequenceId = header.get(3);
        
        // 读取包体
        ByteBuffer payload = ByteBuffer.allocate(length);
        channel.read(payload);
        payload.flip();
        
        packet.payload = payload;
        packet.sequenceId = sequenceId;
        
        return packet;
    }
    
    // 发送包
    public void write(SocketChannel channel, byte[] data, byte sequenceId) 
        throws IOException {
        
        ByteBuffer buffer = ByteBuffer.allocate(4 + data.length);
        
        // 写入长度
        buffer.put((byte) (data.length & 0xFF));
        buffer.put((byte) ((data.length >> 8) & 0xFF));
        buffer.put((byte) ((data.length >> 16) & 0xFF));
        
        // 写入序号
        buffer.put(sequenceId);
        
        // 写入数据
        buffer.put(data);
        
        buffer.flip();
        channel.write(buffer);
    }
}
```

## 四、协议类型

```
MySQL协议包类型：
  0x00: OK包
  0xFE: EOF包
  0xFF: Error包
  1-250: 协议序列

客户端→服务端：
  COM_QUIT: 退出
  COM_QUERY: 查询
  COM_INIT_DB: 选择数据库
  COM_STMT_PREPARE: 预处理
  COM_STMT_EXECUTE: 执行
```
